# MINHWA 민화 — 기법 해설

## 핵심

민화는 손으로 그린 소원입니다. 그림 속 모든 소재에는 뜻이 담겨 있습니다 — 소나무 아래 호랑이와 그 위에서 울어대는 까치(호작도)는 기쁜 소식과 액막이의 소원이고, 만개한 모란(모란도)은 부귀영화의 소원이며, 늙지 않는 열 가지를 해와 달 아래 모은 그림(십장생도)은 가장 깊은 소원, 장수입니다. MINHWA는 조선 후이 이름 없는 화공의 손에서 나온, 소원을 담은 그림들의 갤러리입니다 — 문인화가 다듬은 자리를 거칠게, 문인화가 아낀 자리를 넉넉하게, 마침보다 뜻을 먼저.

설계 베팅: 민화는 수상작 사이에 직접 경쟁자가 없습니다 (갤러리 레퍼런스 문서가 이를 "한국 전통 미학의 갭"이라 이름했습니다). 단청 선(GIWA의 영역)이 아니라 민화 그림 자체를 현대화한 엔트리 — 촌스럽고, 선명하고, 상징적이고, 쾌활한 — 가 이 빈자리를 메웁니다. 페이지는 KEMURI의 엄격한 선(禪)과 정반대의 밝은 결입니다.

## 아트 디렉션

바탕은 오래된 한지 `#f5efe1`이고, 먹 검정 `#1a1410`(흑, 명암비 13.6:1 AAA)이 중요한 모든 글자를 담당합니다. 액센트 체계는 **오방색(오방색)** — 한국 장식 전통의 오색 우주관 — 이지만, 색견본 집합이 아니라 **역할이 있는 체계**로 다룹니다. 각 색은 방향과 계절과 오행과 그 색이 사는 소재를 가집니다:

- `jeok` `#c8362b` (적 · 남/여름/불) — 기능적 빨강: 포커스, 선택, 서명 획 (4.7:1 AA)
- `cheong` `#2f6e6a` (청 · 동/봄/나무) — 소나무, 호랑이 하늘, 작은 글 (5.0:1 AA)
- `hwang` `#d99a1f` (황 · 중앙/흙/임금) — 해, 장수의 금빛; **장식 전용** (2.6:1, 글자 금지)
- `baek` `#fbf8ef` (백 · 서/가을/쇠) — 달, 호분의 숨결, 카드 바탕
- `heuk` `#1a1410` (흑 · 북/겨울/물) — 먹 선, 기능적 글자 전부

세 서체, 세 목소리. **Gaegu**(촌스러운 붓 글씨 마을체)는 워드마크 민화와 소재 제목과 도장입니다. **Noto Serif KR**은 한국어 본문 전부 — 따뜻한 해설사입니다. **Fraunces**는 영문 갤러리 카드 부제입니다. 한국어가 항상 앞서고, 영문은 기울임 동반자로 결코 더 크지 않습니다. 페이지는 한글 우선입니다.

## 기법 — 스탬프와 함께하는 소제 등장 (char-split-reveal + spring-press)

민화의 각 소재는 엄격한 잉크 가라앉음이 아니라 쾌활한 팝으로 등장합니다 — 민화는 쾌활하니까요. 스탬프(motif-stamp: 투명도 0→1, 크기 0.82→1, `cubic-bezier(0.34, 1.56, 0.64, 1)` 살짝 과장, 760ms)가 그림을 마운트에 찍어내고, 그 아래에 소제의 뜻이 글자 단위로 타이핑됩니다 — 큐레이터가 소리 내어 읽듯이. 히어로 호작도가 온전한 처리를 받고, 모란과 십장생 패널은 같은 방식으로 시차를 두고 등장합니다.

환경이 동작 감소일 때 모든 소재는 이미 찍혀 있고 모든 줄은 이미 타이핑돼 있습니다 (팝도, 글자 박자도 없이). 자바스크립트가 꺼져도 페이지는 완성된 민화 갤러리입니다 — 모든 소원이 보이고, 모든 뜻이 캡션으로 달려 있습니다.

## 기법 — 오방색 내비게이터 (팔레트를 체계로)

ObangsaekNavigator는 페이지의 교육적 심장입니다. 다섯 색 중 하나를 선택하면 루트의 `--minhwa-accent-live`가 바뀌어, 포커스 링과 섹션 눈금과 호버 물감이 즉시 다시 물듭니다 — **물감은 이징하지 않습니다.** 각 색의 행은 그 색의 방향·계절·오행·사는 소재를 설명하므로, 팔레트가 스스로를 가르칩니다. 이것이 오방색을 실제 단청 장인이 쓰는 방식 — 우주론적 색 체계로서의 오방색 — 이지 장식이 아닙니다.

## 기법 — 인장 도장 (서명 인터랙션)

민화 화공은 다 그린 그림에 붉은 주황 인장을 찍어 서명합니다 — "이 그림은 내 것, 이 그림은 다 됐다"는 표시입니다. MINHWA는 이것을 UI 확인 패턴으로 바꿉니다. 방문객이 인장 버튼을 누르면(또는 터치 환경·동작 감소에서는 패널이 화면에 들어올 때 스스로 찍히고), 붉은 도장이 모란 섹션 위로 균일한 `ease-press`로 내려와 착지하고, 1px 먹 테두리가 퍼지며 날인을 증명합니다. 도장을 누르는 행위는 물리적 행위라 — 버튼은 유일한 장난기로 `4px 5px 0` 손그림자 오프셋을 가집니다. `useSealStamp` 훅은 터치와 동작 감소에서 스스로 찍히게 해 아무도 날인을 놓치지 않게 하고, 정밀 포인터에서는 직접 누르기를 기다립니다 — 누르는 것이 요점이니까요.

## 접근성

- **동작 감소:** 소제는 이미 찍혀 등장하고, 나레이션은 온전히 타이핑되어 착지하며, 인장은 찍힌 채로 있고, 십장생 스크럽은 마지막 소제에 고정되며, 오방색 물감 교체는 즉시로 유지됩니다 (동작이 아니므로). 하나의 온전히 구성된 민화 갤러리.
- **자바스크립트 없음:** 마운트 시 추가되는 `.minhwa-js` 뒤에 모든 사전 등장 상태가 있습니다. 자바스크립트가 꺼지면 페이지는 완성된 갤러리입니다 — 모든 소원이 보이고, 모든 뜻에 캡션이 있고, 인장은 쉬고 있습니다.
- **대비:** heuk 13.6:1, ink-soft 6.5:1, jeok 4.7:1, cheong 5.0:1 — 전부 AA 이상. hwang(2.6:1)은 장식 전용이거나 heuk 위의 점으로만. baek은 한지 위 글자가 아닙니다.
- **의미론:** 각 민화 패널은 이중언어 `<figcaption>`을 가진 `<figure>`이고, 십장생 소재는 순서 있는 목록이며, 포커스는 오방색을 따라 다시 물드는 2px jeok `:focus-visible` 링입니다.

## 퍼포먼스

- 프레임당 쓰기는 두 CSS 커스텀 프로퍼티(`--minhwa-longevity`는 sticky 무대, `--minhwa-accent-live`는 루트)뿐입니다 — 둘 다 transform/opacity에서 파생됩니다.
- 두 환경적 히어로 주기(해 7.1초, 달 6.3초)는 공약수가 없어, 쉴 때 히어로가 눈에 띄게 반복되지 않습니다. rAF는 화면 밖과 `visibilitychange`에서 멈춥니다.
- 등장은 하나의 IntersectionObserver에서 명령적으로 `class`와 `transitionDelay`를 씁니다 — 스크롤에서 React 재렌더링이 없습니다.
- 세 점의 AVIF 일러스트(약 439 KB)가 5120 KB 한도 안에 있습니다. 의존성 0; 가장 무거운 자산은 세 서체이며, Latin/CJK 서브셋은 next/font에 맡깁니다.

## 리믹스 아이디어

1. **혼례 병풍.** 세 소제를 혼례폭병으로 바꿔 — 각 패널이 부부를 향한 소원이 되고, 인장으로 결합을 "확인"합니다.
2. **꿈의 책.** 방문객이 (원클릭 선택기로) 소제 하나를 "꿔" 페이지가 개인화된 민화 소원을 생성하고 그것에 도장을 찍습니다 — 민화 점사 인터랙션.
3. **접기.** 실제 팔폭 민화 병풍이 가로로 스크롤을 따라 펼쳐지고, 각 패널이 계절을 드러냅니다. 오방색 내비게이터가 계절별로 다시 물듭니다.
