# 그라디언트 플라자 — 제작기

## 훅: 폐점 후에만 존재하는 몰

어딘가에 폐점 후에만 존재하는 쇼핑몰이 있다. 그리고 그 몰의 안내방송
시스템은 "폐점"이라는 부분을 끝내 전달받지 못했다. FM 88.8은 2층 분수
광장에서 밤새 송출된다. 새벽 2시에는 에스컬레이터 재즈, 3시 반에는
푸드코트에 내리는 비, 그리고 영원히 듣지 못할 쇼핑객들을 향한 안내방송.
페이지는 이 설정을 한 번도 설명하지 않는다. 당신은 새벽 3시 33분에
도착하고, 바닥은 이미 끝나지 않는 석양을 향해 달리고 있고, 방송국은
당신이 일부러 남았다고 믿는다.

이 디자인의 도박은 하나다. 베이퍼웨이브는 필터가 아니라 *장소*로 다룰 때
작동한다는 것. 그리드와 창과 안내방송이 전부 하나의 가상 건물에 속해
있으면, 향수는 무드보드이기를 멈추고 서 있을 수 있는 어딘가가 된다.

## 아트 디렉션

베이퍼웨이브는 이 갤러리에서 가장 망치기 쉬운 미학이다. 원본 자체가
디자인된 적 없이 그냥 퇴적된 것이기 때문이다. 그래서 이 엔트리는 고전
키트(그리드, 태양, 네온, 창)를 원본이 가져본 적 없는 절제로 다룬다.
강조색 하나(`#ff71ce` 핑크), "라이브" 색 하나(`#05ffa1` 민트), 그리고
읽어야 하는 것은 전부 라벤더 화이트(15.4:1), 헤이즈(10.0:1),
시안(9.9:1), 민트, 핑크 중 하나로만 — 그보다 어두운 색은 금지라는 강한
규칙. 바이올렛은 5.7:1로 AA를 통과하는데도 라벨과 크롬으로 강등됐다.
5.7:1짜리 긴 문단은 수치가 통과해도 *안개처럼* 느껴지기 때문이다.

서체 셋, 배역 셋. Monoton은 입구 위의 네온관 — 간판은 본문용 서체가
아니므로 디스플레이 크기에서만 쓴다. Gugi는 페이지의 모든 한글을 맡는다.
타일 깔린 아트리움의 스피커 같은 목소리인데, 한국어 카피가 정확히 그
말투로 쓰였다. Azeret Mono는 방송 콘솔 — 편성표, 창 크롬, 본문. 두 작업
스택 모두 로마자 서체 다음에 Gugi를 두어 한글이 마크업 없이 자동으로
떨어진다. 폴백 산세리프 글리프 하나면 픽션 전체가 깨진다.

## 기법 1 — 달리는 바닥 (원근 그리드의 수학)

히어로 그리드는 2D 캔버스이고, 착시의 전부는 나눗셈 하나다. 지평선을
히어로 높이의 58%에 두고, 모든 그리드 가로줄에 "레인 단위"의 깊이 `z`를
준다. 깊이 0의 카메라는 그 줄을 화면의 이 위치에 투영한다:

```ts
// Hero.tsx — d: 화면 아래 끝에서 1 → 지평선에서 0
const d = NEAR / z;                       // NEAR = 0.92
const y = horizon + (h - horizon) * d;
```

가로줄들은 정수 깊이에서 태어나 공유된 `offset`을 타고 다가온다. `z`가
`NEAR` 아래로 떨어진 줄은 카메라를 지나친 것이므로 건너뛴다. `d`가 `z`에
대해 쌍곡선이라 먼 줄은 지평선에 밀집하고 가까운 줄은 화면 아래로
가속하며 빠져나간다 — 그 가속이 곧 원근이고, 이징 커브는 하나도 없다.
세로줄은 더 싸다. 모든 레인이 소실점으로 수렴하므로, 지평선 중앙에서
캔버스 아래 등간격 점들로 뻗는 직선일 뿐이고, 수직 페이드 그라디언트
하나를 `strokeStyle`로 재사용해 부채 전체를 긋는다.

시그니처는 `offset`을 모는 방식이다. `useScrollEnergy` 훅은 스크롤의
*속도*를 — 위치가 아니라 — 측정해 비대칭 lerp(어택 0.16, 릴리스 0.05)로
0→1 에너지로 다듬는다:

```ts
// useScrollEnergy.ts — 몰은 당신을 빨리 알아채고, 천천히 잊는다
const target = Math.min(1, v / V_MAX);    // 2.4 px/ms에서 풀 러시
const k = (target > energy ? ATTACK : RELEASE) * (dt / 16.7);
energy += (target - energy) * Math.min(1, k);
```

```ts
// Hero.tsx — 산책: 초당 0.5레인. 질주: 약 9배, 지평선도 타오른다
offset += (dt / 1000) * (BASE_SPEED + energy * RUSH_SPEED);
```

가만히 있으면 바닥은 스스로 잔잔히 흐른다(터치 기기도 호버 없이 살아
있는 장면을 얻는다). 세게 스크롤하면 바닥이 달리고, 지평선 발광이
`0.55 · energy`만큼 밝아지고, 태양의 헤일로가 넓어진다. 태양 자체는
핑크→바이올렛으로 보간되는 2px 가로 슬라이스로 그리는데, 블라인드 틈에
걸리는 슬라이스는 건너뛴다. 틈의 주기가 `period = 9 + dy · 0.55`라서
지평선 쪽으로 갈수록 저절로 넓어진다 — "썰린 태양" 레시피의 전부다.
별들은 해시된 반짝임 위상을 가진 파티클이고 위치가 시간의 순수 함수라서,
모션 최소화의 정지 프레임도 같은 하늘의 정직한 한순간이다.

## 기법 2 — 던질 수 있는 창 (드래그 물리와 z-순서)

부스는 레트로 OS 창 세 개를 그린다. 자바스크립트가 없으면 문서 흐름에
쌓인 평범한 섹션들이고, 모든 트랙과 라벨이 읽힌다. `.plaza-js`가 붙는
순간 스테이지는 데스크톱이 되고(`position: relative`, 고정 높이) 각 창은
CSS의 스폰 지점을 잡는다. 드래그는 `left`/`top`을 절대 건드리지 않는다.
위치는 ref에 살고 `translate3d`로 쓰이므로, 드래그의 비용은 레이아웃이
아니라 컴포지터가 낸다.

무거운 일은 포인터 캡처가 한다. 타이틀 바 그립이 down에서 포인터를
캡처하므로 커서가 바를 앞질러도 move는 계속 도착한다. 그립에만 —
그립에만 — `touch-action: none`을 주면 엄지로 창을 끌면서도 페이지의
나머지는 그대로 스크롤된다:

```ts
// useDrag.ts — 놓아도 관성이 조금 남는다
const decay = Math.pow(FRICTION, dt / 16.7);   // 정규화 프레임당 0.9
vel.current.x *= decay;
vel.current.y *= decay;
if (Math.hypot(vel.current.x, vel.current.y) > STOP_SPEED) {
  rafRef.current = requestAnimationFrame(step);
}
```

속도는 마지막 ~12ms의 포인터 이동에서 샘플링하고, 놓으면 창이 지수
마찰로 미끄러진다. 스테이지 안에 ±16px 여유로 클램프되고, 벽에 닿으면 그
축의 속도를 0으로 만들어 창이 떨리는 대신 *툭* 부딪힌다. 모션 최소화에선
글라이드를 통째로 건너뛴다. 드래그 자체는 사용자가 모는 입력이므로
남는다 — 손이 멈추는 곳에서 창도 멈출 뿐이다.

z-순서는 카운터가 아니라 raise 스택이다. 부모가 창 id 배열을 들고,
창을 올리면 그 id가 배열 끝으로 이동하며, 각 창의 `z-index`는
`10 + indexOf(id)`다. 끝없이 자라는 `++zIndex`가 없고 렌더는 순수한
상태다. 그립은 진짜 버튼이라 같은 창이 키보드로도 움직인다. 화살표로
24px(Shift는 4px, Home은 제자리), 그립의 이중 언어 `aria-label`이 이를
안내한다. 최소화와 닫기도 진짜 버튼이고, 닫힌 창은 `aria-pressed`가 달린
태스크바로 돌아온다 — 장식적 크롬을 정직하게 만드는 방법이다.

## 기법 3 — 번짐 (필터 없는 색수차)

고전적인 RGB 분리는 보통 SVG `feOffset` 필터 그래프나 캔버스 채널 복사로
만든다. 둘 다 *읽어야 하는* 픽셀을 이펙트 안에 넣는다 — 수학이 조금만
어긋나도 제목이 읽히지 않는다. 이 엔트리는 레이어를 뒤집는다. 기본
글리프는 절대 움직이지 않고, 색수차는 그 위에 얹힌 두 개의 의사 요소
*고스트*다:

```css
/* styles.css — 고스트는 JS가 쓰는 두 커스텀 프로퍼티를 탄다 */
.plaza-ab::before,
.plaza-ab::after {
  content: attr(data-text) / "";     /* 빈 대체 텍스트: 리더에게 침묵 */
  position: absolute;
  inset: 0;
  opacity: var(--plaza-ghost, 0);
  mix-blend-mode: screen;            /* 가산광 — 밝아지기만 한다 */
  pointer-events: none;
}
.plaza-ab::before { color: var(--plaza-cyan); transform: translate3d(calc(var(--plaza-shift) * -1), 0, 0); }
.plaza-ab::after  { color: var(--plaza-pink); transform: translate3d(var(--plaza-shift), 0, 0); }
```

`useScrollEnergy`가 바닥을 모는 것과 같은 에너지로 루트에
`--plaza-shift`(0–7px)와 `--plaza-ghost`(0–0.85)를 쓴다. 숫자 하나가
페이지 전체의 박자를 맞춘다: 질주하면 제목이 시안-왼쪽/핑크-오른쪽으로,
컨버전스 나간 브라운관처럼 번지고, 멈추면 릴리스 커브를 타고 다시
또렷해진다. `screen` 블렌딩이 대비 보험이다 — 고스트는 움직이지 않는
기본 텍스트 위에 빛을 *더할* 수만 있으므로, 최악의 경우는 더 밝은
제목이지 더 어두운 제목이 아니다.

PA 티커가 같은 숫자로 루프를 닫는다. JS 마퀴(복제된 `aria-hidden` 트랙,
절반 너비의 모듈로로 래핑, 스크린 리더용 시각적 숨김 목록)의 속도가
`42 + 300 · energy` px/s다. 산책하면 안내방송이 흘러가고, 달리면 방송이
당신을 쫓아 서두른다.

## 대비의 규율

숫자로 적는다. 베이퍼웨이브는 숫자 없이는 죽는다 (모두 `#1a0b2e` 기준):

- 라벤더 화이트 `#efe6ff` — 15.4:1, 모든 본문
- 민트 `#05ffa1` — 14.0:1, 라이브 마커
- 헤이즈 `#c7b8e0` — 10.0:1, 보조 텍스트와 한국어 글로스
- 시안 `#01cdfe` — 9.9:1, 시각, 버튼, 포커스
- 핑크 `#ff71ce` — 7.5:1, 강조색 (본문도 합법이지만 디스플레이에 쓴다)
- 바이올렛 `#b967ff` — 5.7:1, **라벨과 크롬 전용**

네온 위의 어두운 글자도 검산했다. 타이틀 바 그라디언트 위의 딥 퍼플
텍스트는 핑크 끝에서 7.5:1, 바이올렛 끝에서 5.7:1을 유지하고, 태스크바
버튼은 눌리면 바이올렛 위 보이드로 6.1:1이 된다. 히어로 카피는 래디얼
스크림(`void` 82%) 위에 앉아, 가장 밝은 그리드 선이 밑을 지나가도
본문 크기 텍스트가 AA를 지킨다.

## 성능 노트

- rAF 루프는 총 셋(그리드, 티커, 비주얼라이저)이고, 전부
  IntersectionObserver로 화면 밖에서, `visibilitychange`로 탭이 숨을 때
  멈춘다. 스크롤 에너지 루프는 더 엄격하다: 첫 스크롤 이벤트에 깨어나고
  에너지가 0.004 아래로 빠지면 스스로 주차한다 — 가만히 있는 페이지는 0
  프레임을 쓴다.
- 캔버스는 DPR 2 상한. 그리드는 가로줄 ~26개, 그라디언트 하나를
  재사용하는 세로줄 부채, 별 사각형 ~150개, 태양 슬라이스 ~100개를
  그린다 — 중급 하드웨어에서 1ms 아래다. 비주얼라이저는 색으로 배치를
  묶는다: 민트 막대 전부, 그다음 핑크 캡 전부 — 프레임당 `fillStyle`
  대입 두 번.
- 창은 오직 `transform`으로만 움직이고, 올리기는 상태 변화에 따른
  `z-index` 쓰기이며, 레이아웃 속성을 애니메이션하는 곳은 어디에도 없다.
- 커스텀 프로퍼티 쓰기(`--plaza-shift`, `--plaza-ghost`)는 에너지 루프
  안에서 프레임당 최대 한 번이고, 그것이 먹이는 속성은 승격된 의사
  요소의 transform과 opacity다.

## 접근성 노트

- **모션 최소화:** 그리드는 조합된 정지 프레임 하나를 그리고(손으로 고른
  `t`의 순수 함수라 빈 화면이 아니라 진짜 프레임이다), 색수차 고스트는
  `content: none`과 0으로 만든 변수로 제거되고, 티커는 시작하지 않고,
  램프와 프로그레스 애니메이션은 벗겨지고, 드래그 관성은 꺼진다. 드래그
  자체는 남는다 — 방문자가 움직일 때만 움직인다.
- **No-JS:** `.plaza-js`가 데스크톱 배치, 캔버스, 티커 모션을 게이트한다.
  없으면 창들은 CSS 그라디언트 하늘 위에 흐름대로 쌓이고 페이지의 모든
  단어가 그대로 읽힌다.
- **시맨틱:** 두 캔버스 모두 이중 언어 설명이 달린 `role="img"`.
  움직이는 티커 트랙은 `aria-hidden`이고 시각적 숨김 목록이 접근 가능한
  사본이다. 그립, 최소화, 닫기, 태스크바, 트랙 행은 전부 네이티브 버튼.
  재생 중 트랙은 `aria-current`, 창의 표시 여부는 정직한 `hidden`.
- **포커스:** 핑크 헤일로를 두른 시안 `:focus-visible` 링 — 어두운
  스테이지에서도, 네온 타이틀 바 안에서도 보인다.

## 리믹스 아이디어

1. **심야 여객선 터미널.** 뼈대는 그대로, 건물만 바꾼다: 그리드 바닥은
   달빛 길이 놓인 어두운 바다가 되고, 창은 DEPARTURES.EXE와 조수
   비주얼라이저가 되고, PA는 1998년에 발길이 끊긴 승객들을 위한 출항을
   안내한다. 핑크를 시그널 오렌지로 바꾸되 규율은 유지할 것.
2. **24시 빨래방 FM.** 비주얼라이저는 드럼 회전 레이더가 되고, 편성표는
   탈수 사이클을 프로그램으로 올리고, 색수차는 스크롤 대신 *세탁기의*
   박자 — 4.2초마다 오는 느린 맥동 — 에 반응한다. 좋은 연습이다: 에너지
   봉투는 남기고 입력만 바꾸는 것.
