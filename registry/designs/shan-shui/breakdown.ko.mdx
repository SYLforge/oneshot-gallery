# 산수 — 제작기

## 훅: 스스로를 그리는, 끝나지 않는 그림

그림 하나가 스스로를 그리고, 결코 끝나지 않는다. 관객은 언제나 그렇듯,
한참 진행된 두루마리의 한가운데에 도착한다 — 산은 이미 거기 있었다 —
이 두루마리가 요구하는 것은 단 하나, 계속 읽어 내려가는 것이다. 왜냐하면
읽는 행위 자체가 중국식 손두루마리(手卷)를 펼치는 행위이기 때문이다. 산은
작아지는 것이 아니라 옅어지면서 멀어진다, 실제 산능선이 안개 속에 녹아드는
그대로. 안개는 골짜기에 고이고, 거의 알아채지 못할 만큼 살며시 당신의
손을 향해 기운다. 그리고 두루마리 어딘가 깊은 곳에서 날씨는 봄의 매화안개에서
겨울의 잠긴 구름으로, 한 번의 단절도 없이 넘어간다. 발기(跋)에 다다르면
페이지는 정직하게 말한다. 당신이 본 것은 그 어떤 것도 그림이 아니었다.
산맥은 값잡음의 합이었고, 안개 한 줌은 방사형 그라디언트였으며, 전경의
소나무 한 그루는 재귀적 붓터치였다. 그림은 곧 코드다.

이 디자인의 도박은 하나다. 이미지 예산이 0바이트인 밝은 페이지에서,
*절차적으로 생성된 산수*가 어떤 정적 사진보다 깊은 명상의 무게를 지닐 수
있다는 것. 산 사진은 멈춰 있는데, 두루마리야말로 사진이 결코 될 수 없는
단 하나 — 끝이 없는 것 — 이기 때문이다.

## 벤치마크, 그리고 이 엔트리가 하는 일

링둥(LingDong)의 `{Shan, Shui}*`(https://github.com/LingDong-/shan-shui-inf)는
"코드가 그림이 될 수 있는가?"라는 질문에 대한 정전(正典)이다. 산과 나무,
안개와 폭포를 잡음에서 합성하여 **SVG**로 그려 내는, 끝없이 펼쳐지는 중국
산수. 널리 존경받는 이 작업과 이 엔트리는 정중한 대화 속에 있다. 세 가지
의도적 차이가, 적어도同等의 솜씨로 느껴지기를 겨눈다.

1. **SVG가 아니라 캔버스.** 수묵의 진짜 매체는 종이 위의 젖은 먹이다.
   캔버스의 방사형 그라디언트 안개, 번진 산능선 윗면, 깃털처럼 스며든
   수풀은 물감으로 읽힌다. SVG 실루엣은 벡터로 읽힌다. 맞바꾼 것은
   해상도 독립성이고 얻은 것은 공기감 — 이 단점은 받아들이고 발기에
   적었다.
2. **잉크 농도가 곧 깊이.** 다섯 겹의 산맥, 가장 먼 것부터. 이것은
   远山如黛(원산여대, 먼 산은 눈썹먹 같다)를 — 거리는 원근이 아니라 먹의
   알파라는 북송의 원리를 — 채우기 색의 알파 채널에서 문자 그대로 실현한다.
3. **반응한다.** 벤치마크는 고정되어 자동 재생되는 두루마리다. 이쪽은
   숨 쉰다. 스크롤이 곧 펼침이고(페이지 스크롤이 잡음의 world-x 오프셋을
   모는데), 안개는 포인터를 향해 기울고, 날씨는 읽는 동안 사계절을 천천히
   돈다.

## 아트 디렉션

화지·수묵은 절제에서 산다. 바탕은 크림색 쌀종이(`#f4ede0`), 목소리는
먹(`#1a1714`, 15.3:1), 따뜻한 색은 정확히 하나 — 주홍 `#a83232` — 그리고
그 색은 도장이 실제로 찍히는 자리에만 나타난다. 낙관, 하나의 악센트, 그림
속으로 들어가는 화가. 두 번째 따뜻한 색이 끼어드는 순간 이 페이지는
관광 엽서가 된다.

어려운 일은 제약 두 개가 대신 해 줬다. 첫째, 주홍은 화지 위에서 5.7:1
(큰 텍스트와 비텍스트에 대해 AA)이지만 먹 위에서는 2.7:1에 불과하다.
그래서 어두운 발기에서는 장식용으로만 쓰고(도장이라는 사물), 일하는
토큰 `vermillion-deep #8a2626`(7.6:1)을 양쪽 바탕에서 기능해야 하는 자리 —
아이브로, 포커스 링, `::selection` 바탕 — 위해 파생했다. 둘째, 서체는
뻔한 짝을 거절했다. 코모란 가라몬드는 문인의 손이 붓을 잡는 방식 —
가볍게 — 그대로 영어를 말하고, 마산정(馬善政, 硬筆行楷)은 모든 중국어
디스플레이 글리프를 곧게 세운다. 본문 크기에서 붓체가 읽히지 않는 자리는
노토 세리프 SC가 맡는다. 스택 순서 덕에 거대한 山水는 마크업 없이도
라틴 서체를 통과해 붓체에 도착한다. 두 언어는 크기가 아니라 자세로
구분된다 — 영어는 기울고, 서예는 결코 기울지 않는다.

## 기법 1 — 값잡음, 산능선, 그리고 거리로서의 먹 농도

산을 그리는 순진한 방법은 폴리라인을 손으로 짜는 것이다. 그러면 로고처럼
읽힌다. 진짜 산능선은 스케일을 가로질러 자기유사하다 — 봉우리엔 봉우리가
있고, 그 봉우리엔 다시 봉우리가 있다. 그 구조가 바로 프랙탈 브라운
운동(fBm)이 주는 것 — 여러 옥타브의 매끄러운 잡음을 배가하는 주파수와
반감하는 진폭으로 합친 것:

```ts
// hooks/landscape.ts — 산능선 프로필은 값잡음 네 옥타브의 합
const OCTAVE_FREQS = [1, 2.05, 4.4, 9.1];
const OCTAVE_WEIGHTS = [1, 0.52, 0.27, 0.13];

function ridgeHeight(layer, x, scrollX, stageH) {
  const wx = (x + scrollX) * layer.freq + layer.phase;
  let h = 0, wSum = 0;
  for (let o = 0; o < OCTAVE_FREQS.length; o++) {
    h += (valueNoise(wx * OCTAVE_FREQS[o], layer.phase * OCTAVE_FREQS[o]) - 0.5)
         * OCTAVE_WEIGHTS[o];
    wSum += OCTAVE_WEIGHTS[o];
  }
  h /= wSum;
  // 능선 날카롭게: 0.5를 기준으로 반사 — 뾰족한 봉우리, 둥근 골
  const ridge = 0.5 - Math.abs(h);
  return layer.baseY * stageH - ridge * layer.amp * stageH * 2.2;
}
```

두 디테일이 이것을 팔아 낸다. 첫째, `0.5 - Math.abs(h)` 능선 변환. 날것의
fBm은 구르는 언덕을 만들지만, 산의 실루엣은 뾰족한 봉우리와 둥근 골짜기를
가진다. 잡음을 중간값 기준으로 반사시키면 주파수 내용은 바꾸지 않고
봉우리만 날카로워진다 — "산처럼 보이게" 만드는 가장 싼 트릭. 둘째,
치환 테이블은 시드(`1368`, 명나라 연호)로 결정되어, 두루마리가 매 방문마다
*같은 그림*이다. 결정성이 산수와 화면보호기의 차이다.

그리고 산수의 원리 자체가 알파 채널에서 실현된다:

```ts
// hooks/landscape.ts — 다섯 겹, 먹 농도가 곧 깊이다
const RIDGE_LAYERS = [
  { depth: 0, ink: 0.14, rgb: [154,148,136] }, // pale-wash — 녹아드는
  { depth: 1, ink: 0.26, rgb: [125,117,106] }, // ink-soft
  { depth: 2, ink: 0.42, rgb: [90,82,73]   },  // ink-mid
  { depth: 3, ink: 0.62, rgb: [58,52,44]   },  // ink-strong
  { depth: 4, ink: 0.82, rgb: [26,23,20]   },  // sumi — 전경
];
```

远山如黛 — "먼 산은 눈썹먹 같다". 가까운 층일수록 짙고, 높고, 종이의 아래쪽에
자리잡는다. 가장 먼 산능선은 거의 종이 그 자체다. 그것이 다섯 개의
`rgba()` 채우기로 표현된 공기 원근이며, 이 페이지 전체의 깊이 모델이다.
어디에도 원근 투영은 없다.

스크롤은 world-x 오프셋을 모는데, 각 층은 깊이에 비례해 시차 이동한다
(`worldX × (0.4 + depth·0.16)`), 먼 산능선이 가까운 것보다 느리게 흐르도록 —
투영 없이 깊이를 강화한다. `ridgeHeight`가 world-x의 순수 함수이므로,
같은 호출이 양쪽 방향으로 영원히 두루마리를 생성한다. 손두루마리가 문자 그대로
끝이 없는 것은 그것이 에셋이 아니라 함수이기 때문이다.

## 기법 2 — 포인터에 반응하는 안개

안개는 중국 산수의 영혼이다 — 산이 숨 쉬는 자리다. 약 26개의 안개 덩어리는
각각 부드러운 방사형 그라디언트(쌀종이색, 투명하게 잦아들며)이며, 두 산능선
층 *사이의* 골짜기에 고인다 — 첫 프레임이 이미 구성되도록 숨결 중간에 태어난다:

```ts
// LandscapeCanvas.tsx — 안개는 골짜기에 고이고, 손을 향해 기운다
const grad = ctx.createRadialGradient(m.x, m.y, 0, m.x, m.y, m.r);
grad.addColorStop(0,    `rgba(244,237,224,${alpha})`);
grad.addColorStop(0.45, `rgba(244,237,224,${alpha * 0.5})`);
grad.addColorStop(1,    `rgba(244,237,224,0)`);
```

두 힘이 안개를 움직이되, 어느 쪽도 결코 조종하지는 않는다. 첫째는 자율적인
두 사인 바람(`7·sin(0.13t) + 4·sin(0.31t+0.7)`, 주기는 서로 소)이 덩어리를
수평으로 흐르게 한다 — 터치 기기에서, 혹은 손이 없을 때, 이것이 유일한
움직임이며 그것으로 충분하다. 둘째는 포인터다. 정밀 포인터는 각 덩어리가
*고이는* 자리(수직으로, 손의 y를 향해)를 편향시키는데, 에너지 값은 비대칭으로
보간된다 — 어택 0.045, 릴리즈 0.016 (60fps 프레임당):

```ts
// LandscapeCanvas.tsx — 그림은 손을 알아채고, 천천히 잊는다
const reach = Math.exp(-(dx * dx) / (W * W * 0.08)); // 가까운 덩어리만
targetY = m.ry + (ptr.y - m.ry ... ) * reach * PTR_MIST_PULL;
const k = tgt.e > ptr.e ? PTR_ATTACK : PTR_RELEASE; // 비대칭
ptr.e += (tgt.e - ptr.e) * k * fr;
```

반감기(들어갈 때): 약 15프레임(0.25초). 반감기(나갈 때): 약 40프레임(거의
3초). 안개는 그림이 누군가 가까이 다가온 것을 알아채리만큼 민감하고, 연기가
기류를 잊듯 잊어버린다. 스크롤 속도는 상한이 있는 수평 돌풍(±60 px/s)을 더해,
휙 넘기면 골짜기 사이로 보이는 날씨가 흐른다 — 하지만 흐름에 맛을 낼 뿐,
결코 덮어쓰지는 않는다. 이것이 `pointer-parallax`이자 `canvas-particles`
기법이다: 행동이 겹으로 쌓인 입자장(자율 바람 + 포인터 당김 + 스크롤 돌풍),
각 힘은 상한이 있어 합이 항상 날씨로 머무르고 커서 장난감이 되지 않는다.

## 기법 3 — 스크럽되는 시차, 그리고 사계절

스크롤 가이드(02절)가 `scroll-scrub-pinned`의 순간이다. 하나의 무대가
네 글자 — 遮山如黛 — 를 네 깊이와 크기, 불투명도에 놓고, 스크롤이 지나갈
때 그것들이 갈라진다. 작동 원리는 하나의 사용자 정의 속성이다:

```ts
// ScrollGuide.tsx — 스크롤에서 숫자 하나를 쓴다, 레이아웃 기록 없이
const update = () => {
  const rect = art.getBoundingClientRect();
  const travel = window.innerHeight + rect.height;
  const p = Math.max(0, Math.min(1, (innerHeight - rect.top) / travel));
  art.style.setProperty("--p", p.toFixed(3));
};
const onScroll = () => { if (!raf) raf = requestAnimationFrame(update); };
```

CSS는 `--p`를 순수 transform 수학에서 소비한다 — 각 층이 서로 다른 배수로
이동하고 축적되어, 글자 네 개가 읽는 동안 깊이로 갈라진다. 그것은 이 절이
설명하는 삼원(三遠) 원리를 행위로 보여 주는 것이다. 스크롤에 따른 React
재렌더링도, 닿은 레이아웃 속성도 없고, 한 요소에 대한 rAF 게이트 쓰기 하나뿐.
감소 모션에서는 `--p: 0.5`에서 쉰다 — 갈라짐 중간의 정물.

두루마리 전체를 덮는 것은 페이지에서 가장 느린 움직임: 사계절. world-x
위치가 끝없는 봄→여름→가을→겨울 순환(경계당 2400 world-px) 위의 한 점으로
대응되고, 인접한 두 계절이 0..1 인자로 *혼합*된다 — 안개 밀도, 먼 산능선의
퇴색, 희미한 하늘 물들임, 드문 계절 악센트(봄의 주홍 매화 점, 겨울의
떨어지는 눈발)가 모두 교차 페이드된다. 날씨는 결코 단절되지 않는다; 날씨가
바뀌는 방식으로 바뀐다. 한 바퀴는 약 9600 world-px — 결심한 독자라면 사계를
모두 볼 수 있을 만큼 짧고, 무심한 독자에게는 날씨로 보일 만큼 길다.

## 모션 수학

세 숫자가 산수에 기질을 부여한다:

- **포인터 어택 0.045 / 릴리즈 0.016.** 60fps로 정규화된 안개 에너지 보간.
  반감기(들어갈 때) ≈ 15프레임, (나갈 때) ≈ 40프레임. 주의는 한 호흡에
  도달하고, 먹 냄새가 방을 떠나듯 빠져난다.
- **바람 주기 0.13 / 0.31 rad/s.** 약 48초와 20초, 서로 소(공약수 없음)라
  세션 안에서 자율 드리프트가 눈에 띄게 반복되지 않는다.
- **계절 경계 2400 world-px.** 보통 스크롤 속도에서 계절당 약 30초 — 날씨로
  느껴질 만큼 느리고, 결심한 독자가 네 개를 모두 볼 만큼 빠르다.

나머지는 모두 두 이징을 따른다: `ease-scroll`(`cubic-bezier(0.22,1,0.36,1)`)은
이동을 앞쪽에 몰아넣고 꼬리를 아올 pixel에 정착시켜, 900~1100ms가 느림이
아니라 차분함으로 읽힌다. `ease-stone`(`cubic-bezier(0.65,0,0.35,1)`)은
대칭적이고 무게감 있는, 두 물리적 동작 — 낙관 찍기와 버튼 먹 채우기 — 를
위한 것이다. 앤비언트 순환(안개 숨결 6.8초, 힌트 드리프트 11.3초)은 서로,
그리고 두 캐든스와도 서로 소다.

## 성능 메모

- 산수를 위한 rAF 루프 하나, ~64fps로 스로틀(`FRAME_MS = 15.5`), `dt`는
  48ms로 클램프되어 백그라운드 탭에서 돌아올 때 욱신이지 않는다. 루프는
  IntersectionObserver(120px 여백) *와* `visibilitychange` 양쪽으로 멈춘다 —
  화면 밖이나 탭 숨김에 프레임 0.
- DPR은 2로 상한. 산능선 층 하나당 `beginPath` 1회 + 약 W/6 선분 + `fill` 1회
  (5층), 젖은 먹 테두리를 위한 번진 윗면 워시 1회. 안개는 방사형 그라디언트
  26개. 나무는 world 타일별로 캐시(결정적)되고 보이는 타일 범위만 그려진다.
- `--p`를 위한 스크롤 리스너는 rAF 게이트이며 사용자 정의 속성 하나를 쓴다;
  스크롤 속도 훅은 ref에 기록만 하고 React를 재렌더링하지 않는다.
- 화지 결은 정적 `feTurbulence` 사각형 하나 — 이 페이지 어디서도 필터가
  애니메이션되지 않는다.
- 모든 CSS 애니메이션은 `transform`, `opacity`, `filter`. 리빌은 관찰자
  콜백 하나에서 `class` + `transitionDelay`를 쓴다. 페이지는 마운트 후 정확히
  두 번 재렌더링된다: 로더 리프트 시작, 로더 언마운트.

## 접근성 메모

- **감소 모션:** 로더는 결코 마운트되지 않고(`display: none`까지 벨트로),
  산수는 웜업하여 하나의 정물(안개 숨결을 멈춘 채, 매화봄 중간)을 그리고,
  시차 층은 `--p: 0.5`에서 쉬며, 미디어 쿼리가 모든 트랜지션과 앤비언트
  순환을 평평하게 한다. 결과는 멈춘 비디오가 아니라 완성된 그림이다.
- **No-JS:** 모든 사전 리빌 상태는 마운트 시 추가되는 `.shan-js` 뒤에
  숨겨진다. SSR 페이지는 완성된 페이지 — 쌀종이 결, 정적 카피, 낙관 자국이
  스크립트 없이도 선다.
- **의미론:** 캔버스 무대는 이중언어(영/중) 설명을 단 `role="img"`;
  스플릿 텍스트는 시각적으로 숨겨진 원문을 유지; 워드마크는 `aria-label`을
  가진 `h1`; 낙관은 `aria-live` 카운트를 가진 진짜 `<button>`; 장식 낙관과
  힌트는 `aria-hidden`.
- **키보드와 터치:** 낙관 버튼과 되감기 링은 `:focus-visible`에 2px
  vermillion-deep 고리 — 발기에서는 vermillion. 안개는 터치에서 자체
  흐른다; 캔버스는 정밀 포인터에서만 포인터 이벤트를 잡아 터치는 항상
  스크롤된다.
- **대비:** sumi 15.3:1, ink-strong 10.6:1, ink-mid 6.6:1, vermillion-deep
  7.6:1 (xuan 위); xuan 15.3:1, xuan-dim 11.7:1 (sumi 위). pale-wash,
  mist, sumi 위의 vermillion은 결코 의미를 담지 않는다. 워드마크는
  종이색 text-shadow를 가져 지나가는 안개 아래에서도 ≈ 8:1로 읽힌다.

## 리믹스 아이디어

1. **세로 축(立軸, 입축).** 축을 뒤집어라: world-y가 생성을 모고, 산이
   위로 쌓이며, 안개는 고이는 대신 떨어진다. 두루마리는 한꺼번에 보이는
   하나의 구도가 된다 — 송나라 입축 형식, 가로가 긴 것이 두루마리라면
   세로가 긴 것은 입축이다.
2. **산행도(行旅圖).** 결정적으로 타일마다 배치된, 작은 인물(한 획)이
   스크롤에 따라 전경 산능선을 걷게 하라 — 범관(范寬)의 모티프, 인간이
   산 앞에서 하나의 구두점이 되는.
3. **강에 내리는 비.** 두 번째 입자장 — 스크롤 속도에 반응해 각도가
   바뀌는 사선 빗줄기, 밑단에 암시된 수면에 파문으로 고이게. 여름이
   장마가 되고, 안개는 보슬비로 짙어진다.
