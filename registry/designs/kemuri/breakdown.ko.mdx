# 케무리 — 제작기

## 훅: 향이 아니라 시간을 파는 가게

교토에 1927년부터 같은 불을 피워 온 공방이 있다. 이 집은 향을 팔지
않는다. 향 한 대가 다 타는 데 걸리는 한 시간 — 그 시간을 판다. 페이지를
열면 향로에는 이미 불이 붙어 있다. 방문자는 언제나 그렇듯, 한 시간의
한가운데에 도착한다. 화면에서 움직이는 것은 서두르기를 거절하는 한 줄기
연기뿐이고, 나머지 전부는 그 속도에 당신을 맞추려는 장치다. 향시계의
눈금대로 한 글자씩 나타나는 시, 물에서 이 초에 걸쳐 가라앉는 그림,
그램이 아니라 시간에 값을 매긴 차림표.

이 디자인의 도박은 하나다. 이미지 예산이 0인 밝은 페이지에서, *시뮬레이션된
연기*가 어떤 사진보다 짙은 공기를 만들 수 있다는 것. 연기 사진은 멈춰
있는데, 멈춤이야말로 연기가 결코 하지 않는 단 한 가지이기 때문이다.

## 아트 디렉션

화지·수묵은 절제에서 살고 절제에서 죽는다. 바탕은 따뜻한 종이(`#efe7d8`),
목소리는 먹(`#1c1814`, 14.4:1), 따뜻한 색은 정확히 하나 — 잉걸불
`#c96f2e` — 그리고 그 색은 실제로 불이 있는 자리에만 나타난다. 향의 끝,
낙관, 버튼을 채우는 먹물. 두 번째 따뜻한 색이 끼어드는 순간 이 페이지는
카페 메뉴판이 된다.

어려운 일은 제약 두 개가 대신 해 줬다. 첫째, 브리프의 잉걸불과 금박은
화지 위에서 둘 다 AA에 미달한다(2.9:1, 2.5:1). 그래서 밝은 바탕에서는
사물로 강등시키고 — 낙관은 물건이지 문장이 아니다 — 일하는 토큰 둘을
파생시켰다. 물을 탄 먹 `ink-soft #4f4a43`(7.1:1)이 모든 보조 텍스트를,
재를 덮어 둔 불 `ember-deep #a04f16`(4.7:1)이 기능해야 하는 자리 —
아이브로, 차림표의 시각, 포커스 링, `::selection` — 를 맡는다. 잉걸불이
제 목소리를 온전히 되찾는 곳은 페이지 끝의 어두운 방 단 한 군데다.
먹 위에서는 4.9:1이 나온다.

둘째, 서체는 뻔한 짝을 거절했다. EB 가라몬드는 이 집이 영어를 말하는
방식 — 좋은 펜으로 천천히 — 그대로이고, 젠 올드 민초는 페이지의 모든
일본어 글리프를 곧게 세워 든다. 스택 순서(`EB Garamond, Zen Old Mincho,
serif`) 덕분에 한자와 가나는 마크업 없이도 라틴 서체를 통과해 민초에
도착한다. 두 언어는 크기가 아니라 자세로 구분된다. 영어는 기울고,
민초는 결코 기울지 않는다.

## 기법 1 — 컬 노이즈, 혹은 이 연기가 색종이처럼 보이지 않는 이유

연기를 움직이는 순진한 방법은 입자마다 무작위로 배회시키는 것이다.
그러면 정전기나 눈발처럼 읽힌다. 독립적인 무작위 걸음에는 *구조*가
없기 때문이다. 진짜 연기는 유체다. 이웃한 덩어리는 함께 움직이고,
소용돌이는 생겨나 한동안 머물고, 결정적으로 — 흐름은 압축되지 않는다.
입자들이 뭉쳐 덩어리지거나 찢어져 구멍이 나는 일이 없다.

유체 솔버 없이도 그 전부를 얻을 수 있다. 매끄러운 스칼라 장 ψ(퍼텐셜)
하나를 잡고, 그 기울기를 90도 돌린 것을 속도로 삼으면 된다:

```ts
// SmokeCanvas.tsx — 발산 없는 흐름: 속도 = (∂ψ/∂y, −∂ψ/∂x)
const cx =
  (psi(x, y + CURL_EPS, t) - psi(x, y - CURL_EPS, t)) / (2 * CURL_EPS);
const cy =
  -(psi(x + CURL_EPS, y, t) - psi(x - CURL_EPS, y, t)) /
  (2 * CURL_EPS);
```

이것이 ψ의 2차원 *컬*이고, 회전시킨 기울기는 정의상 발산이 0이다:
∂vx/∂x + ∂vy/∂y = ∂²ψ/∂x∂y − ∂²ψ/∂y∂x = 0. ψ에 어떤 노이즈를 부어
넣든, 그 결과로 나오는 흐름은 면적을 보존한다. 이 장을 따라 흘러가는
입자들은 퍼텐셜의 등고선을 영원히 감아 돌 뿐 절대 압축되지 않는다.
연기와 색종이의 차이가 이 성질 하나다.

ψ 자체는 시드가 고정된 밸류 노이즈 두 옥타브(결의 크기 약 150px과
56px)인데, 착시 전체를 팔아 치우는 트릭이 하나 들어 있다:

```ts
// SmokeCanvas.tsx — 노이즈 장 자체가 연기와 함께 떠오른다
function psi(x: number, y: number, t: number): number {
  const yr = y + t * FIELD_RISE; // 26 px/s
  return (
    valueNoise(x * EDDY_LARGE, yr * EDDY_LARGE) - 0.5 +
    (valueNoise(x * EDDY_SMALL + 37.2, yr * (EDDY_SMALL * 1.35) + 11.8) - 0.5) * 0.45
  );
}
```

노이즈 도메인의 y를 `t · 26px/s`만큼 밀어 올리면 소용돌이 자체가 위로
흐른다. 이게 없으면 연기 기둥은 얼어붙은 난류 사이를 헤엄치는 꼴이
되어 밧줄을 타고 오르는 것처럼 보인다. 이게 있으면 소용돌이 하나하나가
제가 속한 연기와 함께 태어나서, 떠오르고, 사라진다.

입자들(`Float32Array`, 무대 면적에 따라 약 260–820개)은 장에 곧바로
복종하지 않는다. 초당 2.4의 속도로 장의 속도를 향해 느슨하게 다가갈
뿐인데, 그 느슨함이 관성이고, 관성이 무게다. 그리고 바깥의 힘 둘이
리본을 조종하지 않으면서 구부린다:

- **스크롤.** `useScrollVelocity` 훅은 px/ms 단위의 생 샘플만 기록한다.
  rAF 루프는 140ms보다 오래된 샘플을 "멈췄다"로 읽고, 지연값을
  `속도 × 210px/s`(상한 ±190)를 향해 lerp한 뒤 높이에 비례해 아래로
  건다. 빠르게 스크롤하면 기둥은 옮겨진 찻잔 뒤의 연기처럼 휘고, 멈추면
  당신이 짠 애니메이션이 아니라 스무딩 상수의 속도로 회복한다.
- **포인터.** 커서를 향한 수평 당김(상한 ±54px/s)이 에너지 값에
  곱해지는데, 이 에너지의 lerp가 비대칭이다 — 어택 0.055, 릴리스
  0.018(60fps 기준). 연기는 사분의 일 초 만에 당신의 손을 알아차리고,
  잊는 데는 삼 초를 쓴다. 터치 기기이거나 포인터가 2.6초 침묵하면
  주기 약 30초와 13초(공약수 없음)의 사인 두 개로 만든 미풍이 도로
  넘겨받는다.

기둥은 부드러운 방사형 스프라이트 세 장으로 그린다. 갓 태어난 연기는
더 짙은 먹 스프라이트로 그리다가 흩어질수록 옅은 회색으로 크로스페이드
되고, 입자 수명의 첫 반 초 동안은 잉걸불의 반짝임이 함께 올라탄다 —
숯의 빛이 연기 속으로 기어오르는 순간이다. 첫 페인트 전에 85ms 스텝
120번을 미리 돌려 두므로, 연기가 태어나는 장면은 아무도 보지 못한다.
당신이 도착하기 전부터 타고 있었던 것이다.

## 기법 2 — 가라앉는 먹

세 폭의 그림은 이미지가 아니다. 각각이 층층이 쌓은 벡터 먹이다 — 젖은
가장자리는 반경을 달리한 `feGaussianBlur`로, 공기원근은 불투명도로,
세 번째 그림의 연기 획은 같은 획을 세 번(굵기 13/6/2.2, 점점 짙게)
그어 먹을 머금은 붓이 제 자리를 되짚는 것처럼. 그 위에 리빌이 얹힌다:

```tsx
{/* InkPlates.tsx — 판마다 제 난류가 변위 맵을 먹인다 */}
<filter id="kemuri-p1-settle" x="-14%" y="-14%" width="128%" height="128%">
  <feTurbulence type="fractalNoise" baseFrequency="0.011 0.017" numOctaves="2" seed="19" result="w" />
  <feDisplacementMap in="SourceGraphic" in2="w" scale="0" xChannelSelector="R" yChannelSelector="G" />
</filter>
```

마크업은 `scale="0"` — 이미 가라앉은 상태 — 로 배송된다. 그것이 이
페이지의 no-JS 상태이자 모션 최소화 상태다: 완성된 그림. 자바스크립트가
마운트된 뒤에야 혼돈이 끌어올려지고, 판이 뷰포트에 들어서는 순간에만
가라앉는다:

```ts
// InkPlates.tsx — 움직이는 숫자는 단 하나: 변위 스케일
const tick = (now: number) => {
  const p = Math.min((now - t0) / SETTLE_MS, 1);
  const ease = 1 - (1 - p) * (1 - p) * (1 - p); // 큐빅 아웃 — 한숨처럼
  fe.setAttribute("scale", String(SETTLE_FROM * (1 - ease)));
  if (p < 1) rafs.add(requestAnimationFrame(tick));
};
```

성능의 모양새가 중요하다. `baseFrequency`와 `seed`는 절대 바뀌지
않으므로 브라우저는 난류를 한 번만 래스터화해서 캐시하고, 가라앉는
동안 매 프레임이 지불하는 비용은 변위 패스뿐이다. 만지고 싶어지는
손잡이인 `baseFrequency`를 대신 움직이면 노이즈가 매 프레임 다시
계산되어 중급 하드웨어에서 버벅인다. 스케일 110에서 그림은 날씨였다가,
0에서 먹이 된다. 큐빅 이징 덕에 "알아볼 수 있는" 지점은 일찍 지나가고,
1900ms의 대부분은 마지막 몇 픽셀에 쓰인다 — 젖은 먹이 마르는 방식
그대로.

## 기법 3 — 향시계의 캐던스

스플릿 텍스트는 모양이 잘 알려진 접근성 함정이다. 글리프 span은 스크린
리더에서 낱글자 샐러드로 읽힌다. 이 페이지의 모든 스플릿 문자열 —
워드마크, 세로쓰기 시 — 은 원문을 접근성 트리에 남겨 두고 사본만
움직인다:

```tsx
{/* Cadence.tsx — 보조기술에는 진짜 문자열을, 눈에는 aria-hidden 글리프를 */}
<span className="kemuri-sr">{text}</span>
<span aria-hidden="true" className="kemuri-cadence__row">
  {glyphs.map((g, i) => (
    <span className="kemuri-cadence__g"
      style={{ "--kd": `${start + i * step}ms` } as CSSProperties}>
      {g}
    </span>
  ))}
</span>
```

글리프마다 제 지연 시간을 커스텀 프로퍼티로 들고 있고, 리빌 시점에
자바스크립트가 하는 일은 컨테이너 하나에 클래스 하나를 붙이는 것이
전부다 — IntersectionObserver는 글리프를 건드리지 않는다. 거기서부터는
CSS가 안무 전체를 진행한다. `transition-delay: var(--kd)`가 글리프를
차례로 세우고(시에서는 글자당 60ms, 세로줄당 620ms — 어떤 UI 관습보다
일부러 느린, 향시계의 박자), 리빌 이전 상태(투명도 0, 5px 블러,
0.16em의 낙하)는 루트의 `.kemuri-js` 클래스 아래에만 존재한다.
자바스크립트가 없으면, 숨겨지는 시도 없다.

시의 일본어는 `writing-mode: vertical-rl`로 세운다. 이 속성은 조용히
옳은 일을 다 해 준다. DOM 순서가 곧 읽기 순서가 되고, 단은 오른쪽에서
왼쪽으로 흐르고, `line-height: 2.5`는 단과 단 사이의 간격이 된다 —
행간의 間을 배치가 아니라 조판으로 얻는 것이다.

## 모션의 수학

연기의 기질은 사실상 숫자 두 개가 만든다.

- **속도 완화 2.4 s⁻¹.** 입자는 매 프레임 자기 속도와 장의 속도 사이
  간격의 `1 − e^(−dt·2.4)`만큼을 좁힌다. 반감기 약 290ms. 이 지연이
  연기(실려 가는 것)와 불꽃(복종하는 것)의 차이다. `dt`로 계산하므로
  프레임레이트와 무관하다.
- **어택 0.055 / 릴리스 0.018.** 포인터 에너지의 lerp, 60fps 기준.
  반감기는 들어올 때 약 12프레임, 나갈 때 약 38프레임. 주의는 숨
  한 번에 도착해서 연기 냄새처럼 떠난다 — 이 브랜드 전체를 상수 두
  개로 적으면 이렇게 된다.

나머지는 전부 이징 두 개에 복종한다. `ease-smoke`
(`cubic-bezier(0.22,1,0.36,1)`)는 이동량을 앞에 몰아넣고 꼬리를
서브픽셀로 가라앉히므로 900–1150ms가 느린 게 아니라 차분하게 읽힌다.
`ease-ink`(`cubic-bezier(0.65,0,0.35,1)`)는 대칭적이고 묵직해서
"물리적인" 두 동작 — 로더의 종이 들림, 버튼의 먹 채움 — 에 쓴다.
배경 주기(잉걸불 숨 5.6초, 힌트 부유 7.4초)는 서로와도, 두 캐던스와도
공약수가 없어서 쉬고 있는 페이지가 루프로 읽히는 일이 없다.

## 성능 노트

- 연기의 rAF 루프는 하나, 약 64fps로 스로틀(`FRAME_MS = 15.5`),
  `dt`는 48ms로 클램프 — 백그라운드 탭에서 돌아와도 휘청하지 않는다.
  루프는 IntersectionObserver(80px 마진)*와* `visibilitychange` 양쪽으로
  멈춘다. 화면 밖에서도, 숨긴 탭에서도 0프레임.
- DPR 상한 2. 입자 수는 무대 면적에 비례(260–820). 입자 하나의 비용은
  컬 계산용 밸류 노이즈 평가 8회(ψ 샘플 4 × 옥타브 2)와, 미리 구워 둔
  그라디언트 스프라이트의 `drawImage` 최대 3회 — 입자마다
  `createRadialGradient`를 만들지 않고, 그림자도 없다.
- 워밍업(85ms × 120스텝)은 마운트 때 동기로 한 번: 시뮬레이션 시간
  약 10초를 프레임 하나 값에 산다.
- 판들은 SVG 어트리뷰트 딱 하나를 움직이고, 난류는 판마다 한 번만
  계산된다. 화지 그레인은 정적 `feTurbulence` 사각형 하나 — 이
  페이지에서 필터는 어디서도 애니메이션되지 않는다.
- CSS 애니메이션은 전부 `transform` / `opacity` / `filter`. 리빌은
  옵저버 콜백 하나에서 클래스와 `transitionDelay`를 직접 쓴다 —
  스크롤 중 리액트 리렌더는 0회. 마운트 이후 페이지 전체의 리렌더는
  정확히 두 번: 로더가 들리기 시작할 때, 로더가 사라질 때.

## 접근성 노트

- **모션 최소화:** 로더는 아예 마운트되지 않고(안전벨트로 `display:
  none`까지), 연기는 워밍업 후 잘 구성된 정지 화면 한 장을 그리고
  멈추며(기둥은 공중에, 잉걸불은 켜진 채), 판들은 `scale=0`에서 결코
  흐트러지지 않고, 미디어 쿼리가 남은 전환과 배경 주기를 전부 걷어
  낸다. 결과는 일시정지된 비디오가 아니라 완성된 그림이다.
- **JS 없이:** 리빌 이전 스타일은 마운트 때 붙는 `.kemuri-js` 아래에만
  존재하고, 변위의 혼돈은 자바스크립트만이 *끌어올릴* 수 있다. SSR
  상태가 곧 완성된 페이지다.
- **시맨틱:** 캔버스 무대는 영·일 병기 설명이 달린 `role="img"`,
  세 폭의 판 SVG도 각각 `role="img"`에 제 레이블을 가진다. 스플릿
  텍스트는 시각적으로 숨긴 원문을 유지하고, 워드마크는 `aria-label`을
  단 `h1`이다. 낙관과 향로는 장식이므로 `aria-hidden`.
- **키보드와 터치:** 상호작용 요소 둘(버튼, 맨 위로 링크)은
  `:focus-visible`에서 2px 잉걸불 링을 받는다 — 화지 위에서는
  `ember-deep`, 어두운 방에서는 `ember`, 둘 다 비텍스트 AA 통과.
  연기는 터치에서 스스로 흐르고, 호버 뒤에 숨은 의미는 어디에도 없다.
- **대비:** 화지 위에서 먹 14.4:1, ink-soft 7.1:1, ember-deep 4.7:1.
  먹 위에서 화지 14.4:1, washi-dim 7.9:1, 잉걸불 4.9:1. 연기 회색과
  금박은 의미를 나르지 않는다. 지나가는 연기 아래 최악의 경우에도
  워드마크는 약 8:1로 읽힌다.

## 리믹스 아이디어 셋

1. **찻집.** 컬 필드는 그대로 두고 리본을 *김*으로 — 수명은 짧게,
   소용돌이는 촘촘하게(결 크기 둘 다 절반), 차림표 가장자리의 찻잔에서
   피어오르게 한다. 팔레트는 잉걸불을 짙은 말차로 바꾸고, 시는 다도의
   순서가 된다.
2. **벼루.** 먹 바탕으로 뒤집고 연기를 *아래로* 흘린다 — 물에 퍼지는
   먹(부력을 음수로, 완화는 세 배로). 판들은 가라앉는 대신 조립된다:
   판이 들어설 때 변위 스케일이 0 → 110 → 0으로 지나간다. 물방울이
   벼루를 때리고, 맑아지는 것.
3. **자정의 범종.** 기둥 대신 타종을: 종을 칠 때마다 컬 필드의 게인이
   한 번 맥동하고(SWIRL × 감쇠 사인), 글리프 캐던스가 타종 간격에
   맞춰지고, 차림표에는 제야의 백팔번뇌 108번의 타종이 오른다 —
   물론, 한 번에 한 타씩만 판다.
