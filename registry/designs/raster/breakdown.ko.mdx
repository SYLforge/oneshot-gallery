# 라스터 — 제작기

## 훅: 그리드를 인증하는 관청

어딘가에는 기준 킬로그램 원기를 금고에 보관하는 연구소가 있다. 라스터는
그 연구소의 타이포그래피 버전이다. 그리드 시스템을 인증하고, 도장을 찍고,
등록부에 올리고, 때로는 — 1998년의 그 유명한 판결처럼 — 폐지한다("홀수
단은 나누어지지 않는다"). 페이지는 이 농담을 한 번도 설명하지 않는다.
인증국에게 이것은 농담이 아니기 때문이다. 방문자는 표준 명세 제10호를,
인증 그리드 제4호 위에 조판된 상태로 건네받고, 검수를 권유받는다.

이 디자인의 도박은 절제를 구경거리로 만드는 것이다. 색 셋, 서체 가족
하나, 그리고 모든 표현력을 가변 폰트의 축 하나 — Archivo의 폭 축, 62에서
125 — 에 몰아넣었다. 페이지가 부릴 수 있는 재주가 하나뿐이라면, 그
재주가 하중을 견뎌야 한다.

## 아트 디렉션

스위스 타이포그래피는 대개 분위기로 소환된다. 여기서는 법으로 집행된다.
종이 `#f7f7f5`, 잉크 `#111111`, 그리고 빨강 하나 `#e30613` — 철도 시계와
연방 표지판의 그 스위스 빨강. 이 빨강은 정직함에 대한 교훈이 되었다.
브리프는 종이 위에서 약 4.0:1로 AA에 미달한다고 단언했지만, 제대로
계산하면 **4.55:1 — 0.05 차이의 합격**이다. 여유 없는 합격은 인증이
아니다. 그래서 정책은 미달인 것처럼 유지된다. 빨강은 마크, 헤어라인,
도장, 대형 디스플레이 면만 칠하고, 모든 빨간 상태 마크 옆에는 검정
텍스트가 반드시 병기되며, 작은 빨간 글자는 이 명세서에 존재하지 않는다.

타이포 위계는 크기나 색이 아니라 거의 전부 *폭*으로 세운다. 본문은
`'wdth' 100`, 제목은 116, 워드마크는 125에서 쉬고, 호버하면 글리프
하나가 62까지 좁아진다. 굵기는 거의 움직이지 않는다. 이것이 이 페이지의
성격 전부다. 다른 사이트라면 두 번째 색이나 더 큰 글자를 집어 들 자리에서,
라스터는 넓어진다.

구조는 일부러 노출한다. 고정 오버레이가 실제 컬럼 그리드를 — 콘텐츠와
같은 1440px 프레임, 같은 거터, 같은 트랙을 — 16% 빨강 헤어라인과 2.5%
틴트로 그리고, 히어로에는 24px 베이스라인 그리드가 더해진다. 페이지가
자기 청사진을 입고 있으니 판돈이 커진다. 그리드를 벗어난 요소는 공개적으로,
눈에 보이게 틀리게 된다.

## 기법 1 — 6↔12 FLIP 재인증

시그니처 인터랙션. 세그먼트 컨트롤이 명세서를 6단 또는 12단으로
재인증하면, `data-flip`이 붙은 모든 요소가 새로 승인된 칸으로 미끄러져
간다. 방식은 고전적인 FLIP — First, Last, Invert, Play — 이고, React는
`flushSync`로 두 측정 사이에 레이아웃 변경을 동기적으로 끼워 넣는다:

```tsx
// GridProvider.tsx — 상태 변경이 곧 FLIP의 L
flip(() => {
  flushSync(() => setColsState(next));
});
```

```ts
// useFlip.ts — 측정, 변경, 반전, 해방
const first = targets.map((el) => el.getBoundingClientRect()); // FIRST
mutate();                                                      // 이 순간 LAST가 존재한다
const l = el.getBoundingClientRect();
el.style.transform =
  `translate(${f.left - l.left}px, ${f.top - l.top}px) ` +
  `scale(${f.width / l.width}, ${f.height / l.height})`;       // INVERT
// 두 프레임 뒤: 인라인 transform을 지우면 styles.css의
// `transition: transform 560ms`가 제자리까지 데려간다. PLAY.
```

데모 수준이 아니라 실전 수준이 되게 하는 디테일 셋:

- **중단 안전성.** `getBoundingClientRect`는 진행 중인 transform까지
  포함하므로 First는 요소의 *시각적* 위치다. 애니메이션 중간에 다시
  토글해도 새 모프는 눈이 보고 있는 지점에서 출발한다. 보관해 둔 rAF
  핸들을 취소해서, 이전 토글의 해방 코드가 새 반전을 덮어쓰지 못하게
  했다.
- **읽기/쓰기 규율.** First 전부 읽기 → 스타일 전부 쓰기 → 변경 한 번 →
  Last 전부 읽기. 토글당 레이아웃 패스는 정확히 두 번, 둘 다 사용자가
  누른 순간이다. 스크롤 중에는 아무것도 측정하지 않는다.
- **화면 밖 생략.** First와 Last가 모두 뷰포트에서 멀리 벗어난 블록에는
  transform을 주지 않는다. 아무도 안 보는 애니메이션은 하지 않는다.

컬럼 오버레이는 일부러 FLIP하지 **않는다**. 12단과 6단, 두 벌의 완전한
그리드를 240ms 불투명도로 교차시킬 뿐이다 — 재스냅 560ms의 절반쯤.
법이 먼저 바뀌고, 콘텐츠가 눈에 보이게 따라간다. 이 순서가 이 픽션의
전부다.

## 기법 2 — 글리프 단위로 쪼갠 폭 축

워드마크는 글리프마다 `font-variation-settings: 'wdth'`를 62에서 125로,
55ms 시차를 두고 움직인다. 글자들이 좁게 도착해서 넓게 정착하는 —
페이지 전체가 딛고 선 그 축의 실연이다. 스플릿 텍스트는 접근성의
함정이라, 분할은 표준 안전 패턴을 따른다. `<h1>`이
`aria-label="RASTER"`를 들고, 글리프 스팬들은 `aria-hidden` 래퍼 안에
산다. 스크린 리더는 단어 하나를 받고, 애니메이션은 요소 여섯 개를 받는다:

```tsx
<h1 aria-label="RASTER">
  <span className="raster-mark" aria-hidden="true">
    <span className="raster-mark__ch" style={{ "--i": 0 }}>R</span>
    …
```

CSS에는 훔쳐 갈 만한 미묘함이 하나 있다. 등장 애니메이션은 `forwards`가
아니라 `backwards` 필을 쓰고, 100% 키프레임은 휴지 상태와 바이트 단위로
동일하다:

```css
.raster-mark__ch {
  font-variation-settings: "wdth" 125;                 /* 휴지 상태 = 100% 키프레임 */
  transition: font-variation-settings 300ms var(--raster-ease);
}
.raster-js .is-on .raster-mark__ch {
  animation: raster-wdth-in 1100ms var(--raster-ease) backwards;
  animation-delay: calc(var(--i) * 55ms);
}
```

`forwards` 필 애니메이션은 끝난 뒤에도 속성을 계속 덮어써서, 나중의 호버
트랜지션을 소리 없이 죽인다. `backwards`라면 애니메이션이 끝나는 순간
요소는 기본 스타일로 떨어지고 — 값이 같으니 튀지 않는다 — 커서 아래
글리프가 62로 좁아지는 300ms 호버 트랜지션이 그냥 작동한다. 푸터의 판정문
STILL CORRECT는 같은 컴포넌트를 IntersectionObserver로 점화해서 재사용한다.
단어는 `white-space: nowrap` 스팬으로 감싸고 사이에 진짜 공백을 두어,
360px 명세서에서 단어 사이에서는 줄이 바뀌되 단어 안에서는 절대 바뀌지
않는다.

## 기법 3 — 스스로 그려지는 도장

인증 도장(원 두 개, 눈금 네 개)과 푸터의 인증선은 SVG 대시 트릭으로
그려지는데, `pathLength`로 정규화했다:

```tsx
<circle cx="66" cy="66" r="62" pathLength={1} />
```

```css
.raster-js .is-on .raster-stamp__draw {
  stroke-dasharray: 1;      /* 대시 하나, 간격 하나 — 각각 경로 전체 */
  stroke-dashoffset: 1;     /* 완전히 숨긴 채 시작 */
  animation: raster-draw 900ms var(--raster-ease) forwards;
}
@keyframes raster-draw { to { stroke-dashoffset: 0; } }
```

`pathLength={1}`이면 자바스크립트가 `getTotalLength()`를 잴 일이 없다.
반지름이 얼마든, 푸터의 100유닛 직선이든, 같은 CSS 세 줄로 움직인다.
원들은 0/200/420ms로 시차를 두고, 모노 레터링은 형태가 생긴 뒤인
720ms에 떠오른다 — 도장은 먼저 찍히고, 그다음에 읽힌다.

모션 최소화 처리에는 기록해 둘 함정이 있다. 애니메이션만 죽이면 선들이
`dashoffset: 1`에서 얼어붙는다 — 영원히 보이지 않는다. 그래서 미디어
쿼리가 대시 속성까지 리셋한다:

```css
@media (prefers-reduced-motion: reduce) {
  .raster-root .raster-stamp__draw,
  .raster-root .raster-cert__line {
    stroke-dasharray: none !important;
    stroke-dashoffset: 0 !important;
  }
}
```

## 모듈러 스케일, 스크롤로 문지르기

섹션 02는 타입 시스템을 실연으로 가르친다. 비율 1.333(완전4도: 기준의
1 / 1.333 / 1.777 / 2.369 / 3.157 / 4.209배)의 여섯 단계를 Space Mono로
주석 달고, 스크롤이 각 단어를 제 크기의 한 단계 아래에서 인증된 값까지
끌어올린다.

수학이 핵심이다. `font-size`를 애니메이션하면 매 프레임 문서 전체가
리레이아웃된다. 그래서 각 단어의 `font-size`는 처음부터 정착값이고,
스크롤은 `transform: scale()`을 `1 / 1.333 ≈ 0.750`에서 `1`까지 몬다 —
**정확히 비율 한 단계만큼의 여정**. 제목은 스케일 자체를 걸어서 제
크기에 도착하고, 일은 전부 컴포지터가 한다:

```ts
const p = clamp01((vh * 0.92 - rect.top) / (rect.height + vh * 0.4));
const local = clamp01((p - i * 0.09) / 0.5);   // 단계당 0.09 시차
const eased = local * local * (3 - 2 * local); // smoothstep
el.style.transform = `scale(${(S0 + (1 - S0) * eased).toFixed(4)})`;
```

smoothstep(`3t² − 2t³`)은 양 끝에서 속도가 0이라, 각 단어는 정지에서
부드럽게 출발해 꺾임 없이 정착한다. 스크롤 연동 애니메이션은 시간 기반
트랜지션이 대신 이징해 주지 않으므로, 이징을 매핑 *안에* 넣어야 한다.
rAF당 `getBoundingClientRect` 한 번, 읽기 다음 쓰기, 섹션이 뷰포트에서
1.6화면 이상 멀면 조기 탈출. transform은 이 이펙트가 쓰는 인라인
스타일로만 존재하므로, JS 없음과 모션 최소화는 구조적으로 정착된
스케일을 렌더링한다.

## 성능 노트

- **상주 루프 없음.** 크로스헤어와 스케일 스크럽은 입력 이벤트
  (`pointermove`, `scroll`)당 rAF 하나를 예약하고 실행되면 핸들을 비운다.
  가만히 있는 페이지의 프레임 비용은 0이다.
- FLIP은 토글당 레이아웃 패스 두 번 — 둘 다 사용자가 누른 순간이고,
  스크롤 중에는 절대 아니다. 화면 밖 블록은 통째로 건너뛴다.
- 움직이는 것은 `transform`, `opacity`, `font-variation-settings`,
  `stroke-dashoffset`뿐이다. 레이아웃도, 페인트 폭풍도 없다.
  `font-variation-settings`는 글리프를 다시 래스터화하지만, 워드마크는
  로드 시 단 한 번 움직이는 한 줄이다.
- 오버레이는 정적 DOM(스팬 18개)이고 움직이는 속성은 불투명도 하나다.
  베이스라인 그리드는 `repeating-linear-gradient` 한 장이다.
- 리스너, 옵저버, rAF 핸들은 언마운트 때 전부 정리된다. 상태 표시줄은
  DOM 조작이 아니라 React 상태라 하이드레이션이 어긋나지 않는다.

## 접근성 노트

- **모션 최소화:** 루트 전체에 `animation/transition: none` 블랭킷(모든
  기본 상태가 완성된 문서이므로 안전하다), 선 그리기의 대시 속성 리셋,
  그리고 JS 쪽 조기 반환. FLIP은 즉시 재배치가 되고, 스케일은 정착 상태로
  렌더링되며, 크로스헤어는 "x 0.500 y 0.500" — 인증된 중심에 선다.
- **JS 없음:** `.raster-js`가 모든 등장 애니메이션과 FLIP 트랜지션을
  게이트한다. 정적 페이지는 완성된 12단 명세서다.
- **키보드와 터치:** 06/12 컨트롤은 라벨 있는 그룹 속 네이티브 버튼 두
  개(`aria-pressed`)다. `role="status"` 줄이 재인증을 음성으로도
  공표한다. 크로스헤어는 `(hover: hover) and (pointer: fine)`에서만
  무장한다.
- **시맨틱:** 워드마크와 판정문은 aria-label + aria-hidden 글리프 패턴,
  도장은 장식(`aria-hidden`) — 인증 주장은 옆의 명세 텍스트가 들고 있다.
  등록부는 모든 헤더에 `scope`를 갖춘 진짜 `<table>`이다.
- **대비:** 잉크 17.6:1, 주석 5.6:1, 빨강 블록의 흰 라벨 4.9:1. 빨간
  글자는 디스플레이 크기에만 존재하고, 모든 빨간 마크 옆에는 검정
  텍스트가 병기된다.

## 리믹스 아이디어

1. **도량형 인증국.** 전부 그대로 두고 그리드를 질량으로 바꾼다 — FLIP은
   기준 분동 표를 미터법과 야드파운드법 사이에서 재스냅하고, 도장은
   "1.000 kg. 여전히 1.000 kg."을 인증한다.
2. **폐지 사무소.** 정서를 뒤집는다. 등록부가 자라고, 빨간 ✕가 늘고,
   푸터 판정문은 NO LONGER CORRECT로 움직인다 — 같은 무표정으로 조판된,
   표준이 죽어 가는 페이지.
3. **폭 축 시계.** `'wdth'`를 로드가 아니라 시각으로 몬다 — 워드마크가
   24시간에 걸쳐 62→125로 숨 쉰다. 하루에 한 번, 알아챌 수 없는 인증.
   크로스헤어는 타임스탬프를 읽는다.
