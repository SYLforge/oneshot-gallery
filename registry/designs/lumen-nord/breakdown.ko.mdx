# 루멘 노르드 — 제작기

## 훅: 혼돈을 예보하는 기관

기상청은 매일 밤 혼돈에 대해 숫자로 약속을 하고, 아무도 그것을 이상하게
여기지 않는다. 그렇다면 오로라도 같은 표정으로 예보하는 기관이 있어도
되지 않을까. 노르웨이 능선 위의 관측소 하나, 서울의 사무소 하나. 발행
번호가 있고, 신뢰도가 있고, 관측 창이 있고 — 그리고 하룻밤에 딱 한 줄,
정체를 들키는 문장이 있다. "오늘 밤, 하늘은 자신이 커튼이었음을 기억할
확률 78%."

이 디자인의 도박은 하나다. 숫자를 끝까지 진지하게 다루면, 그 사이에 끼운
시가 더 세게 박힌다는 것. 그리고 이미지 파일 한 장 없이, 셰이더 하나가
페이지의 시각 전부를 짊어질 수 있다는 것.

## 아트 디렉션

글래스 퓨처리즘이 망하는 경로는 뻔하다. 카드에 블러를 얹고, 그라디언트를
깔고, SaaS 랜딩 페이지가 되어 버리는 것. 그 길을 막는 결정이 둘 있었다.
첫째, 디스플레이 서체가 세리프다 — 한글과 로마자를 처음부터 함께 그린
함렛(Hahmlet)이 소라(Sora)의 계기판 목소리 위에 선다. 오로라 빛 위의
세리프 제호는 "새 장비를 들인 오래된 기관"으로 읽히고, 그게 정확히 이
픽션이다. 둘째, 본문 폰트 스택이 `Sora, Hahmlet, sans-serif`라서 한글
본문은 *일부러* 세리프로 떨어진다. 한국어는 영어의 번역 레이어가 아니라
자기 목소리를 가진 서울 사무소의 손이고, `word-break: keep-all`이 어절을
지킨다.

팔레트는 브리프의 다섯 토큰에, 대비 계산이 만들어 낸 파생 하나를 더했다.
수평선 파랑 `#27406b`은 1.9:1 — 아름답지만 글자로는 실격이라 괘선과 능선
발광으로 강등시키고, 보조 텍스트는 얼음색 72% 알파(약 `#a4adc5`, 8.1:1)가
맡는다. 마젠타 베일은 5.8:1 — 레이블과 옷단까지는 되고, 긴 본문은 규칙으로
금지.

## 기법 1 — 오로라 (webgl-shader)

하늘은 오버사이즈 삼각형 하나 위의 프래그먼트 셰이더 하나다. 라이브러리
없음, 쿼드의 대각선 이음매 없음, 드로우 콜 한 번. 필드는 고전적인 도메인
워핑이다. 3옥타브 값 노이즈 fbm이 두 번째 fbm의 정의역을 흔들고, 그
결과를 능선 함수로 접어 커튼을 만든다:

```glsl
float flow = fbm(vec2(q.x * 1.1 + t * 0.6, q.y * 1.8 - t * 0.25));
float rib  = fbm(vec2(q.x * 2.4 + flow * 2.2 - t * 0.4,
                      q.y * 0.9 + flow * 0.8 + t * 0.15));

float rid = 1.0 - abs(rib * 2.0 - 1.0);  // 노이즈를 접어 능선으로
rid = pow(rid, 2.6);                     // 능선을 조여 커튼으로
```

`1 − |2n − 1|`은 매끄러운 노이즈를 주름으로 접고, 거듭제곱이 그 주름을
드리운 띠로 조인다. 가는 세로 노이즈(`q.x * 24.0`)가 띠를 빗질해 광선을
만들고, 색은 높이와 흐름에 걸린 혼합으로 정해진다 — 아래는 초록 치맛단,
위는 자주색 옷단. 실제 커튼이 층을 이루는 방식 그대로.

포인터는 조명을 끌고 다니는 게 아니라 장(場) 자체를 휘게 한다:

```glsl
vec2 d = q - ptr;
float fall = exp(-dot(d, d) * 4.5);        // 가우시안 — 설계로 상한이 잡힌다
q += normalize(d + 0.0001) * fall * 0.22 * u_energy;
```

CPU가 포인터 위치(프레임당 0.07)와 에너지(attack 0.05 / release 0.02)를
lerp로 늦춰 주기 때문에, 이 뒤틀림은 커서가 아니라 날씨처럼 도착한다.
포인터가 3.2초 침묵하면 — 혹은 터치 기기라면 — 느린 리사주 드리프트가
핸들을 넘겨받고, 하늘은 혼자서 훑기 시작한다.

훔쳐 갈 만한 디테일 둘. 최종 색에 더하는 프레임당 해시 디더(채널당
±0.003)는 어두운 그라디언트가 일반 패널에서 드러내는 밴딩을 지운다.
별은 픽셀 단위 난수가 아니라 셀 단위 해시라서, 각자 제자리에 붙박인 채
저마다의 속도로 깜빡인다.

**폴백은 코드 경로가 아니라 레이어 순서다.** 항상 칠해져 있는 CSS
오로라 — 블러 처리된 방사형 얼룩 셋과 원뿔형 스윕, 26초/19초 교대
드리프트 — 가 캔버스 밑에 깔려 있고, 캔버스는 첫 프레임이 실제로 그려진
뒤에야(`draw()` 다음의 `is-live` 클래스) 그 위로 페이드인한다. WebGL이
없어도, 컴파일이 실패해도, `webglcontextlost`가 떨어져도 결론은 같다.
CSS 하늘은 이미 거기 있었다. 렌더 시점에 감지할 것도 없고, 빈 히어로는
구조적으로 불가능하다.

## 기법 2 — 당신을 알아채는 유리 (glass-specular)

패널은 `backdrop-filter: blur(14px) saturate(150%)`다. 그런데 민무늬 색
위의 블러는 그냥 회색 사각형이다. 그래서 게시판 섹션은 패널 뒤에 여러
크기의 반복 방사형 그라디언트로 별 반점을 깔아 둔다 — 블러가 부술 디테일을
주기 위해서만 존재하는 배경이다. 그 부서진 디테일이 곧 "유리"로 읽힌다.

스펙큘러 하이라이트는 커스텀 프로퍼티 두 개와 리렌더 0회다:

```tsx
// GlassPanel.tsx — 리액트 상태 없이 요소에 직접 쓴다
const r = el.getBoundingClientRect();
el.style.setProperty("--lumen-gx", `${(((ev.clientX - r.left) / r.width) * 100).toFixed(2)}%`);
el.style.setProperty("--lumen-gy", `${(((ev.clientY - r.top) / r.height) * 100).toFixed(2)}%`);
el.style.setProperty("--lumen-gs", "1");
```

```css
.lumen-glass__spec {
  background: radial-gradient(
    230px 175px at var(--lumen-gx) var(--lumen-gy),
    rgba(223, 233, 255, 0.16), rgba(223, 233, 255, 0.05) 46%, transparent 72%
  );
  opacity: var(--lumen-gs);
}
```

`pointerleave`에서는 프로퍼티를 *지운다*. 그러면 CSS 기본값이 돌아온다 —
상단 중앙, 55% 세기로 쉬고 있는 하이라이트. 이 대기 상태가 곧 터치
스토리다. 폰에서 패널은 죽은 유리가 아니라 불 켜진 유리이고, 페이지의
의미는 어디에서도 호버에 기대지 않는다. 세기가 부드럽게 오르내리는 건
`--lumen-gs`를 `@property`로 등록했기 때문인데, 미지원 브라우저에서는
그냥 탁 바뀐다. 그걸 버그로 신고할 사람은 없다.

대비는 기도가 아니라 예산이다. 패널 몸통은 극야 톤 58% 알파 유리라서,
잔광의 가장 밝은 조각이 패널 바로 뒤로 흘러들어도 얼음색 텍스트는
5.7:1 아래로 내려가지 않는다.

## 기법 3 — 스스로 그려지는 예보 (svg-line-draw)

밤의 예측 강도는 일곱 개의 Kp 값을 지나는 캣멀롬 스플라인이고, 모듈
스코프에서 큐빅 베지어로 구워 둔다(이건 상태가 아니라 데이터니까).
드로잉 트릭은 `pathLength` 정규화다:

```tsx
<path className="lumen-spark__line" d={LINE} pathLength={1} />
```

```css
.lumen-js .lumen-spark__line { stroke-dasharray: 1 1; stroke-dashoffset: 1; }
.lumen-js .lumen-spark.is-visible .lumen-spark__line {
  stroke-dashoffset: 0;
  transition: stroke-dashoffset 1900ms cubic-bezier(0.33, 0, 0.2, 1) 200ms;
}
```

`pathLength`를 1로 고정하면 대시 계산에 `getTotalLength()`가 필요 없다 —
레이아웃 읽기도, 리사이즈 리스너도 없고, CSS는 곡선의 실제 길이를 영영
몰라도 된다. `is-visible` 클래스는 페이지의 모든 스크롤 리빌을 굴리는
바로 그 IntersectionObserver에서 온다. 스파크라인은 같은 신호로 더 많은
일을 할 뿐이다. 점들은 선이 지나간 뒤 인라인 딜레이로 하나씩 켜지고,
면 채움이 마지막에 떠오른다. Kp 5 폭풍 문턱은 레이블 달린 점선으로
박아 두었다 — 문턱 없는 예보는 그냥 낙서니까.

숨김 전 상태(대시로 가린 선, 투명한 점)는 전부 `.lumen-js` 뒤에 있다.
자바스크립트가 없으면, 차트는 처음부터 다 그려져 있다.

## 스크롤 온도

이 페이지에서 시간에 따라 변해도 되는 색은 딱 하나, `--lumen-blend`다.
스크롤 진행도에 따라 JS가 마젠타 베일에서 오로라 초록으로 lerp한다
(rAF로 묶은 패시브 리스너). 섹션의 계기 레이블, 유리 패널의 상단
헤어라인, 관측 방법 목록의 왼쪽 괘선이 모두 이 색을 빌려 쓴다 — 그래서
페이지 상단은 황혼의 마젠타로, 하단은 깊은 밤의 초록으로 단이 둘러진다.
배경은 같은 변수로 `#080b20`을 향해 `color-mix`되고, 셰이더는 같은 스크롤
진행도를 3배로 증폭한 `u_temp`를 받아 히어로가 아직 화면에 있는 동안
색 가중치를 옮기고 바탕을 어둡힌다. 요소 하나하나로는 미묘하고, 전체로는
분명하다.

## 모션의 수학

- **포인터 lerp 0.07/프레임** (60fps 정규화): 반감기 약 9.6프레임,
  약 160ms. 뒤틀림이 손보다 반 박자 늦는다 — UI가 아니라 대기(大氣)의
  지연이다.
- **에너지 attack 0.05 / release 0.02:** 하늘은 반감기 13프레임 만에
  당신을 알아채고, 잊는 데는 34프레임이 걸린다. 관심은 빨리 오고 천천히
  식는다. 이 비대칭이 예보국의 성격 전부다.
- **뒤틀림 상한:** `exp(−4.5·d²) × 0.22` — 최대 에너지에서 변위는 화면
  높이의 5분의 1 언저리에서 꼭짓점을 찍고 그 반경 안에서 죽는다. 하늘은
  휘어지되, 찢어지지 않는다.
- 리빌 이징 `cubic-bezier(0.22, 1, 0.36, 1)`은 이동량을 앞쪽에 몰아넣어
  700ms를 느림이 아니라 차분함으로 읽히게 한다. 스태거는 옵저버 배치
  단위 80ms — 같이 도착한 요소끼리만 줄을 선다.

## 성능 노트

- 페이지 전체에 rAF는 하나, 셰이더가 소유한다. 히어로가 뷰포트를 떠나면
  (IntersectionObserver, 80px 마진), 탭이 숨으면(visibilitychange),
  모션 최소화면(아예 시작 안 함), 방문자가 "하늘 멈춤"을 누르면 멈춘다.
- DPR 상한 2 — 그라디언트 필드는 그 위로는 얻는 게 없다. `dt`는 50ms로
  클램프해서 백그라운드 탭에서 돌아와도 시뮬레이션이 휘청하지 않는다.
- 스크롤 온도 리스너는 패시브 + rAF 병합: 프레임당 최대 한 번의 스타일
  쓰기, 전부 커스텀 프로퍼티, 레이아웃 읽기 없음.
- 스펙큘러 추적은 pointermove당 요소 하나에 커스텀 프로퍼티 두 개를 쓰는
  게 전부다. 리액트 상태도, 리렌더도 없다.
- CSS가 움직이는 건 transform과 opacity뿐이고, 불투명한 캔버스가 CSS
  오로라를 덮는 순간 그 드리프트는 `animation-play-state`로 정지된다 —
  대역은 무대에 주인공이 서면 숨 쉬는 것도 멈춘다.

## 접근성 노트

- **모션 최소화:** 셰이더는 잘 구성된 프레임 하나(t = 38초, 드리프트
  중앙, 에너지 0.55)만 그리고 루프를 시작하지 않는다. CSS 오로라는 정지,
  리빌과 스파크라인은 그냥 보이는 상태, 멈춤 버튼은 숨는다 — 멈출 게
  없으니까.
- **JS 없이:** 리빌 전 상태는 전부 마운트 때 붙는 `.lumen-js` 아래에만
  있다. 자바스크립트가 꺼져 있으면 이 페이지는 CSS 오로라 위의 완성된
  예보문이고, 처음부터 끝까지 읽힌다.
- **멈춤 컨트롤:** 히어로는 5초를 넘는 자율 애니메이션이므로 진짜
  `button`과 `aria-pressed`를 받는다(WCAG 2.2.2). 셰이더와 CSS 폴백을
  둘 다 얼린다.
- **시맨틱:** 오로라 래퍼는 `aria-hidden`(그건 콘텐츠가 아니라 날씨다).
  스파크라인은 `role="img"`에 실제 곡선을 서술하는 한·영 레이블, 게이지는
  옆의 텍스트 값에 딸린 장식이다.
- **대비:** 얼음 14.9:1, 보조 8.1:1, 초록 13.1:1, 마젠타 5.8:1(레이블
  한정). 제호는 밤색 스크림 위에 앉아 최악의 커튼 배경에서도 AA를 넘고,
  수평선 파랑은 의미를 나르지 않는다.

## 리믹스 아이디어 셋

1. **조석 예보국.** 뼈대는 그대로, 해수면 팔레트로. 셰이더의 커튼을
   90도 돌려 너울의 띠로, 스파크라인은 조석표로. 자문 패널은 항구가
   숨을 참는 시각을 알려 준다.
2. **유성우 감시소.** 하늘은 두고, 셀 해시 하나를 더 얹어 시간 게이트가
   걸린 유성 줄무늬를 긋는다. Kp 게이지는 ZHR 계수기로. 멈춤 버튼의
   문구는 "하늘 붙들기"가 된다.
3. **광공해 지도국.** 픽션을 뒤집어 어둠 자체를 예보한다. 스크롤 온도는
   도시의 주황에서 진짜 검정으로 흐르고, 유리 패널은 오늘 밤 별이 보일
   동네를 보고한다.
