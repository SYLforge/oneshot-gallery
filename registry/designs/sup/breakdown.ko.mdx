# 숲 SUP — 제작기

## 훅: 서두르지 않는 페이지

산림욕의 규칙은 하나뿐이고, 그마저 규칙이라 부르기 민망하다. 생각했던
것보다 천천히 걸을 것. 이 디자인의 도박은 랜딩 페이지에게 그 규칙을
지키게 하는 것이었다. 이 페이지의 어떤 것도 주목을 요구하지 않는다 —
나무는 당신이 보든 말든 자라고, 풀은 손에 대답하지만 손을 필요로 하지
않으며, 유일한 인터랙티브 컨트롤은 "넷까지 같이 세어 드릴까요"라고
제안하는 버튼 하나다. 사진은 한 장도 없다. 숲의 사진은 다른 어딘가의
기억이니까. 가지와 잎과 풀잎 전부가 코드로 생성된다. 그래서 이 페이지는
숲을 *보여주는* 게 아니라, 분재가 그러하듯 — 서툴게, 일부러 — 숲이
되어 본다.

## 아트 디렉션

라이트 테마지만, 스튜디오의 흰색이 아니라 숲의 빛이다. 크림
`#f2efe6`(빈터), 읽어야 하는 모든 것에는 롬 `#2e2a24`, 클릭을 청하는
유일한 색으로 모스 `#4a5d3a`. 페이지에서 가장 예쁜 색인 라이큰
`#a8b89a`는 크림 위에서 1.8:1 — 그래서 목소리가 아니라 잎사귀로만 쓴다.
잎, 먼 풀, 장식용 좌표 한 줄. 보조 텍스트는 대신 파생 토큰이 맡는다.
롬 72% 알파(`bark`, 약 `#65615a`, 5.4:1).

서체 둘은 화자가 다르다. Instrument Serif는 안내자의 목소리 — SUP
워드마크와 모든 이탤릭 방백("눈에 보이는 바람을, 우리는 풀이라고
부릅니다"). 고운돋움은 모든 것이 딛고 선 땅이다. 모든 한글과 본문,
둥글고 서두르지 않는 조판. 디스플레이 스택을 `Instrument Serif, Gowun
Dodum, serif` 순서로 두었기에 거대한 '숲' 한 글자는 마크업 없이 라틴
서체를 통과해 고운돋움에 도착한다 — 폰트 스택의 우연을 구조로 만든
셈이다.

## 기법 1 — 스스로를 그리는 나무 (svg-line-draw)

히어로의 식물은 괄호 L-시스템이다. 공리 `X`, 규칙 `X → F[+X][-X]FX`와
`F → FF`, 5회 반복. 시드 고정 PRNG를 쥔 터틀이 해석하므로 서버와
클라이언트가 바이트 단위로 같은 SVG를 그린다. 문법이 구조를 주고,
터틀은 문법이 말하지 못하는 것을 보탠다. 마디마다의 흔들림, 빛을 향해
되돌아가는 굴광성, 깊이에 따라 줄어드는 걸음.

```ts
// plant.ts — 터틀의 걸음
if (ch === "F") {
  const step = 10 * Math.pow(0.92, f.depth) * (0.85 + rnd() * 0.3);
  f.ang += (rnd() - 0.5) * 8;          // 흔들림 — 살아 있는 것은 곧지 않다
  f.ang += (-90 - f.ang) * 0.034;      // 굴광성 — 빛 쪽으로 다시 기운다
  ...
  f.dist += step;                       // 성장 시계
}
```

핵심은 `f.dist` — 씨앗에서부터의 누적 거리다. 모든 가지가 그 시계 위에서
자기가 시작하고 끝나는 지점을 0–1로 기록하고, 모든 패스는
`pathLength={1}`을 받아 실제 길이와 무관하게 대시 계산이 성립한다.
그다음, CSS 변수 하나가 나무 전체를 키운다:

```css
/* styles.css — 변수 하나, 250개의 사적인 타임라인 */
.sup-growth__branch {
  stroke-dasharray: 1 1;
  stroke-dashoffset: clamp(0, calc((var(--t1) - var(--sup-grow, 1)) / var(--tw)), 1.02);
}
.sup-growth__leaf {
  --sup-unfurl: clamp(0, calc((var(--sup-grow, 1) - var(--lt)) * 12), 1);
  opacity: var(--sup-unfurl);
  transform: scale(var(--sup-unfurl));
}
```

rAF 훅(`useGrowth`)이 `--sup-grow`를 약 5.2초에 걸쳐 0.74까지 부드럽게
올리고, 나머지 26%는 스크롤이 맡는다. 프레임당 `setProperty` 한 번이면
줄기가 가지보다 먼저, 가지가 수관보다 먼저 그려지고, 잎은 저마다 시계의
8% 구간 안에서 톡 하고 벌어진다. 접근성 서사를 지탱하는 디테일 둘:
폴백이 `var(--sup-grow, 1)`이라 JS가 없거나 모션을 줄인 방문자는 헐벗은
나무가 아니라 완성된 나무를 보고, 숨김 상태의 오프셋을 1.0이 아닌
1.02로 클램프해 둥근 선 끝이 가지 밑동에 유령 점을 남기지 못하게 했다.

## 기법 2 — 눈에 보이는 바람 (canvas-particles)

풀밭은 DPR 상한 2의 캔버스 위 450여 개의 인스턴스 풀잎이고, 풀잎
하나하나가 자기만의 스프링 상태를 가진 이차 곡선 하나다. 포인터는
풀잎을 직접 건드리지 않는다. 대신 *공기*에 기록한다 — 옆으로 번지고
서서히 잊히는 96칸짜리 1차원 바람 필드에, 들고 있던 음이 잦아들듯.

```ts
// WindField.tsx — 손이 지나간 자리가 날씨가 된다
const v = clamp((ev.clientX - lastPX) / max(now - lastPT, 8), -2.4, 2.4);
deposit(nx, v * 1.4);                  // 바람 칸에 가우시안으로 새긴다
// 매 프레임:
windTmp[i] = wind[i] + (l + r - 2 * wind[i]) * 0.22 * k;  // 번짐
wind[i] = windTmp[i] * Math.pow(0.945, k);                 // 잊힘
```

임펄스가 포인터의 *속도*를 담기 때문에 느린 손짓은 한숨이 되고 빠른
손짓은 돌풍이 된다. 그리고 칸에서 칸으로 번지기 때문에, 손이 멈춘
뒤에도 미풍은 눈에 보이게 계속 나아간다. 각 풀잎은 `ambient +
windAt(x)`를 향해 작은 스프링을 돌린다:

```ts
bl.vel += (target - bl.bend) * bl.stiff * k;   // 강성 0.03–0.06
bl.vel *= Math.pow(bl.damp, k);                // 감쇠 0.88–0.93
bl.bend += bl.vel * k;
```

이것이 커튼이 아니라 풀로 읽히는 이유는 스프링 상수의 산포다. 풀잎마다
제 속도로, 부드러운 오버슛 한 번씩을 거쳐 일어선다. 드로잉은 배치로
처리했다 — 깊이 세 줄 × 톤 두 개, 그룹마다 하나의 패스에 이어 붙여
프레임당 `strokeStyle` 변경이 여섯 번이면 끝난다. 터치이거나 포인터가
아예 없어도 앰비언트 항(서로 무관한 사인 셋 + 약 33초마다 지나가는
돌풍)이 들판에 날씨를 남겨 두고, 탭하면 손끝에서 풀이 바깥으로 갈라진다.

## 기법 3 — 가라앉는 깊이 (pointer-parallax)

히어로에는 깊이 평면이 셋 있다. 하층 관목(같은 L-시스템의 3회 반복,
다른 시드), 본 나무, 그리고 카피. 훅이 변수 두 개를 프레임당 0.05로 —
일부러 통상의 절반 속도로 — 포인터 쪽으로 러프하고, CSS가 이를 상한이
있는 반대 부호 이동으로 곱한다. 최대 기울임에서 관목 −7px, 나무 +12px,
카피 −4px.

```css
.sup-hero__plant--main {
  transform: translate3d(calc(var(--sup-px, 0) * 12px),
                         calc(var(--sup-py, 0) * 7px), 0);
}
```

이 러프 속도에서 포인터 움직임의 반감기는 약 13프레임 — 레이어는 손을
*추적*하는 게 아니라, 새가 날아간 뒤의 가지처럼 손이 지나간 자리에
*가라앉는다*. 파인 포인터 전용이고, 터치에서는 평면이 쉬는 대신 스웨이
애니메이션이 깊이감을 맡는다. 폴백이 둘 다 0이라 JS 없는 페이지는
그저 단정하게 놓여 있다.

## 숨 — 가르치지 않고 곁에 있는 모션

4·4·4 가이드는 가장 정직해야 했던 섹션이다. 설계 원칙은 하나:
**애니메이션이 죽어도 지시문은 살아남을 것.** "넷을 세며 들이쉬고, 넷을
머금고, 넷을 세며 내쉬세요"라는 문장은 오브 위에 항상 보이는 평문이다.
오브와 헤일로와 카운트다운은 전부 `aria-hidden` — 문장 위에 얹힌
장식이지, 그 반대가 아니다.

하나의 rAF 시계가 오브 스케일(`--sup-breath`, 하프사인 이징)과 단계
단어 + 숫자를 함께 몰기 때문에 글과 도형은 어긋날 수가 없다. 상태
업데이트는 표시되는 초가 실제로 바뀔 때만 — 12초 주기에 렌더 열두 번.
모션 최소화에서도 같은 버튼이 그대로 일한다. 변수 쓰기만 건너뛴
텍스트 전용 카운트가 시작된다(단어와 숫자는 여전히 돈다). 호흡 운동은
모션을 줄인 사용자야말로 찾아왔을 법한 기능이니까. 일시정지는 주기
안의 위치를 저장하고, 재개는 시계를 다시 그 자리에 고정한다 — 숨이
"들이쉬세요"로 튕겨 돌아가지 않고 멈춘 데서 이어진다.

## 모션의 산수

- **성장 러프 0.06/프레임** (60fps 정규화): 반감기 약 11프레임 ≈ 190ms.
  스크롤이 스크러버가 아니라 날씨처럼 느껴지는 느림이다.
- **바람 감쇠 0.945/프레임**: 돌풍의 반감기는 약 12프레임이지만, 죽는
  동안에도 확산이 옆으로 옮겨 심는다. 세게 한 번 스치면 미풍이 들판
  너비 하나쯤을 건너가 보이는 이유다.
- **앰비언트 주기 9.4초 / 12.7초 / 17초 / 12초** (나무 스웨이, 관목
  스웨이, 햇빛, 호흡)는 공약수가 없다 — 페이지는 눈에 띄게 반복되지
  않는데, 실무에서 "부산하지 않게 살아 있다"의 대부분은 이것이다.
- 리빌 이징 `cubic-bezier(0.22, 0.8, 0.24, 1)`은 950ms 중 앞 3분의
  1에서 이동량의 70%를 소화한다. 20px 이동이면 꼬리는 서브픽셀로
  잦아들어, 느림이 굼뜸이 아니라 차분함으로 읽힌다.

## 성능 노트

- 프레임당 `setProperty` 한 번으로 SVG 요소 약 250개가 자란다. 스타일
  재계산은 브라우저가 일괄 처리하고, stroke-dashoffset / opacity /
  transform은 전부 페인트·컴포지트 레벨 — 성장 중에 레이아웃은 한 번도
  돌지 않는다.
- 캔버스 루프는 약 64fps로 스로틀(`FRAME_MS = 15.5`), `dt`는 48ms로
  클램프, IntersectionObserver(여유 120px)와 `visibilitychange`가
  켜고 끈다 — 화면 밖에서 0프레임, 숨은 탭에서 0프레임.
- 풀잎 드로잉은 프레임당 배치 스트로크 여섯 번, 꽃가루는 fillStyle 한
  번. DPR 2 × 1440px에서도 내장 그래픽으로 60fps 예산 안에 넉넉히 든다.
- 세 개의 rAF 훅(성장, 드리프트, 호흡) 모두 화면 밖에서 스스로 멈추고,
  언마운트 시 옵저버와 리스너를 정리한다.
- 식물은 모듈 로드 시 한 번(~4ms) 생성되어 서버와 클라이언트가 공유한다
  — 하이드레이션 불일치가 구조적으로 불가능하다.

## 접근성 노트

- **모션 최소화:** `usePrefersReducedMotion`(`matchMedia` 위의
  `useSyncExternalStore`)이 네 개의 모션 훅을 전부 끄고, CSS 폴백이
  모든 것을 완성된 정지 화면에 내려놓는다 — 다 자란 나무들, 바람이
  머무는 순간의 들판 한 컷, 중간 숨(0.62 스케일)에서 쉬는 오브. 호흡
  운동은 텍스트만으로 온전히 쓸 수 있다.
- **No-JS:** 리빌 이전의 숨김 상태는 `.sup-js` 아래에만 존재하고, 모든
  애니메이션 값에는 완성 상태의 폴백이 있다. 소스 보기를 하면 완성된
  숲이 보인다.
- **시맨틱:** 캔버스는 이중 언어 설명이 달린 `role="img"`, 식물 SVG는
  `aria-hidden` 장식(의미는 타이틀이 나른다), 일시정지는 `aria-pressed`
  를 가진 네이티브 `<button>`, 움직이는 리드아웃은 `aria-hidden`이라
  스크린리더는 메트로놈 대신 차분한 지시문 하나를 받는다.
- **대비:** 크림 위에서 롬 12:1, 모스 6.2:1, 바크 5.4:1. 호흡 단어는
  오브의 가장 밝은 부분 위에서도 9:1 이상, 라이큰(1.8:1)은 아무것도
  이름 붙이지 않는다.
- **키보드:** CTA → 일시정지 버튼 → 푸터 링크 순서 그대로, 각각 모스
  `:focus-visible` 링과 부드러운 헤일로가 붙는다.

## 리믹스 아이디어

1. **갯벌.** 바람 필드를 90° 돌린다. 풀잎은 갈대 그림자가 되고, 1차원
   필드는 물높이가 되고, 호흡 오브는 여섯 시간 밀물 여섯 시간 썰물을
   세는 물때 시계가 된다. 팔레트는 크림 → 젖은 모래, 모스 → 바다유리.
2. **대나무 숲.** L-시스템 규칙을 기둥형 문법(`X → F[+X]F[-X]X`, 더
   좁은 각)으로 바꾸고 획 굵기를 고정한 뒤, 포인터 바람이 숲을
   마림바처럼 연주하게 한다 — 풀잎 스프링의 오버슛마다 희미한 목질
   타격음 하나.
3. **해질녘 도시 공원.** 전부 그대로 두고 반전한다. 롬 배경에 크림
   텍스트, 햇빛은 나트륨등의 호박색이 되고, 꽃가루는 바람 대신
   포인터가 마지막으로 쉬었던 자리로 모여드는 반딧불이가 된다.
