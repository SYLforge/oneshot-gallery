# 페일 시그널 — 제작기

## 훅: 아무도 끄지 않은 터미널

2019년에 퇴역한 산꼭대기 관측소가 있다. 그런데 그 사실을 터미널에게는
아무도 말해 주지 않았다. 그래서 터미널은 지금도 밤마다 부팅하고, 세 개의
접시 중 두 개를 하늘로 돌리고, 들리는 것을 받아 적는다. 펄서의 박동,
달의 웅웅거림, 빗소리를 닮은 잡음. 페이지는 이 설정을 한 번도 설명하지
않는다. 방문자는 교대 근무 중간에 도착한 사람처럼 BIOS가 올라가는 화면
앞에 서게 되고, 관측소는 당신이 여기 온 이유를 이미 안다는 듯 군다.

이 디자인의 도박은 하나다. 타임스탬프 하나하나가 문장 노릇을 해낸다면,
로그 파일이 히어로 이미지보다 더 많은 감정을 실어 나를 수 있다는 것.

## 아트 디렉션

터미널 코어는 절제력이 전부다. 초록 하나(`#33ff66` 인광), 검정
하나(`#0a0f0a` — 순검정이 아니라 밝기를 낮춘 브라운관), 그리고 페이지
전체에서 정확히 두 번만 등장하는 경고색 하나. 두 번째 강조색이 끼어드는
순간 이 픽션은 "해커 테마"로 무너진다.

서체 둘은 배역이 다르다. VT323은 기계다 — 로마자, 숫자, `......`로 이어지는
상태 점선. 나눔고딕코딩은 사람이다 — 페이지의 모든 한글을 맡고, VT323의
베이스라인 리듬에 앉도록 `0.8em`으로 정돈했다. 폰트 스택 순서도 일을 한다.
`VT323, Nanum Gothic Coding, monospace` 순서라서, 부팅 `<pre>`에 마크업
없이 흘러든 한글도 로마자 폰트를 통과해 자동으로 한국어 폰트에 도착한다.

명세가 한 군데 꺾인 지점은 대비였다. 희미한 트레이스 `#1d5c31`은 튜브
블랙 위에서 2.4:1 — 아름답지만 읽을 수 없는 수치다. 그래서 순수 장식
(괘선, 섹션 번호, 가장 어두운 ASCII 셀)으로 강등시키고, 보조 텍스트에는
파생 토큰을 새로 만들었다. 인광 72% 알파, 합성하면 대략 `#27b54a`,
대비 7.2:1.

## 기법 1 — 거짓말하지 않는 타자기

부팅 로그는 방문자가 처음 보는 것이자 첫 번째 접근성 함정이다. 타이핑
효과는 대개 "기다릴 수 없는 사람들"에게서 텍스트를 숨긴다. 두 가지
결정으로 피했다.

첫째, 서버는 *부팅이 끝난* 화면을 렌더링한다. `play`는 이펙트 안에서만
켜지므로, 자바스크립트가 없으면 이 페이지는 죽은 커서가 아니라 완성된
터미널이다:

```tsx
// BootSequence.tsx — SSR 상태는 부팅이 끝난 화면
const [play, setPlay] = useState(false);
useEffect(() => {
  if (reduced) { finish(); return; }        // 모션 최소화: 완성 상태 유지
  const id = window.setTimeout(() => setPlay(true), 60);
  return () => window.clearTimeout(id);
}, [reduced, finish]);
```

둘째, 타이핑 속도는 일부러 고르지 않다. 실제 UART는 버스트로 밀어내니까.
틱의 35%는 글자를 두 개씩 뱉고, 모든 틱에 지터가 붙고, 한국어 사인오프
직전 줄에서는 3.5배 길게 숨을 고른다 — 중요한 말을 하기 전의 호흡:

```ts
// useTypewriter.ts — 일부러 고르지 않게
const burst = Math.random() < 0.35 ? 2 : 1;
char = Math.min(char + burst, current.length);
timerRef.current = window.setTimeout(step, charMs + Math.random() * jitterMs);
// …줄 사이:
const beat = line === lines.length - 1
  ? linePauseMs * 3.5                        // 마지막 줄 앞의 뜸
  : linePauseMs + Math.random() * 160;
```

아무 키, 탭, 휠, 스크롤이면 `finish()`가 호출되고, 7초 하드 타임아웃이
있어서 부팅 화면에 갇히는 일은 구조적으로 불가능하다. 움직이는 `<pre>`는
`aria-hidden`이고, 감싸는 `role="img"` 요소의 `aria-label`에 로그 전문이
들어 있다 — 스크린 리더는 전체 텍스트를 즉시, 한 번에 받는다.

## 기법 2 — ASCII로 그린 하늘

시그니처 섹션은 장면을 문자 밀도로 래스터화한다. DPR 대응 캔버스,
약 14px 셀, 고전적인 램프 `" .:-=+*#%@"`, 셀마다 `fillText`. 장면은 세
개의 장(場)의 합이고, 프레임마다 셀 단위로 평가된다:

```ts
// SignalField.tsx — 밀도 = 별 + 반송파 + 블룸
const r = hash(cx, cy);                      // 셀마다 고정된 결정론적 별
if (r > 0.962) {
  const tw = 0.5 + 0.5 * Math.sin(t * (0.0006 + r * 0.0011) + r * 251);
  v += ((r - 0.962) / 0.038) * (0.22 + 0.55 * tw);
}
const wave =                                  // 서로 무관한 속도의 사인 셋
  Math.sin(x * 8.3 + t * 0.00093) * 0.36 +
  Math.sin(x * 21.1 - t * 0.00061) * 0.17 +
  Math.sin(x * 3.4 + t * 0.00037) * 0.47;
const d = ((y - (0.58 + wave * 0.15)) * rows) / 7.5;
v += Math.exp(-d * d) * (0.42 + 0.3 * (0.5 + 0.5 * Math.sin(x * 26 + t * 0.0016)));
```

별은 `Math.random()`이 아니라 공간 해시다. 셀마다 별의 자리가 프레임을
넘어 고정되고, 반짝임 속도도 해시에서 나온다. 반송파 띠는 공약수가 없는
주파수·속도의 사인 세 개라서 파형이 눈에 보이게 반복되는 일이 없다.

세 번째 장은 방문자 자신이다. 가우시안 블룸이 포인터를 향해 lerp로
따라온다. 터치 기기이거나 포인터가 2.6초 침묵하면 자율 리사주 드리프트가
넘겨받는다 — 관측소는 다시 혼자서 하늘을 훑는다. 셀은 팔레트 토큰 네
개로 버킷팅해서 버킷당 `fillStyle` 한 번씩만 바꾼다. 한 프레임의 스타일
변경이 4천 번이 아니라 네 번이 된다.

캔버스 폰트는 CSS에서 가져온다(`getComputedStyle(canvas).fontFamily`).
그래서 ASCII 글리프가 진짜 VT323이다 — 하늘도 페이지와 같은 목소리로
조판된다.

## 기법 3 — CRT 유리

페이지 전체가 포인터를 통과시키는 고정 3겹 오버레이 아래에 있다. 3px마다
1px 주사선, 위에서 내려오는 유리 글레어, 모서리 비네트. 그런데 이걸
팔리게 만드는 건 그라디언트가 아니라 타이밍이다. 글레어는 9.4초 주기로
두 번 숨 쉬고, 콘텐츠 레이어는 12.4초마다 딱 한 프레임 옆으로 휘청하고,
커서는 1.06초로 깜빡인다. 세 주기에 공약수가 없어서 배경의 움직임이
루프로 읽히지 않는다:

```css
/* 12.4초마다 한 번의 휘청 — 약 19ms짜리 키프레임 세 조각 */
@keyframes ps-jitter {
  0%, 99.4%, 100% { transform: translateX(0); }
  99.55% { transform: translateX(3px) skewX(-0.6deg); }
  99.7%  { transform: translateX(-2px); }
  99.85% { transform: translateX(1px); }
}
```

주사선 잉크는 3px당 `rgba(3,8,4,0.28)` 1px로 상한을 두어 최악의 경우에도
텍스트 대비가 AA를 여유 있게 넘는다. `prefers-reduced-motion: reduce`
아래에서는 배경 애니메이션이 전부 제거되고, 유리는 자기 자신의 정지
사진이 된다.

## 모션의 수학

시그널 필드의 느낌은 사실상 숫자 두 개가 만든다.

- **위치 lerp 0.085/프레임.** 매 프레임 남은 거리의 8.5%를 좁힌다
  (`dt / 16.7`로 정규화해서 144Hz에서 두 배로 빨라지지 않는다). 반감기
  ≈ 7.8프레임 ≈ 130ms — CSS 커서가 아니라 무거운 장비가 선회하는
  듯한 지연이다.
- **에너지 attack 0.06, release 0.025.** 일부러 비대칭이다. 필드는
  반감기 11프레임 만에 당신 쪽으로 밝아지고, 잊는 데는 27프레임이
  걸린다. 관심은 빨리 오고 천천히 식는다 — 상수 두 개에 담긴 이
  관측소의 성격 전부.

리빌 이징 `cubic-bezier(0.16, 1, 0.3, 1)`은 이동량의 약 80%를 앞쪽 40%
구간에 몰아넣는다. 16px 이동이면 꼬리는 서브픽셀로 가라앉아서 600ms가
느린 게 아니라 차분하게 읽힌다. 스태거는 인덱스 순이 아니라 옵저버
배치 단위 70ms — 같이 도착한 요소들끼리만 줄을 선다.

## 성능 노트

- rAF 루프는 하나, 약 64fps로 스로틀(`FRAME_MS = 15.5`) — 고주사율
  디스플레이가 두 배로 계산하지 않는다. `dt`는 48ms로 클램프해서
  백그라운드 탭에서 돌아와도 휘청하지 않는다.
- 120px 마진의 IntersectionObserver가 루프를 켜고 끈다 — 화면 밖에서는
  0프레임.
- DPR 상한 2. 1440×620이면 약 4,500셀이고 그중 대략 3분의 1만 글리프를
  그린다. 버킷 드로잉으로 프레임당 `fillStyle` 대입은 네 번.
- CSS 애니메이션은 전부 `transform` / `opacity` / `filter` —
  컴포지터 전용이다. 주사선·비네트는 정적 그라디언트.
- 로그 리빌은 IO 콜백에서 클래스와 `transitionDelay`를 직접 쓴다.
  항목마다 리액트 리렌더가 없고, 타자기는 `<pre>` 하나만 다시 그린다.

## 접근성 노트

- **모션 최소화:** `usePrefersReducedMotion`(matchMedia 위의
  `useSyncExternalStore`)이 부팅을 즉시 완성시키고, ASCII 필드는 잘
  구성된 정지 프레임 하나(반송파가 보기 좋게 놓이는 `t = 47000`)로
  멈추며, 미디어 쿼리가 플리커·지터·깜빡임·리빌 전환을 제거한다.
- **JS 없이:** 리빌 이전의 숨김 상태는 마운트 때 붙는 `.ps-js` 클래스
  아래에만 존재하고, 접시 노트의 숨김도 같은 방식이다. 자바스크립트가
  꺼져 있으면 페이지 전체가 그냥 읽힌다.
- **시맨틱:** 부팅 로그와 캔버스는 `role="img"`에 전문/설명 레이블
  (캔버스는 한·영 병기), 움직이는 글리프는 `aria-hidden`. 접시 행은
  네이티브 `<button>`에 `aria-expanded`/`aria-controls`. 포커스는
  `:focus-visible`에만 반응하는 2px 인광 링.
- **대비:** 인광 14.5:1, 보조 7.2:1, 앰버 10.6:1 — 주사선 최악 조건에서도
  AA 이상. 2.4:1짜리 희미한 트레이스는 의미를 나르지 않는다.

## 리믹스 아이디어 셋

1. **조위 관측소.** 별 해시를 조석 조화상수의 느린 수평 스캔으로 바꾸고,
   인광을 얼음 파랑(`#7ad7ff` 계열)으로. 바다가 시간마다 한 말을 적는
   로그. 리사주 드리프트는 부표가 된다.
2. **난수 방송국.** 구조는 그대로 두고 로그를 일정 간격의 다섯 자리 숫자
   그룹으로 교체. 앰버로 표시된 항목은 숫자가 달라졌던 단 하룻밤.
   타자기에는 단파 특유의 워블(피치가 흔들리는 타건 간격)을 더한다.
3. **지진계 수장고.** 반송파 띠를 90도 돌려 세로 드럼 기록으로,
   셀은 `|‖▌█`로. 포인터 블룸은 바늘에 걸리는 압력이 되고 — 페이지는
   *당신이* 너무 빨리 움직일 때만 한 프레임 떨린다.
