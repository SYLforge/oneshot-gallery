# 기와 — 처마 아래 단청 · 제작기

이 문서가 원본이다. 영어 브레이크다운은 이 글의 번역 노트다 — 페이지가
그렇듯이.

## 훅: 도감의 여덟 번째 도판

궁궐 처마를 다시 칠하는 공방이 있다고 하자. 그들은 칠하기 전에 기록부터
한다 — 색의 체계, 문양의 짜임, 손의 순서. 이 페이지는 그 기록집
단청도감의 여덟 번째 도판이다. 단청에 *관한* 웹사이트가 아니라 단청의
작업 문서 그 자체: 오방색이 먼저 오고, 문양이 오고, 법식이 오고,
마지막에 공방의 서명이 온다.

디자인의 도박은 하나다. 박물관 라벨의 위엄과 붓을 실제로 잡아 본 사람의
정확함이 같은 문장 안에 있을 수 있다는 것. 그래서 카피는 전부
단정형으로 끝난다 — 한다, 이다. 노래하지 않는다. 기록한다.

## 아트 디렉션

한국 전통을 다루는 페이지가 가장 쉽게 미끄러지는 자리는 장식이다.
용 한 마리, 태극 하나, 세피아 필터 — 그 순간 유산은 무드보드가 된다.
이 페이지의 규율은 반대다: **구조가 곧 장식이다.** 모든 색은 방위를
가진 오방색이고, 모든 문양은 단위 하나와 대칭 연산으로 지어지고, 모든
애니메이션은 실제 단청의 작업 순서 — 초 먼저, 색은 나중 — 를 반복한다.

색의 법도 하나 더. 지붕면은 칠하지 않는다. 구운 기와는 회색으로 남고,
단청은 처마 *밑*에만 산다 — 실제 건물이 그렇기 때문이다. 절제가 곧
고증이다.

대비는 명세가 한 번 꺾인 자리다. 뇌록 `#58796f`은 한지 위에서 4.2:1 —
바탕색으로는 완벽하지만 본문으로는 모자라다. 그래서 뇌록은 면과 큰
글자로 강등시키고, 보조 텍스트(영문 번역 노트, 캡션)에는 파생 토큰
`noerok-deep #3e5850`(6.7:1)을 새로 만들었다. 석황 `#d9a441`은 한지
위에서 2.0:1이므로 텍스트로는 절대 쓰지 않는다 — 씨방과 방울, 금 점.
황이 놓인 자리가 문양의 중심이 되는 것이 원래 이 색의 소임이다.

## 기법 1 — 처마선: 긋고, 물들이기

히어로의 지붕은 SVG 두 장이 같은 기하를 공유한다. 곡선은 손으로 찍은
좌표가 아니라 코드가 계산한다 — 처마선은 3차 베지에 하나로 정의되고,
수막새 스물여섯 개와 연목 열여덟 개는 그 곡선을 오프셋해서 표본을 뽑은
자리에 놓인다:

```tsx
// Hero.tsx — 처마 곡선: 양끝이 들리고, 가운데가 쉰다
const EAVE: Cubic = [
  { x: 48, y: 300 }, { x: 320, y: 352 },
  { x: 880, y: 352 }, { x: 1152, y: 300 },
];

// 막새·연목 줄: 처마의 평행 오프셋 곡선 위 등간격 표본
function row(n: number, dy: number, inset: number): Pt[] {
  const curve = offsetEave(dy);
  return Array.from({ length: n }, (_, i) => {
    const t = inset + (1 - 2 * inset) * ((i + 0.5) / n);
    return cubicAt(t, curve);
  });
}
```

첫 장(먹 레이어)은 스스로를 긋는다. 모든 패스에 `pathLength={1}`을 주면
실제 길이와 무관하게 대시 좌표계가 0–1로 정규화되므로, CSS 한 줄이 모든
획을 다룬다:

```css
.giwa-js .giwa-hero__art .giwa-ink-draw {
  stroke-dasharray: 1;
  stroke-dashoffset: 1;
  animation: giwa-draw 1.1s var(--giwa-ease-brush) both;
  animation-delay: var(--d, 0s); /* 용마루 0s → 처마 .55s → 기둥 1.15s */
}
```

순서는 목수의 순서다: 용마루, 망와, 내림마루, 기왓골 열한 줄, 그리고
가장 긴 획인 처마선은 1.5초를 받는다. 둘째 장(채색 레이어)은 같은
지붕을 다 칠해 놓은 그림이고, 마지막 먹획이 앉는 2.05초에 클립패스
스윕으로 왼쪽부터 밀려 들어온다:

```css
@keyframes giwa-sweep-in {
  from { clip-path: inset(0 100% 0 0); }
  to   { clip-path: inset(0 0 0 0); }   /* 선화가 색을 입는다 */
}
```

두 레이어 구조의 진짜 이유는 접근성이다. 채색 레이어의 숨김 상태는
`.giwa-js`가 붙어야만 존재하므로, 자바스크립트가 없으면 페이지는 처음부터
다 칠해진 도판이다. 모션 최소화 설정에서도 같은 최종 프레임이 즉시
보인다.

## 기법 2 — 연화문: 잎 하나, 회전 여덟 번

문양 갤러리의 규칙은 단청장이 실제로 쓰는 규칙이다: **단위 하나를
정의하고, 나머지는 대칭에 맡긴다.** 연화문 메달리온 전체에서 손으로 그린
잎은 단 하나다:

```tsx
// PatternGallery.tsx — 잎 하나가 여덟 번 돈다
<defs>
  <g id="giwa-lotus-p">
    <path className="giwa-f-seok giwa-s-ink2" d={LOTUS_PETAL} />   {/* 석간주 몸 */}
    <path className="giwa-f-hwang" d={LOTUS_PETAL_IN} />           {/* 석황 속잎 */}
    <path className="giwa-s-baek" fill="none" d={LOTUS_PETAL_LIGHT} /> {/* 호분 빛선 */}
  </g>
</defs>
{Array.from({ length: 8 }, (_, i) => (
  <use key={i} href="#giwa-lotus-p" transform={`rotate(${i * 45} 110 110)`} />
))}
```

잎 자체가 이미 단청의 문법을 담고 있다: 굵은 먹 윤곽, 그 안의 색 띠,
그리고 색과 색 사이의 흰 숨(빛넣기). 잎 사이 22.5도 자리에는 녹화가
돋고, 삼청 띠가 전체를 감고(호분 점선은 `stroke-dasharray`가 그린다),
바깥 열여섯 개 갈래는 같은 패스에 `use`의 fill만 청·적으로 번갈아
물려받는다 — `defs` 안의 도형이 fill을 정의하지 않으면 인스턴스가
물려주기 때문이다.

머리초는 반대 연산이다. 왼쪽 끝을 한 번 짓고 `scale(-1, 1)`로 오른쪽에
비춘다 — 보의 양 끝에 같은 머리초가 앉는 실제 법식 그대로. 금문은 소용돌이
폴리라인 하나를 세 번 겹쳐 긋는다(먹 16.5 아래 호분 13 아래 삼청 7.5 —
테두리 두른 리본이 selector 세 줄로 나온다), 그리고 52픽셀씩 여섯 번
민다.

각 문양에는 작은 먹 레이어 — 컴퍼스 원과 방사선, 진짜 초 그리기의
뼈대 — 가 따로 있어서 스크롤로 들어올 때 먼저 그어지고, 칠해진 문양이
그 위로 물든다. 메달리온은 씨방에서 피어나는 `circle()` 블룸, 보와 띠는
왼쪽에서 스윕, 방울은 위에서 아래로 — 매달린 것은 위부터 칠하니까.

## 기법 3 — 한지의 결

종이는 두 겹이다. 페이지 전체에는 정적 feTurbulence 한 장이 멀티플라이로
깔린다(fractalNoise 0.75, 알파 ≈ 0.07). 법식 섹션의 무대는 세 겹인데,
여기서 중요한 것은 두 번째 레이어의 비등방 주파수다:

```tsx
// HanjiMethod.tsx — 닥섬유에는 결의 방향이 있다
<feTurbulence type="fractalNoise" baseFrequency="0.008 0.18" numOctaves="3" />
```

`baseFrequency`에 값을 두 개 주면 노이즈가 한 축으로 늘어난다 —
가로로 길게 찢어지는 섬유 줄기. 한지를 뜰 때 발이 지나가는 방향이 남는
것과 같은 이치다. 세 레이어는 포인터를 따라 깊이 4 / 8 / −5픽셀로
흐른다(한 겹은 반대로 — 종이 겹 사이의 시차). 훅은 정규화된 좌표를
lerp(프레임당 0.09, dt 보정)로 따라가 CSS 변수 두 개만 쓰고, 상한은
CSS 쪽 곱셈이 갖는다. 터치에서는 아예 붙지 않고, 모션 최소화에서는
종이가 평평하게 눕는다.

## 모션 수학

노리개는 장식이지만 물리는 진짜다. 각도 하나의 감쇠 진자:

```ts
// Footer.tsx — θ″ = −kθ − cω, 스크롤이 각속도에 충격을 준다
theta += omega * dt;
omega += (-16 * theta - 2.1 * omega) * dt;   // k=16 → 고유 진동 ≈ 0.64Hz
omega += clamp(dy * 0.011, -0.9, 0.9);        // 스크롤 임펄스
```

k = 16이면 고유 진동수 √16 / 2π ≈ 0.64Hz — 손가락 길이의 술이 실제로
흔들리는 박자다. 감쇠비 ζ = c ⁄ 2√k ≈ 0.26: 두어 번 되돌아 흔들리고
잠잠해진다. 각도는 ±0.24rad로 클램프해서 아무리 험하게 스크롤해도
문진처럼 뒤집히지 않는다. 루프는 IntersectionObserver로 화면 밖에서
멈추고, |θ|와 |ω|가 바닥나면 스스로 주차한다 — 쉬는 진자는 rAF를 한
프레임도 쓰지 않는다.

패럴랙스 lerp 0.09의 반감기는 약 7.4프레임(≈ 123ms) — 종이가 손을
따라오되, 커서처럼 붙지는 않는 지연이다.

## 성능 노트

- 지속 rAF 루프는 없다. 진자와 패럴랙스 모두 필요할 때 깨어나 수렴하면
  멈추는 자율 주차형이고, 둘 다 화면 밖에서는 완전히 죽는다.
- feTurbulence는 전부 정적이다 — 필터는 한 번 래스터되면 끝. 움직이는
  것은 레이어의 `transform`뿐이라 컴포지터 밖으로 나가지 않는다.
- 히어로의 획 애니메이션은 `pathLength={1}` 덕에 패스당 킥프레임 계산이
  없고, 종료 후에는 채색 레이어가 위를 덮어 먹 레이어는 페인트에서
  사실상 사라진다.
- 리빌은 pale-signal과 같은 IO 패턴: 클래스와 `transitionDelay`를
  명령형으로 써서 항목당 리렌더가 없다.
- 오방색 패널의 색 전환은 트랜지션이 없다 — 의도이자 절약이다.

## 접근성 노트

- **모션 최소화:** 훅(useSyncExternalStore + matchMedia)이 리빌·패럴랙스·
  진자를 JS에서 끄고, CSS 미디어 블록이 획·플러드·등장을 모두 지운다.
  결과는 처음부터 완성된 도판 — 정보 손실이 0이다.
- **无 JS:** 숨김 전 상태는 전부 `.giwa-js` 뒤에만 존재한다. 오방색
  노트 다섯 개는 JS 없이 카탈로그처럼 전부 펼쳐진 채 읽힌다.
- **터치·키보드:** 오방색 패널은 진짜 button이고 hover·focus·tap이 같은
  일을 한다(`aria-expanded` + `aria-controls`). 포커스 링은 석간주 2px +
  옅은 안료 후광, `:focus-visible` 한정.
- **시맨틱:** 지붕과 각 문양은 `role="img"`에 한·영 서술 라벨을 달고,
  장식 레이어는 `aria-hidden`이다. 루트가 `lang="ko"`, 영문은 전부
  `lang="en"` — 낭독기가 언어를 바꿔 읽는다.
- **대비:** 본문 14.7:1, 강조 6.6:1, 보조 6.7:1. 4.2:1의 뇌록과 2.0:1의
  석황은 의미를 싣지 않는다.

## 리믹스 아이디어

1. **단청 야간 점검.** 같은 기하, 배경만 먹빛으로 — 손전등(포인터)이
   지나가는 자리만 클립패스 원으로 채색이 드러나는 야간 실측 도판.
2. **조각보 도감.** 연화문의 회전 대칭을 조각보의 병진 대칭으로 바꾼다.
   feTurbulence는 모시 결로, 획 그리기는 바느질 선으로 — 시침선
   `stroke-dasharray`가 이미 절반은 해 준다.
3. **수리 전·후 슬라이더.** 히어로의 두 레이어 사이 클립패스를 시간이
   아니라 드래그에 물린다 — 왼쪽은 바랜 단청, 오른쪽은 복원 후. 공방
   픽션이 그대로 인터랙션이 된다.
