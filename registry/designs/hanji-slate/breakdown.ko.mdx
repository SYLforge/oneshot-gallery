# 한지 슬레이트 — 제작기

## 훅: 말을 멈출 줄 아는 화면

서울 어딘가의 작은 회사가, 화면의 문제는 한시도 말을 멈추지 않는 것이라고
결론 내렸다. 그래서 멈출 줄 아는 화면을 만들었다. "한지, 업그레이드" —
천 년을 가는 닥종이에 기억을 달아 준 전자잉크 슬레이트. 페이지는 이 주장을
한 번도 설득하려 들지 않는다. 사양표가 숫자를 말하듯 말할 뿐이다. "한 번의
충전으로 여섯 주의 아침." 고요함이 알아서 설득한다.

이 디자인의 도박은 하나다. 사진 한 장, 그라디언트 하나, 유리 패널 하나
없이도 — 레이아웃 자체가 제품처럼 굴면, 그러니까 무광이고 정확하고 건드리기
전까지 조용하면 — 프리미엄은 성립한다는 것.

## 아트 디렉션

벤토 프로덕트라는 장르는 보통 어두운 그라디언트와 글로우를 뜻한다. 이
페이지는 장르의 온도를 뒤집었다. 책상이 되어 주는 웜 그레이 `#e9e6e0`,
타일의 전자잉크 화이트 `#f7f5f0`, 모든 글자의 흑연 `#26262a`, 그리고 감빛
먹 하나 — 앰버 `#e8830c`. 기기는 SVG로 그렸다. 베젤, 결을 새긴 화면,
이차곡선 몇 가닥으로 흘려 쓴 손글씨 한 페이지와 앰버색 밑줄 한 줄. 그래서
어떤 해상도에서도 무너지지 않고, 독서 모드가 바뀌면 기기도 같이 먹을
갈아입는다.

명세가 꺾인 곳은 대비였다. 밝은 앰버는 밝은 바탕에서 2.4:1 — 아름답지만
읽을 수 없는 수치다. 그래서 앰버를 둘로 나눴다. 파생된 딥 앰버
`#9a5504`(웜 그레이 위 4.6:1)가 기능적인 일 — 핵심 수치, 단위, 포커스 링 —
을 맡고, 밝은 앰버는 장식과 흑연 푸터(그 위에서는 5.8:1)를 맡는다. 같은
색 계열, 서로 다른 두 개의 근로계약서.

서체 셋이 세 가지 목소리를 낸다. Hanken Grotesk는 제품이 말하고, Noto Sans
KR은 페이지의 모든 한글을 맡고(`word-break: keep-all` — 어절이 중간에서
부러지지 않는다), JetBrains Mono는 계기판을 읽는다. 산세리프 스택과 모노
스택 모두 Noto를 두 번째에 두어서, 모노 수치 안에 흘러든 한글조차 마크업
없이 제 서체에 도착한다.

## 기법 1 — 벤토 FLIP (flip-layout)

타일을 펼치면 그리드 배치가 `span 2`에서 `1 / -1`로 바뀌고, 모든 형제
타일이 재배열된다. 그런데 레이아웃은 절대 애니메이션하면 안 되는 바로 그
속성이다. FLIP이 이 모순을 푼다. 레이아웃은 즉시 일어나게 두고, 그 *변화의
겉모습*만 트랜스폼으로 애니메이션하는 것.

사각형 측정은 상태 변경 전에, 동기적으로 일어난다:

```tsx
// BentoGrid.tsx — First는 React가 변화를 알기 전에
const toggle = (id: string) => {
  if (!reduced) flip.capture();          // 모션 최소화: 측정 없음 → 즉시 스냅
  setOpen((prev) => (prev === id ? null : id));
};

useLayoutEffect(() => {
  flip.play();                           // Last-Invert-Play, 페인트 직전에
}, [open, flip]);
```

`play()`는 `useLayoutEffect`에서 돈다 — 그리드가 재배열된 후, 브라우저가
그리기 전. 움직인 타일마다 트랜지션을 끈 채 예전 자리로 되돌려 놓고, 두
프레임 뒤에 풀어 준다:

```ts
// useFlip.ts — Invert…
el.style.transition = "none";
el.style.transformOrigin = "0 0";
el.style.transform = `translate(${dx}px, ${dy}px) scale(${sx}, ${sy})`;
if (inner) inner.style.transform = `scale(${1 / sx}, ${1 / sy})`;

// …Play: 첫 프레임이 반전 상태를 커밋하고, 둘째 프레임이 풀어 준다
requestAnimationFrame(() => {
  requestAnimationFrame(() => {
    el.style.transition = `transform 420ms cubic-bezier(0.22, 1, 0.36, 1)`;
    el.style.transform = "";
  });
});
```

이것이 "늘어나는 상자"가 아니라 "공유 요소 전환"으로 읽히는 데는 두 가지
디테일이 있다. 첫째, 역스케일. 타일 콘텐츠 래퍼가 같은 이징으로 역방향
스케일을 받아서, 상자가 변형되는 동안 글자는 크기를 지킨다. 둘째, 중단이
공짜다. `getBoundingClientRect`는 진행 중인 트랜스폼을 포함하므로, 모핑
도중에 다시 측정해도 타일이 *보이는 바로 그 자리*에서 트윈이 다시 겨냥된다.
아무리 빨리 연타해도 된다.

접근성 쪽은 애니메이션보다 값이 싸다. 토글은 타일 제목 안의 진짜
`<button>`이고 `aria-expanded`와 `aria-controls`를 단다. 타일 전체는 편의용
클릭 영역일 뿐이라, 버튼 위 클릭(이중 발화 방지)과 펼쳐진 본문 안
클릭(텍스트를 긁다가 접히면 안 되니까)은 무시한다. 본문 숨김은 `.slate-js`
뒤에 게이트되어 있어서, 자바스크립트가 없으면 모든 상세 문단이 그냥 보인다.

## 기법 2 — 분해되는 슬레이트 (svg-line-draw)

스크롤하면 기기가 열린다. 280vh짜리 섹션(이 높이 자체가 JS 게이트다)에
스티키 무대가 있고, 스크롤 진행도 훅이 rAF마다 한 번
`p = -rect.top / (rect.height - innerHeight)`를 계산한다. 유리, 패널,
배터리, 마그네슘 등판 — 네 개의 SVG 판이 `p`를 따라 벌어진다:

```ts
// ExplodedView.tsx — 스크럽 전체가 transform / dashoffset / opacity 뿐
const e = smoothstep(clamp01((p - 0.08) / 0.62));
LAYERS.forEach((layer, i) => {
  g.style.transform = `translate3d(0, ${layer.spread * e}px, 0)`;
  const t = clamp01((p - (0.5 + i * 0.09)) / 0.22);
  line.style.strokeDashoffset = String(1 - t);   // pathLength=1: 1→0이 곧 드로잉
  label.style.opacity = String(clamp01(t * 1.4));
});
```

패럴랙스는 숫자 하나다. 바깥 판은 ±132px, 안쪽 판은 ±44px. 이 3:1의 이동
비율이 평면 다이어그램을 깊이로 읽히게 만든다 — 유리와 등판은 "더 먼
곳에서 오는" 것이다.

`svg-line-draw` 태그의 밥값은 콜아웃 선이 한다. 선은 전부
`<path pathLength={1} strokeDasharray={1}>` — 드로잉이 곧 1에서 0으로 가는
dashoffset이라, `getTotalLength()` 측정이 영원히 필요 없다. 그리고 콜아웃은
*자기 판의 그룹 안에* 산다. 선과 라벨이 자기가 설명하는 부품에 올라타
있으니, 어느 스크롤 위치에서든 라벨은 정확히 그 부품을 가리킨다.

정적 상태는 공짜다. 기본값이기 때문이다. 서버가 렌더링하는 마크업은 콜아웃이
다 그려지고 라벨이 다 보이는 *조립된* 다이어그램이다. JS가 없거나 모션
최소화면 그 상태가 그대로 남고(미디어 쿼리가 280vh 활주로도 평범한 섹션으로
접는다), 스크럽은 마운트 이후에 자바스크립트로만 숨김을 *추가*한다.

## 기법 3 — 포인터 패럴랙스, 그리고 전자잉크의 깜빡임

히어로의 슬레이트는 6.5초짜리 CSS 키프레임으로 떠 있고 — 페이지의 유일한
상시 모션이다 — 정밀 포인터에서는 커서 쪽으로 기운다. 기기는 ±7px, 그림자는
반대로 ∓4px. 서로 반대로 움직이는 두 평면이 정직한 패럴랙스의 최소 단위다.
lerp(60fps 정규화 프레임당 `k = 0.09`)는 rAF 루프에서 돌다가 두 평면이
가라앉으면 스스로 주차한다. 가만히 있는 히어로는 프레임을 한 장도 쓰지
않는다. 터치 기기와 모션 최소화에서는 루프가 아예 시작되지 않는다.

푸터의 독서 모드 토글은 루트의 `data-mode="sepia"`로 종이 토큰을 갈아
끼우면서 — 진짜 전자종이처럼 화면 전체를 한 번 반전시킨다:

```css
.slate-js.is-refreshing { animation: slate-eink 120ms linear both; }

@keyframes slate-eink {
  0%, 11%   { filter: invert(0); }
  12%, 82%  { filter: invert(1); }   /* 하드 컷 — 전자잉크는 스냅하지, 페이드하지 않는다 */
  83%, 100% { filter: invert(0); }
}
```

속성 하나, 레이어 하나. 그리고 반전된 프레임은 정말로 페이지 자신의
음화(陰畫)다 — 실제 전자잉크 리프레시가 보여주는 것이 정확히 그것이다.
연타하면 일부러 진행 중인 플래시에 올라탄다(실제 전자종이도 리프레시를
묶어서 처리한다). 모션 최소화에서는 클래스가 아예 붙지 않고, 미디어 쿼리가
키프레임까지 이중으로 막는다. 모드는 그래도 즉시 바뀐다.

## 모션의 수학

- **FLIP 이징.** `cubic-bezier(0.22, 1, 0.36, 1)`는 이동량 대부분을 앞에
  몰아 준다. 420ms 중 펼쳐진 타일은 약 170ms 만에 도착하고 나머지는
  서브픽셀로 가라앉는 데 쓴다 — "느긋함"이 아니라 "확신"으로 읽힌다.
- **역스케일의 함정, 수치로.** 바깥 `scale(s)`와 안쪽 `scale(1/s)`가 같은
  이징을 통과해도, 둘의 곱은 양 끝점에서만 정확히 1이다. 이 그리드의 최악
  케이스(1×1 타일이 전체 폭으로, s ≈ 2.0)에서 트윈 중간의 콘텐츠 흔들림은
  몇 프레임 동안 2% 미만 — 지각 아래이고, `width`/`height`를
  애니메이션하지 않는 대가로는 싸다.
- **스크럽 구간.** 판들은 p∈[0.08, 0.70]을 smoothstep으로 통과한다(양
  끝에서 속도가 0 — 판은 *도착*하지, 부딪히지 않는다). 콜아웃 i는
  p∈[0.5+0.09i, 0.72+0.09i]에서 그려진다. 첫 선은 판들이 아직 흐르는 중에
  시작하고, 마지막 선은 무대가 풀리기 직전에 끝나며, 0.09의 시차 덕에
  다이어그램은 참을성 있는 선생처럼 위에서 아래로 자기 라벨을 단다.
- **120ms.** 실제 전자종이의 전체 리프레시는 100–450ms. 반전 유지 약
  85ms를 품은 120ms는 "글리치"가 아니라 "의도된 책장 넘김"으로 읽히는
  가장 짧은 길이였다.

## 성능 노트

- FLIP은 이미 그려진 타일의 `transform`만 애니메이션한다. 레이아웃은 React
  커밋에서 정확히 한 번 돈다. `will-change`는 트윈 동안만 붙였다 청소된다.
- 스크럽은 프레임당 `getBoundingClientRect` 한 번(rAF 스로틀, passive
  스크롤 리스너), 쓰기는 `transform`·`stroke-dashoffset`·`opacity`뿐 —
  읽기와 쓰기가 절대 섞이지 않는다.
- 히어로 패럴랙스 루프는 가라앉으면 스스로 멈춘다. 쉬고 있는 페이지에
  도는 것은 컴포지터 전용 CSS 애니메이션(플로트) 단 하나다.
- 이미지도 캔버스도 상시 필터도 없다. 전자잉크 반전은 토글당 120ms 존재하고
  사라진다.
- 폰트는 `display: swap`의 next/font 세 종. SVG 기기는 패스 약 2KB다.

## 접근성 노트

- **모션 최소화:** `usePrefersReducedMotion`(`matchMedia` 위의
  `useSyncExternalStore`)이 토글을 스냅으로 만들고(FLIP은 측정조차 안
  한다), 분해도는 라벨이 달린 조립 상태로 활주로까지 접힌 채 남고,
  전자잉크 플래시는 건너뛴다. 미디어 쿼리가 모든 CSS 애니메이션을 한 번 더
  막는다.
- **No-JS:** SSR 상태가 곧 완성된 인쇄물이다 — 모든 벤토 상세가 보이고,
  다이어그램은 조립되어 있고, 모든 카피가 DOM에 있다. `.slate-js`는 오직
  능력을 *더할* 뿐이다.
- **시맨틱:** 타일 토글은 진짜 제목 안의 `<button aria-expanded
  aria-controls>`, 모드 스위치는 `aria-pressed`, 두 SVG는 이중 언어 라벨의
  `role="img"`, 좁은 화면의 분해도 범례는 진짜 `<ol>`이다.
- **대비:** 흑연 12.1:1, 보조 5.3:1, 딥 앰버 4.6:1 이상 — 라이트와 세피아
  두 바탕 모두에서 AA. 밝은 앰버는 밝은 바탕에서 절대 텍스트를 나르지
  않는다. 포커스는 `:focus-visible` 전용 2px 앰버 링 — 밝은 바탕에서는
  딥, 흑연 위에서는 브라이트.

## 리믹스 아이디어

1. **야전 라디오.** 따뜻한 절제는 그대로 두고 슬레이트를 수동 크랭크
   비상 라디오로. 벤토 FLIP은 그대로, 분해도는 스피커/코일/크랭크/셸로
   벌어지고, 모드 토글은 AM/FM 전환 — 120ms 반전은 한 프레임짜리 화이트
   노이즈 *팟* 소리가 된다.
2. **필름 카메라.** 세피아 모드가 "암실"(깊은 적색 토큰)이 되고, 분해도는
   렌즈 스택을 연다 — 선 드로잉 콜아웃이 렌즈군을 라벨링한다. FLIP 타일은
   조리개/ISO/셔터 수치를 모노로 나른다.
3. **기계식 시계.** 분해 축을 돌린다: 판들이 크라운을 중심으로 방사형으로
   벌어진다. 스크럽이 분리와 함께 픽셀당 1도의 밸런스 휠 회전을 몰고, 사양표에는
   시간당 진동수가 오르며, 플래시는 반전 대신 헤어스프링의 반짝임
   (`filter: brightness`)이 된다.
