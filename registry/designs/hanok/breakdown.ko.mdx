# HANOK — 제작 설명

## 시작하는 말

하회 마을에 1807년에 지은 집이 한 채 있고, 집담(집담 아틀리에)이라는 작은
기록실이 그 집의 결구를 하나하나 기록하고 있다. 이 페이지는 그 기록이다.
핵심은 글자 그대로이다 — **집의 목조 구조가 곧 페이지의 레이아웃이다.** 집을
붙잡는 장부이음(못 없이 나무가 나무를 물린 결구)이 그리드의 논리가 되고,
페이지는 진짜 한옥처럼 뚜렷한 기후 구역을 가진다 — 따뜻하고 닫힌 방,
서늘하고 열린 마루, 하늘에 열린 빈 마당. 그리고 스크롤을 따라 집이 다섯
개의 이름 붙은 층으로 분해된다.

설계의 판단: 문화유산 페이지는 사진이 아니라 구조도일 수 있고, 레이아웃
스스로가 집의 열적(熱的) 논리를 담을 수 있다 — 따뜻함은 밀도로, 서늘함은
열림으로, 마당의 빈 공간은 하늘 아래의 비움으로.

## 아트 디렉션

한국 전통은 보통 단청의 오방색으로 간다. 이 엔트리는 대신 *골조*로 간다:
가공하지 않은 소나무 `#c4a87a`, 더 어두운 구조 들보 `#6b5436` / `#3d2f1c`,
옅은 흙벽 `#d4c4a8`, 마당 돌 `#b8b0a4` — 칠하기 전의 나무와 흙의 색이다.
단청의 한 색, **처마 붉은색 `#9c3a2e`**는 진짜 한옥에서 그 색이 나타나는
자리에만 나타난다 — 처마에. 그것은 휘어진 처마선, 지붕의 들린 끝, 포커스
링, 폭발도에서 장부를 가리키는 단 하나의 핵심 콜아웃, 그리고 결구 상세도의
장부 윤곽이다. 기와(GIWA)의 석간주와 같은 색 family에 대한 응답이지만, 기와가
오방색을 채울 때 이쪽은 빨강 하나를 아껴 쓴다 — 실제 한옥이 단청을 처마
장식으로만 남기듯.

텍스처는 나무다 — `feTurbulence` 시트 한 장이 multiply로 뷰포트 위에 고정되어
있고, 톱으로 켠 소나무의 긴 가로 결 방향(`baseFrequency 0.012 × 0.48`)을
새긴다. 결코 움직이지 않는다. 흙벽의 거칠음은 따뜻한 색과 그 아래 결에서
읽힌다. 활자는 구조적이다: 건축 부재의 이름은 Noto Serif KR이, 이름과 구역
제목은 Nanum Myeongjo가, 청사진 치수는 JetBrains Mono가 읽는다. 세리프가
건축적으로 읽힌다 — 짐을 지고, 나무에 새겨진.

## 기법 1 — 폭발 등각 투상도 (scroll-scrub-pinned)

시그니처. 한옥은 평면도가 되기 전에 단면도다: 휘어진 처마 지붕(처마·지붕),
큰 들보(대들보), 기둥(기둥), 마루바닥(마루), 주춧돌(주춧돌)이 명확한
수직 순서로 쌓이고, 장부이음으로 맞물린다. 페이지는 스크롤을 따라 그 쌓임을
폭발 등각 투상도로 분해한다.

섹션은 320vh(높이는 JS가 켠다 — no-JS에서는 접힌다)이고, 100svh의 sticky
무대가 SVG를 품는다. 스크롤 진행도가 매 프레임마다 각 층 그룹에
`translate3d`를 쓴다:

```ts
// ExplodedAxonometric.tsx — 다섯 층, 각각 부호 있는 이동량
const LAYERS = [
  { id: "roof",        cy: 120, spread: -168, ... },
  { id: "beam",        cy: 210, spread:  -86, ... },
  { id: "column",      cy: 320, spread:  -44, ... },
  { id: "floor",       cy: 410, spread:  +96, ... },
  { id: "foundation",  cy: 470, spread: +168, ... },
];

const apply = useCallback((p: number) => {
  const e = smoothstep(clamp01((p - 0.06) / 0.72));
  LAYERS.forEach((layer, i) => {
    g.style.transform = `translate3d(0, ${(layer.spread * e).toFixed(2)}px, 0)`;
    // ...콜아웃 dashoffset + 라벨 opacity...
  });
}, []);
useScrollProgress(sectionRef, apply, !reduced);
```

이동량의 비율이 솜씨다: 지붕과 주춧돌(바깥 평면)은 168px를 가고, 그 사이
샌드위치는 44–96px를 간다 — **바깥:안쪽 = 3.4:1**, 건축 폭발도의 표준 관행이며,
어떤 3D도 없이 z축을 팔 수 있는 비율이다. 콜아웃 선은 `stroke-dashoffset`
(`pathLength=1`, 수학이 단위 무관하게)으로 엇갈린 구간
`p ∈ [0.30+0.07i, 0.48+0.07i]`에 걸쳐 그어지고, 이중언어 라벨(영문 모노 +
한글 세리프 + 풀이)이 같은 구간의 1.5배에서 나타난다. 각 콜아웃은 자기 층의
`<g>` 안에 있어, 라벨이 늘 가리키는 부재를 따라 간다. 대들보의 콜아웃 —
장부이음을 가리키는 것 — 이 **핵심 콜아웃**이며, 처마 붉은색으로 그어진다.

서버가 그리는 초기 상태는 조립된 단면도이고, 콜아웃은 그어져 있으며 라벨은
보인다. no-JS와 모션 감소는 그 상태를 그대로 두고, CSS는 활주로를 접는다.
720px 이하에서는 SVG 안의 콜아웃이 HTML 설명(legend)으로 넘어간다.

## 기법 2 — 휘어진 처마의 길 위 텍스트 (text-path)

히어로의 이중언어 부제는 휘어진 처마(처마) 자체를 탄다. SVG 패스 하나가 진짜
한옥의 처마선처럼 휘어져 있다 — 넓고 얕게 위로 활처럼 — 그리고 글이 그 위를
`textPath`로 지난다:

```tsx
// Hero.tsx — 처마 붉은색 장식선을 한 번 긋고, 그 위에 부제를 앉힌다
<path
  id="hanok-eave-path"
  className="hanok-eave__curve"
  d="M 24 78 Q 120 30, 280 38 Q 440 46, 536 18"
  pathLength={1}
  fill="none"
/>
<text className="hanok-eave__text">
  <textPath href="#hanok-eave-path" startOffset="50%" textAnchor="middle">
    MORTISE & TENON · NO NAILS · 장부이음 · 못 없는 집
  </textPath>
</text>
```

곡선 자체는 마운트에서 `stroke-dashoffset`(`1 → 0`, 1.6초 `ease-eave`)으로 한
번 그어진다 — 붓이 먹을 머금고 선을 긋듯, 붉은 선이 내려앉고 글은 이미 그 위에
앉아 있다. 모션 감소에서는 dashoffset을 `0`에 둔다(곧장 그어짐); no-JS도
마찬가지. 처마는 단청 붉은색이 선이 되는 유일한 자리다 — 실제 한옥의 처마
장식이 지붕 끝을 따라가듯.

그 아래에 두 번째 SVG가 구조 색으로 박공 실루엣(용마루 + 양 끝의 들린
처마 + 붉은 끝 + 네 기둥 + 주춧돌)을 그려, 히어로가 곧장 알린다 — *이것은
건물이고, 그것이 레이아웃이다.*

## 기법 3 — 제목의 글자 분할 (char-split-reveal)

히어로의 이름 HANOK은 한 덩어리로 나타나지 않고 한 글자씩 자리를 찾는다 —
장부가 장부구멍을 찾듯. 분할은 접근성을 지킨다: `<h1>`은 전체 글자를
`aria-label`로 품고, 움직이는 span들은 `aria-hidden`이다 — 스크린 리더는
"HANOK 한옥"을 한 번 온전히 읽고, 보는 사람은 글자가 자리 잡는 것을 본다:

```tsx
// Hero.tsx — aria-label이 진실을 품고, span은 애니메이션만 담는다
<h1 aria-label="HANOK 한옥">
  <span aria-hidden="true">
    {glyphs.map((t, i) => (
      <span
        key={t.key}
        className="hanok-char"
        style={{ "--d": `${200 + i * 90}ms` } as React.CSSProperties}
      >
        {t.ch}
      </span>
    ))}
  </span>
  <span className="hanok-hero__sub" lang="ko">한옥 · 집이 곧 레이아웃</span>
</h1>
```

`useCharSplit`은 `Array.from`을 써서 코드 포인트 단위로 나눈다 — 그래서 한글
음절은 한 글자로 자리를 잡고(자소로 쪼개지지 않으며), 이는 디스플레이 활자의
올바른 단위다. 애니메이션은 `translateY(0.5em) → 0 + opacity`, 820밀리초
`ease-timber`, 90밀리초 엇갈림. `.hanok-js`가 없으면 모든 글자는 그냥 자리에
앉아 있다(보인다); 모션 감소에서는 CSS가 애니메이션을 멈추고 제목은 곧
완전해진다.

## 기후 구역 — 열적 논리로서의 레이아웃

세 구역 섹션은 세 개의 기능 카드가 아니라, 각각의 *구성*이 그 방의 진짜
열적 성격을 담는 세 가지 공간 처리다 — climate-zone-scroll의 개념.

**ONDOL(온돌방) — 따뜻하고, 닫히고, 밀집.** 이중 테두리와 인셋 그림자를 가진
둘러싸인 패널, 따뜻한 흙벽 배경, 그리고 본문은 **두 단**으로 빽빽이 —
더운 방이 사람을 모으듯. 바닥 아래로 따뜻한 붉은 선이 빛난다(온돌을 눈에
보이게 한 것). 밀도가 따뜻함의 은유다.

**MARU(마루) — 서늘하고, 열리고, 바람 통하는.** 넓은 눈색(snow) 띠, 넉넉한
여백, 그리고 **세 개의 성긴 사실이 3단 그리드에 멀리 떨어져**. 글자들이
나무 마루에서 바람을 쐬는 사람들처럼 서로 멀리 앉는다. 열림이 서늘함의
은유다.

**MADANG(마당) — 중심이고, 하늘에 열리고, 빈 히어로의 순간.** 계절이 바꾸는
하늘색 바닥, 그 위를 지나는 처마 그림자, 그리고 **페이지에서 가장 적은
말** — 계절마다 바뀌는 이중언어 하이쿠 하나. 비움이 하늘 아래 마당의
은유다.

각 구역은 처음 스크롤로 들어올 때 자리를 찾아 올라온다(`useInView →
is-shown`, 700밀리초 `ease-timber`, `translateY 22 → 0 + opacity`). `.hanok-js`가
없으면 모든 구역은 자리에 앉아 있다(보인다).

## 계절 — 나무는 그대로, 빛만 움직인다

마당은 하늘에 열려 있어 계절을 따라 바뀐다. 분할 컨트롤로 봄/여름/가을/겨울을
고르거나, 페이지가 알아서 돌게(약 5.2초마다 한 계절) 둘 수 있다. 계절을 고르면
루트의 `data-season`이 바뀌고, CSS는 **오직** 마당의 하늘 바닥(봄 따뜻 /
여름 초록 바람 / 가을 호박 / 겨울 눈)과 처마 그림자의 각도(계절마다 0 / −6 /
+8 / +14 px)만 다시 칠한다. 나무 결, 흙벽, 모든 글은 그대로 — 나무와 흙은
계절을 따라 바뀌지 않고, 빛만 바뀌기 때문이다. 이 규칙이 페이지가 날씨
위젯처럼 읽히지 않게 한다.

자동 순환은 `setInterval`이고, 탭이 숨겨지면 멈추며, **모션 감소에서는
완전히 얼어붙는다**(그러면 계절은 마지막으로 고른 것, 혹은 봄에 고정된다).
각 선택지는 `aria-pressed`를 가진 진짜 `<button>`이다; AUTO는 별도의
버튼이며, 모션 감소에서는 비활성화된다.

## 모션 감소 · 자바스크립트 없음

약속: 페이지는 모션이 있든, 스크립트가 있든 완전하고 읽힐 수 있는 구조다.
모션 감소에서는, 글자 분할이 자리에 멈추고, 처마는 곧장 그어지며, 폭발도의
활주로는 모든 라벨이 보이는 조립된 단면도로 접히고, 구역은 자리에 있으며,
계절은 얼어 있다. 자바스크립트가 없으면, SSR 상태가 곧 완성된 페이지다 —
모든 사전 리빌 스타일은 결코 도달하지 못하는 `.hanok-js` 클래스 뒤에
숨겨져 있어, 글자는 보이고, 처마는 그어져 있고, 구역은 나타나 있고,
폭발도는 조립되어 있고, 계절은 봄이다. 어떤 것도 읽히기 위해 스크립트에
의존하지 않는다.
