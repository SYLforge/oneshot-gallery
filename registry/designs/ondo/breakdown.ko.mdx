# 온도 — 제작기

## 훅: 향의 이름이 온도인 집

한남동 골목 안의 어느 향수 하우스는 자기 향이 무엇을 닮았는지 끝내 말하지
않는다. 대신 온도를 말한다. 36.5도는 몸이고, 영하 2도는 강 위에서 숨을
참는 첫눈이고, 45도는 어제의 불을 간직한 온돌 바닥이다. 페이지는 이 집의
하나뿐인 어두운 방이다 — 무대 위의 병 하나, 그 앞에 세워 둔 오래된 유리
한 장, 그리고 감정이 있는 온도계처럼 읽히는 카피.

이 엔트리의 도박은 스스로 명시되어 있다. "럭셔리"가 순수한 코드만으로
성립한다는 것을 갤러리에 증명하는 일. 여기에는 래스터 이미지가 단 한 장도
없다. 모두가 수천만 원짜리 제품 촬영을 기대하는 그 병은 SVG
그라디언트이고, 셰이더 라이브러리를 기대하는 그 유리는 몇백 번의
`drawImage`와 미리 계산해 둔 노이즈 테이블이다. 이 페이지가 비싸 보인다면,
그건 산수가 해낸 것이다.

## 아트 디렉션

어두운 럭셔리의 실패 구간은 좁다. 한 눈금 밝으면 카페가 되고, 한 눈금
진하면 카지노가 된다. 팔레트는 네 자리만 지킨다 — 누아르 `#16130f`(검정이
아니라 난방이 켜진 어두운 방), 샴페인 `#d8c39a`, 크림 `#efe9df`, 베르메유
`#9c6b3f`. 규율은 배급에 있다. 베르메유는 누아르 위에서 4.0:1밖에 나오지
않아서 의전 역할로 강등되었다. 헤어라인 괘선 둘, 커다란 숫자 하나, 병의
금속. 보조 텍스트는 전부 파생 토큰이 맡는다. 샴페인 72% 알파(합성하면
약 `#a49273`, 6.1:1). 어두운 페이지가 할 수 있는 가장 싸 보이는 일은
저대비 골드를 사방에 바르는 것이고, 가장 비싸 보이는 일은 그것을 거의
쓰지 않는 것이다.

라틴 디스플레이는 이탈리아나(Italiana)가 맡는다 — 굵기가 딱 하나뿐인
헤어라인 세리프라서, 위계는 크기와 자간으로만 세워야 한다. 한글과 본문은
전부 Noto Serif KR이다. 디스플레이 폰트 스택은 `Italiana, Noto Serif KR,
serif` 순서라서, 제목 안에 흘러든 한글은 마크업 없이도 라틴 서체를 통과해
한국어 세리프에 도착한다. 한국어는 번역이 아니라 트랜스크리에이션이다.
"measured in degrees, worn as memory"는 "도(度)로 재고, 기억으로
입는다"로 돌아온다 — 같은 관찰을 다시 느낀 문장.

## 기법 1 — letter-spacing 없는 letter-spacing 애니메이션

히어로 워드마크는 도착과 스크롤에 따라 조밀한 상태(체감 0.02em)에서
공기가 든 상태(0.4em)로 열린다 — 느린 호흡. `letter-spacing`을 직접
움직이면 매 프레임 줄이 리플로우되므로, 이 효과는 그 속성을 아예 건드리지
않도록 설계했다.

h1은 **최종** 자간으로 조판된다. 네 글자는 `aria-label` 뒤에서
`aria-hidden` 스팬으로 분할되고, 각 스팬은 자기 순번만 갖는다. rAF가 모는
커스텀 프로퍼티 하나가 네 글자를 전부 움직인다:

```css
.ondo-wordmark {
  --ondo-breath: 1;      /* JS 없음·모션 최소화의 기본값: 활짝 열린 상태 */
  letter-spacing: 0.4em;
}
.ondo-wordmark__ch {
  display: inline-block;
  transform: translateX(
    calc((var(--ondo-ci, 0) - 1.5) * (1 - var(--ondo-breath)) * -0.38em)
  );
}
```

breath가 0이면 바깥 글자들이 중심 쪽으로 0.57em 끌려온다 — 눈에는 0.02em
자간의 단어다. breath가 1이면 모든 transform이 정확히 0이 된다. 계산이
`em` 단위라서 clamp가 어떤 폰트 크기를 내놓아도 그대로 성립한다. 훅은
`max(intro, scroll)`을 향해 값을 lerp한다:

```ts
// useBreath.ts — 워드마크는 스스로 열리고, 스크롤이 마저 연다
const intro = easeOut(Math.min((now - born) / INTRO_MS, 1)) * INTRO_FLOOR;
const scroll = Math.min(window.scrollY / (window.innerHeight * SCROLL_SPAN), 1);
const target = Math.max(intro, scroll);
value += (target - value) * BREATH_LERP * (dt / 16.7);
```

인트로는 2.6초에 걸쳐 45%까지 연다 — 스크롤하지 않아도 페이지가 숨을
들이쉬는 게 느껴질 만큼. 첫 0.7 뷰포트의 스크롤이 나머지를 완성한다. 다시
올라가면 이름은 인트로 바닥값 쪽으로 가라앉는다. 클립을 재생하는 게
아니라, 숨을 쉬는 것이다. 글자를 실제로 분할했기 때문에 이 엔트리는
`char-split-reveal`을 주장한다. CSS 레이아웃은 한 번도 변하지 않으므로
애니메이션 전체가 컴포지터 전용이다.

## 기법 2 — 유리 한 장 (셰이더 없는 굴절)

시그니처 섹션은 방문자와 SVG 병 사이에 헤어라인 프레임의 캔버스를 세운다.
요령은 캔버스가 *같은 병*을 보여 준다는 데 있다 — 변위 필드를 통과시켜
다시 그린 병. 그래서 이 판은 오버레이 이펙트가 아니라 물결진 유리라는
물체로 읽힌다.

먼저 인라인 SVG가 텍스처가 된다. DOM에서 곧바로 직렬화하고(병 안에
`<text>`가 없는 이유다 — SVG 이미지 안에서는 폰트가 로드되지 않는다),
무대의 단색 배경 위에 오프스크린 캔버스로 한 번만 래스터화한다:

```ts
img.src =
  "data:image/svg+xml;charset=utf-8," +
  encodeURIComponent(new XMLSerializer().serializeToString(svg));
```

그다음 매 프레임, 스트립 변위 2패스. 1패스는 판 영역을 원본 렌더에서 한
줄씩 복사하되 각 줄을 노이즈 필드만큼 수평으로 밀고, 2패스는 그 중간
결과를 세로줄 단위로 복사하며 수직으로 민다:

```ts
// 1패스 — 행: 수평 굴절
const lens = Math.exp(-(dyP * dyP) / 0.055);           // 포인터 가우시안
const n = noiseAt(lut, y01 * 3.2 + t * 0.00022);       // 미리 계산된 필드
let dx = n * (0.9 + 7 * ptr.e * lens) * dpr
       + (ptr.x - 0.5) * 6 * dpr * lens * ptr.e;       // 유리가 "기운다"
midCtx.drawImage(art, offX + dx, sy, paneW, rowH, 0, my, paneW, rowH);
```

노이즈는 64개의 스무스스텝 격자 셀 위에 1024 샘플, 2옥타브로 만든 밸류
노이즈 LUT다. 시드 있는 PRNG에서 마운트마다 한 번 생성되므로 결정론적이고,
이어 붙고, 런타임 비용이 없다. 평상시 진폭은 1px 미만이고, 포인터가
다가오면 손을 따라오는 가우시안 렌즈를 통해 상한 ±7px까지 올라간다 —
프레임당 0.08로 lerp. 이 포인터 항이 `pointer-parallax` 태그의 정직한
근거다. 실제 레이어 둘 — 아래의 선명한 병, 위의 일그러진 유리 — 이 있고,
위층이 포인터를 따라 흐르며, lerp와 상한이 걸려 있다. backdrop-filter도
WebGL도 없다. 태그는 구현과 일치한다.

스트립은 행 1.5px, 열 2px(CSS 기준)라서 440px 판은 프레임당 약 500번의
`drawImage`로 끝난다 — 예산 안에 넉넉히 들어오고, 각 호출은 GPU 쪽
블릿이다. 소스 오프셋이 부동소수점이라 서브픽셀 보간이 공짜로 따라오는데,
물결이 모자이크가 아니라 유리로 보이는 건 그 덕분이다.

실패와 부재도 설계된 상태다. 판은 첫 진짜 프레임이 그려진 뒤에야
나타나고, 이미지가 실패하면 캔버스는 숨고 선명한 SVG가 그대로 남는다.
자바스크립트가 없으면 빈 헤어라인 프레임 너머의 병을 보게 된다 — 마침
완벽하게 고요한 유리 한 장.

## 기법 3 — 쿠튀르 와이프

컬렉션의 다섯 카드는 `clip-path`로 나타나고, 감각의 전부가 베지어 하나에
들어 있다:

```css
.ondo-js [data-reveal="clip"] .ondo-card__art {
  clip-path: inset(0 100% 0 0);
  transition: clip-path 950ms cubic-bezier(0.77, 0, 0.18, 1);
  transition-delay: var(--ondo-reveal-delay, 0ms);
}
.ondo-js [data-reveal="clip"].is-visible .ondo-card__art {
  clip-path: inset(0 0 0 0);
}
```

`cubic-bezier(.77, 0, .18, 1)`은 양 끝이 똑같이 느린 대칭 곡선이다. 커튼이
속도를 얻었다가 그만큼 오래 감속한다 — 흔한 ease-out이 UI로 읽힐 때 이
곡선은 천으로 읽힌다. 옵저버는 같은 배치로 도착한 카드를 90ms 간격으로
세우고, 지연을 `transition-delay`와 `--ondo-reveal-delay` 양쪽에 적는다.
덕분에 카드의 *텍스트*가 같은 박자를 물려받아 이미지보다 220ms 늦게
따라온다 — 이미지 먼저, 캡션 다음. 룩북의 페이지를 넘기는 순서다.

카드 아트도 무이미지 원칙을 정색하고 지킨다. 각 "제품 사진"은 그 향의
온도를 그린 SVG 그라디언트 견본이고, 헤어라인 눈금선의 높이가 곧 온도다 —
영하 2도는 낮고 창백하게, 62도는 높고 달아오른 채로. 시스템 하나, 검침
다섯 번.

## 모션의 수학

- **호흡 lerp 0.055/프레임** (60fps 정규화): 반감기 약 12프레임 ≈ 200ms.
  자간이 스크롤을 따라오는 감각은 무겁고 기름칠이 잘 된 물건이다 —
  존재하되, 조급하지 않다.
- **렌즈 lerp 0.08 / 에너지 attack 0.05 / release 0.02.** 유리는 손을
  향해 도는 데 약 130ms, 밝아지는 데 반감기 230ms, 잊는 데 570ms가
  걸린다. 이 비대칭이 성격이다. 관심은 정중하게 도착해서 오래 머문다.
- **변위 상한 ±7px/±4px.** 그 위로는 병이 "굴절"을 멈추고 "녹기"
  시작한다. 상한은 튜닝 제안이 아니라 하드 리밋이고, 비싼 것과 싸구려
  눈속임의 경계선이 바로 여기다.
- **공유 주기 없음.** 노이즈 필드는 두 축에서 0.22와 0.17셀/초로
  흐르고, 유휴 리사주는 서로 무관한 진동수 셋으로 돌고, 힌트는 4.6초로
  맥동한다 — 무엇도 눈에 띄게 맞물려 돌지 않는다.

## 성능 노트

- rAF 루프는 애니메이션 계열마다 하나씩, 약 64fps로
  스로틀(`FRAME_MS = 15.5`)하고 `dt`는 48ms로 클램프해서 백그라운드 탭에서
  돌아와도 휘청하지 않는다. 두 루프(호흡, 굴절) 모두
  IntersectionObserver로 화면 밖에서 멈추고, `visibilitychange`에서도
  멈춘다.
- DPR 상한 2. 판의 백킹 스토어는 최대 800×1000 디바이스 픽셀 언저리다.
  스트립 드로잉은 프레임당 약 500번의 `drawImage` 블릿이고 픽셀 단위
  자바스크립트는 0이다 — 노이즈 LUT는 픽셀마다가 아니라 스트립마다,
  약 500번 읽힌다.
- 병 SVG의 래스터화는 리사이즈마다 정확히 한 번(ResizeObserver),
  프레임마다는 절대 아니다. 오프스크린 캔버스 둘은 컴포넌트가 사는 동안
  재사용된다.
- CSS가 움직이는 것은 전부 `transform`, `opacity`, `clip-path`다. 호흡은
  한 요소에 커스텀 프로퍼티 하나를 프레임당 한 번 쓰고, 리빌 옵저버는
  클래스를 직접 쓴다 — 스크롤마다 리액트 리렌더는 없다.

## 접근성 노트

- **모션 최소화:** `usePrefersReducedMotion`(matchMedia 위의
  useSyncExternalStore)이 워드마크를 완전히 열린 상태(CSS 기본값)에
  두고, 굴절은 보기 좋게 고른 `t = 4200`의 정지 프레임 한 장으로만
  그리며(루프도 리스너도 없음), 미디어 쿼리가 모든 리빌 전환과 힌트
  맥동, 판의 페이드를 제거한다. 페이지는 자기 자신의 잘 나온 사진이 된다.
- **JS 없이:** 리빌 이전 상태는 전부 마운트 때 붙는 `.ondo-js` 뒤에만
  존재한다. `--ondo-breath: 1`이 기본값이라 워드마크의 무JS 상태는 완성된
  공기 든 상태이고, 병과 카피와 카드 전부가 그냥 DOM 콘텐츠다.
- **시맨틱:** 워드마크는 `aria-label="ONDO"`를 단 h1 아래의 aria-hidden
  스팬들이고, 병 SVG는 한·영 설명이 붙은 `role="img"`이며, 견본은 장식으로
  숨겨진다. 레일은 레이블 있는 포커스 가능한 `region`이다 — 방향키가
  카드 하나씩, Home/End가 양 끝으로, 휠과 드래그가 통하고, 터치는
  네이티브로 스크롤되며, 키보드의 부드러운 스크롤은 모션 최소화에서 즉시
  이동으로 바뀐다.
- **대비:** 크림 15.3:1, 샴페인 10.7:1, 샴페인 딤 6.1:1 — 기능하는
  텍스트는 전부 AA 이상. 베르메유(4.0:1)는 디스플레이 숫자보다 작은
  것은 절대 나르지 않는다.

## 리믹스 아이디어 셋

1. **위스키 하우스.** 무대와 유리판은 그대로 두고 SVG를 디캔터로 다시
   그린 뒤, 컬렉션의 이름을 온도 대신 캐스크 연수로 바꾼다. 눈금선은
   숙성 연도 표기가 되고 — 굴절은 이미 두꺼운 바 글라스로 읽힌다.
2. **온천 료칸.** 팔레트가 김 쪽으로 이동한다. 누아르 → 깊은 슬레이트,
   샴페인 → 달빛 흰색. 판의 노이즈 필드는 느려지되 *아래쪽*에서만 진폭을
   얻는다 — 목욕탕 간판 앞으로 피어오르는 안개 — 그리고 컬렉션은 탕을
   수온으로 나열한다.
3. **유리공방.** 트릭을 뒤집는다. SVG는 베르메유로 달아오른 *공작물*이고,
   포인터 에너지가 변위와 함께 판의 `filter: brightness()`를 올린다 —
   시선이 닿는 곳의 유리가 물러진다. 가마에서 막 나온 것처럼.
