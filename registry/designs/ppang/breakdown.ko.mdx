# 빵! — 제작기

## 훅: 빵집이 아니라 한 편의 새벽

은평구 골목에 새벽 4시에 문을 여는 빵집이 있다. 안 해도 될 일을
하는 게 아니라, 주인의 할머니가 그 시간에 빵을 구웠고, ‘빵이
깨어나는 시간’이 대를 이어 물려진 것이다. 이 페이지는 빵을 팔지
않는다. 그 새벽 한 시간에 관한 *웹툰 한 화*다. 스크롤은 네이버
웹툰을 넘기는 손가락과 같다. 아직 어두운 하늘 아래 빵집 전경,
말풍선으로 소개되는 제빵사, 이름이 불리는 시그니처 빵, 넓은 화면으로
도는 부엌, 그리고 새벽에 어디서 이 빵집을 만날 수 있는지 적어 둔
콜로폰. 새벽 자체가 한 명의 인물이다 — 히어로 뒤의 하늘이 읽는
동안 감청색에서 금빛으로 오르고, 부엌에 닿을 즈음이면 아침이
도착해 있다.

이 디자인의 도박은 하나다. 페이지는 가게가 되지 않고도 **어떤 장소에
관한 이야기**가 될 수 있고, 생성 일러스트는 모든 패널이 한 편의 한
컷일 때 — 패널마다 드러나고, 웹툰처럼 자막이 붙을 때 — 비로소 자리를
얻는다는 것. 양산형 카드 격자가 아니라.

## 아트 디렉션

웹툰은 온기와 호흡으로 산다. 바탕은 크림빛 종이(`#faf6ee`), 목소리는
따뜻한 갈색 먹(`#3d2817`, 12.0:1), 따뜻한 색은 정확히 하나 —
살구 `#e89b4c` — 그리고 그 색은 새벽 빛이 있는 자리에만 나타난다.
간판의 ‘OPEN’ 불, 효과음 도장, 포커스 링. 두 번째 따뜻한 색이
의미를 갖는 순간 이 페이지는 빵집 메뉴판이 된다.

어려운 일은 제약 두 개가 대신 해 줬다. 첫째, 강조색인 살구는
크림 위에서 3.0:1밖에 나오지 않는다. 그래서 **크림 위에서는 절대
기능하는 텍스트가 되지 못한다** — 강조색이 말을 해야 하는 자리마다
살구를 걷어 `amber-deep #a85f1c`(4.8:1, AA)로 낮추었다. 섹션 아이브로,
시간 표의 숫자, 포커스 링, `::selection` 바탕. 살구는 여전히
표면이고 큰 디스플레이 색이다. 둘째, 히어로 텍스트는 어두운
스크림(웹툰의 자막띠) 위에 얹었다 — 세 단계의 하늘 그라디언트 중
어떤 것이 보이든 밝은 텍스트가 AA를 지킨다. 텍스트 색을 보간하는
것보다 견고하고, 웹툰이 패널에 자막을 넣는 방식 그대로다.

서체는 뻔한 짝을 거절했다. 블랙 한 산스는 한국어 디스플레이
목소리 — 워드마크 빵!, 패널 제목, 효과과 레터링 — 크고 따뜻하게,
한 화의 표지처럼. 본문의 모든 한글 글리프는 노토 세리프 KR의
따뜻한 세리프가 맡아, 나레이션이 메뉴가 아니라 편지처럼 읽힌다.
프론세스는 영어 목소리, 이탤릭, 한국어 짝의 0.62em을 넘지 않는다 —
두 언어는 자세로 구분된다. 영어는 기울고, 한글은 선다. 스택은
언제나 한국어 서체가 먼저 와, 한글이 기본 산스로 떨어지는 일이
없다.

## 기법 1 — 열리는 패널 (clip-path-reveal)

이미지를 드러내는 단순한 방법은 페이드다. 그건 슬라이드쇼처럼
읽힌다. 웹툰의 한 컷은 페이드인 되지 않는다. *도착한다*, 종이 위에
새 컷이 얹히듯. 그래서 모든 일러스트는 `clip-path` 와이프로 드러나고,
세 패널은 각기 다른 방향을 써서 컷이 반복되지 않는다.

```css
/* styles.css — 방향 셋, 패널당 하나 */
.ppang-js [data-reveal="panel"]:not(.is-visible) .ppang-panel__mount {
  clip-path: inset(0 100% 0 0);       /* 제빵사: 좌 → 우 */
}
.ppang-product [data-reveal="panel"]:not(.is-visible) .ppang-panel__mount {
  clip-path: inset(48% 48% 48% 48%);  /* 시그니처: 중심 → 밖 */
}
.ppang-kitchen [data-reveal="panel"]:not(.is-visible) .ppang-panel__mount {
  clip-path: inset(0 0 100% 0);       /* 부엌: 상 → 하 */
}
```

와이프는 일러스트가 아니라 패널의 종이 *마운트*에 걸려 있다 — 이미지
자체는 옮겨지지 않고 풀리기만 해서 레이아웃 시프트도, 일러스트의
재래스터화도 없다. 트랜지션은 `ease-page`(`cubic-bezier(0.16,1,0.3,1)`)
위 1100ms, 이동량을 앞에서 몰아 쓰고 꼬리에서 아픽셀로 가라앉는
이징이라 와이프가 느릿이가 아니라 *의도적*으로 읽힌다. `clip-path`는
컴포지터 친화적(레이아웃을 건드리지 않음)이고, `is-visible` 클래스는
`useReveal`의 `IntersectionObserver` 하나가 넘겨 준다 — 페이지 전체에
옵저버 하나, 패널마다 JS 없다.

마크업은 패널이 풀린 채로 배송된다(`:not(.is-visible)` 규칙은
`.ppang-js` 아래에서만 적용). 그래서 자바스크립트가 꺼져도 모든
일러스트는 그냥 거기 있다. 리듀스 모션에서는 미디어 쿼리가 모든 패널
마운트에 `clip-path: none`을 강제한다 — 한 편의 인쇄된 만화, 일시정지된
애니메이션이 아니다.

## 기법 2 — 소리 내어 읽는 나레이터 (char-split-reveal)

한꺼번에 나타나는 자막은 라벨로 읽힌다. 글리프 단위로 도착하는
자막은 *누군가 말하는 것*처럼 읽힌다. 페이지의 모든 나레이션 —
제빵사의 말풍선, 부엌 자막, 상품명 — 은 한 글자 / 한 음절 단위로
쪼개지고, 진짜 문장은 보조 기술을 위해 온전히 보존된다.

```tsx
/* Narration.tsx — AT엔 진짜 문장, 눈엔 aria-hidden 글리프 */
<span className="ppang-sr">{text}</span>
<span aria-hidden="true" className="ppang-narration__row">
  {glyphs.map((g, i) => (
    <span className="ppang-narration__g"
      style={{ "--pn": `${start + i * step}ms` } as CSSProperties}>
      {g === " " ? "\u00A0" : g}
    </span>
  ))}
</span>
```

각 글리프는 지연을 커스텀 프로퍼티(`--pn`)로 갖고, 리빌 순간에
자바스크립트가 하는 일이라고는 패널에 클래스 하나를 더하는 것뿐 —
옵저버는 글리프를 건드리지 않는다. 거기서부터는 CSS가 안무를
몬다. `transition-delay: var(--pn)`이 글리프를 엇갈리게(나레이션은
38ms, 상품명은 90ms)하고, 리빌 전 상태(`opacity: 0`, 0.5em 낙하)는
`.ppang-js` 루트 아래에서, 그리고 *가장 가까운 패널이 아직
is-visible이 아닐 때만* 존재한다. 그러니 어떤 나레이션도 제 패널이
열리기 전에는 말을 시작하지 않는다 — 자막이 그림을 따른다, 웹툰이
그러하듯.

쪼개기는 `Array.from`로 해서 한글 음절과 자모 묶음이 한 코드 포인트로
살고(음절이 스팬을 가로질러 쪼개지지 않는다), 빈칸은 줄 끝에서 무너지지
않도록 no-break space가 된다. 영어 줄은 한국어보다 반 박자 늦게, 자기
`--pn` 지연으로 도착한다 — 기대어 드는 두 번째 목소리.

## 기법 3 — 디오라마 (pointer-parallax + 스크롤 깊이)

히어로는 평면 프레임이 아니라 디오라마다. 네 레이어가 스크롤하며
각기 다른 속도로 흐르고(웹툰의 깊이), 정밀 포인터에서는 커서를
따라 기운다.

```css
/* styles.css — 레이어마다 깊이 배수가 다르다 */
.ppang-hero__far   { transform: translate3d(calc(var(--ppang-px) * -6px),
                       calc(var(--ppang-py) * -4px + var(--ppang-sky-y) * 0.4), 0); }
.ppang-hero__art   { transform: translate3d(calc(var(--ppang-px) * 10px),
                       calc(var(--ppang-py) * 6px + var(--ppang-sky-y) * 0.7), 0); }
.ppang-hero__sign  { transform: translate3d(calc(var(--ppang-px) * 18px),
                       calc(var(--ppang-py) * 12px + var(--ppang-sky-y) * 1.1), 0); }
```

먼 언덕이 가장 느리고, 매달린 간판이 가장 빠르다 — 착각을 파는
깊이다. 상한은 레이어 바로 옆, CSS에 있다(포인터 ±6 / ±10 / ±18 px,
스크롤 60px). 변수 두 개를 공급하는 훅이 있다. `usePointerParallax`는
커서로부터 `--ppang-px/py ∈ [-1,1]`을 쓴다(lerp 0.08/frame, 수렴하고
포인터가 떠나면 스스로 멈춘다). `useScrollProgress`는 히어로가 뷰포트에서
얼마나 벗어났는지로 `--ppang-sky-y`를 쓴다. 둘 다 transform 전용 —
페이지 전체에서 프레임마다 쓰이는 건 루트의 CSS 커스텀 프로퍼티 네
개뿐, 모두 `transform`이 소비한다. 터치에서는 포인터 훅이 스스로
꺼지고(디오라마는 가만히), 스크롤 패럴랙스는 여전히 돈다. 리듀스
모션에서는 네 변수 모두 0으로 고정되고 디오라마는 한 점의 정물이 된다.

간판은 인라인 SVG(불러온 디스플레이 서체로 된 진짜 한글 빵!, 숨 쉬는
작은 OPEN 불)로 처마 위에 칠해진다 — 일러스트는 설계상 텍스트를 갖지
않으므로, 실제 간판은 어떤 해상도에서도 날카운 벡터이며 접근성도
완전하다(`role="img"`, 이중 언어 라벨). 6.2초 주기로 ±1.1° 흔들리고,
불은 5.3초 주기로 숨 쉰다 — 둘은 공약수가 없어 아이들 상태의 히어로가
눈에 띄게 루프하지 않는다.

## 스크롤이 치는 새벽

시그니처 배경: 페이지의 처음 55%를 스크롤하는 동안 히어로 뒤의 하늘이
감청색 새벽에서 분홍빛 여명을 지해 아침의 금빛으로 오른다.
그라디언트 스톱을 애니메이션(비싸고 흔들리는)하는 대신, 하늘은
**불투명도가 크로스페이드인 세 개의 겹친 그라디언트 요소**다.

```css
/* styles.css — A단계(감청)는 dawn이 0.5를 넘으면 빠지고 */
.ppang-hero__sky::before {
  background: linear-gradient(180deg, #2a2547, var(--ppang-dawn-indigo), …);
  opacity: calc(1 - clamp(var(--ppang-dawn) * 2, 0, 1));
}
/* C단계(금빛)는 dawn이 0.5를 넘으면 들어온다 */
.ppang-hero__gold {
  background: linear-gradient(180deg, #c99a6a, #e9bf7e, …);
  opacity: clamp((var(--ppang-dawn) - 0.5) * 2, 0, 1);
}
```

`--ppang-dawn`은 `[0,1]` 안의 숫자 하나, `useScrollProgress`가
`scrollY / maxScroll`로부터 rAF마다 한 번씩 쓴다(페이지의 처음 55%로
제한 — 부엌에 닿을 때쯤이면 아침은 도착해 있고 고정된다). 분홍빛
단계는 새벽의 한가운데서 정점을 찍는데, `1 - max(dawn - 0.5,
0.5 - dawn) * 2`로 계산한다 — `max()/min()`으로 만든 삼각 정점으로,
같은 결과를 주면서도 `abs()`(비교적 새롭고 지원이 고르지 않은)를
의도적으로 피했다. JS가 꺼지면 `--ppang-dawn`은 선언된 기본값 0에
머문다 — 히어로는 온전한 감청색 새벽으로 열리고, 이것이 이야기의
올바른 첫 박자다. 리듀스 모션에서는 1로 고정 — 아침이 도착해 있다.

## 생성 일러스트

네 패널은 ComfyUI로 생성했다. Illustrious-XL 병합
`zenijiMixKWebtoon_v10`에 `toonystarkKoreanWebtoonFlux` LoRA를 0.75로,
832×1216 / 1024×1024 / 1024×768, 30스텝, CFG 6.5, Euler a, 각각
RealESRGAN으로 2배 업스케일. 전체 레시피는 `image-recipe.md`,
내보낸 그래프는 `workflows/`에 있다.

두 결정이 중요하다. 첫째, **어떤 이미지도 텍스트를 갖지 않는다** —
프롬프트와 네거티브 프롬프트로 — 그래서 빵집 이름, OPEN 불, 효과과(쫀득!),
모든 자막은 진짜 HTML/SVG/CSS다. 한글은 어떤 해상도에서도 날카운
벡터로 남고, 스크린 리더는 진짜 텍스트를 받으며, 이미지는 라벨 하나
다시 치지 않고 재생성(또는 감사 통과 체크포인트로 교체)할 수 있다.
둘째, 라이선스 감사: 체크포인트는 `docs/model-licenses.md`에서
**아직 감사되지 않았고**, 이는 유지보수자가 감사 행을 공동 서명하기
전까지 G4 게이트 차단 요소다. 이 엔트리는 그 감사를 위해 준비된
상태고, 체크포인트를 교체하면 `/media/ppang/`의 파일 넷만 재생성하면
된다.

## 모션 수학

- **글리프 케이던스 38ms.** 나레이터의 읽기 속도 — 타자기(약 30ms)보다
  느리고, 말처럼 느껴질 만큼은 빠르다. 상품명은 90ms를 쓰는데, 이것은
  나레이션이 아니라 *제목*이기 때문이다.
- **포인터 lerp 0.08/frame**(60fps에서 반감기 ≈ 8프레임). 디오라마는
  커서를 깜빡임 안에 알아채고 가라앉는 데는 순간이 걸린다 — 복종이
  아니라 흐름. `dt`로 계산해 프레임 독립적이다.
- **패널 와이프 1100ms, 자막 800ms에 +350ms.** 자막은 패널이 의미 있게
  열린 뒤에才开始시작하므로, 눈은 먼저 그림을 읽고 다음에 글자를
  읽는다 — 웹툰이 강제하는 순서 그대로.
- 두 앰비언트 주기(흔들림 6.2초, 불 5.3초)와 힌트 드리프트(4.6초)는
  서로 소수 관계라 아이들 상태의 히어로가 눈에 띄게 루프하지 않는다.

## 성능

- 페이지의 모든 리빌에 `IntersectionObserver` 하나(`useReveal`),
  임계값 0.16 — 요소마다 옵저버도 없고, 스크롤에 리액트 리렌더도
  없다. 옵저버는 `class`와 `transitionDelay`만 쓴다.
- `useScrollProgress`는 rAF 하나, 스크롤/리사이즈/가시성에서만
  발화하도록 제한, 커스텀 프로퍼티 둘을 쓴다; `visibilitychange`(탭
  숨김)에서는 완전히 멈춘다. `dt`는 48ms로 제한.
- `usePointerParallax`는 *수렴하는 동안에만* rAF를 돌린다 — lerp가
  수렴하고 포인터가 떠나면 스스로 주차해, 아이들 히어로는 프레임을
  하나도 쓰지 않는다.
- 모든 애니메이션은 `transform`, `opacity`, `filter`, 또는
  `clip-path`. 프레임마다 쓰이는 CSS 변수 넷은 모두 `transform`이
  소비한다. 레이아웃 프로퍼티 애니메이션은 어디에도 없다.
- 이미지: AVIF `<source>` + WebP `<img>` 폴백, 치수 예약(CLS 없음),
  `loading="lazy"` + `decoding="async"`, 히어로만 예외(`eager`/`sync`,
  첫 페인트). 총 페이로드 AVIF 약 582 KB — 5120 KB 예산을 크게
  밑돈다.

## 접근성

- **리듀스 모션:** `--ppang-dawn` 1로 고정(아침 도착), 패럴랙스 변수
  모두 0, 앰비언트 주기 전부 사망, 모든 리빌 전 상태를 최종 구도로
  강제(패널 `clip-path: none`, 자막 `opacity: 1`, 글리프 가시). 페이지는
  인쇄된 한 편으로 읽힌다.
- **No-JS:** 모든 리빌 전 스타일은 마운트 시 더해지는 `.ppang-js`
  뒤에 게이트. SSR 페이지는 완성된 한 편 — 모든 일러스트와 모든 줄이
  보인다.
- **시맨틱:** 히어로 워드마크는 `aria-label`을 가진 `h1`; 스플릿
  텍스트는 시각적으로 숨겨진 원본을 보존; SVG 간판은 `role="img"`에
  이중 언어 라벨; 장식 요소(먼 언덕, 불, 힌트, 효과음)는
  `aria-hidden`.
- **키보드와 터치:** 두 대화형 요소(주소 링크, 처음으로 링크)는
  `:focus-visible`에 2px amber-deep 링. 패럴랙스는 정밀하지 않은
  포인터에서 스스로 꺼진다; 새벽은 여전히 스크롤에 맞춰 오른다.
- **대비:** 먹 12.0:1, ink-soft 6.6:1, amber-deep 4.8:1 (크림 위);
  히어로 텍스트는 어두운 스크림 위에서 새벽의 모든 단계에서 AA를
  지난다. 살구는 크림 위에서 결코 기능하는 텍스트가 아니다.

## 리믹스 아이디어

1. **심야 라멘 카운터.** 패널 와이프 문법과 스크롤 하늘(일몰 → 밤 →
   수증기)을 그대로 두고 빵집을 6석 카운터로 바꾼다. 말풍선은 주인의
   두 줄 인사, 효과음은 면 삶는 소리로.
2. **닫아 올리는 책방.** 새벽을 뒤집는다: 스크롤하면 하늘이 황혼에서
   밤으로 *내려*오고, 패널은 열리는 대신 닫힌다(`inset(0)` →
   `inset(0 100% 0 0)`), 콜로폰은 그날 반려된 책의 목록이 된다.
3. **웹툰 그 자체.** 패널을 진짜 세로 컷 시퀀스(8–12컷)로 승격시키고,
   스크롤이 와이프 진행을 직접 *스크럽*하게 하고(`clip-path inset`을
   관측자 일회성이 아니라 스크롤 진행에 묶기), 패널마다 스톰프
   효과음 인터랙션을 더한다.
