# 여백 — 제작기

## 훅: 스스로를 증명해야 하는 잡지

한 문예 계간지가 일곱 번째 호를 여백에 바쳤다. 그리고 이 호는 그 주제를
끝까지 밀고 간다. 표지는 거의 비어 있고, 권두 에세이는 자신이 조판된
바로 그 빈 공간에 대해 말하고, 화보는 거의 아무것도 없는 판 세 장을
걸고, 마지막 한 장은 일부러 비워 두었다. 페이지는 스스로를 한 번도
설명하지 않는다. 종이처럼 굴고, 에세이가 말하는 방식 그대로 —
독자를 믿는다.

이 디자인의 도박은 하나다. 화려한 모션으로 가득한 갤러리 안에서, 가장
조용한 엔트리 — 유일한 스펙터클이 타이포그래피의 리듬과 '없음'의 의도적
사용뿐인 페이지 — 가 제 자리를 지켜낼 수 있다는 것. 여기서는 여백이 곧
디자인이다.

## 아트 디렉션

모든 일은 아이보리(`#faf6ef`) 위에서 일어난다. 말하는 것은 먹색
잉크(`#232020`, 15:1)이고, 옥스블러드(`#7d2a26`, 8.7:1)는 단 하나의
강조색으로서 작고 값비싼 것들로만 나타난다 — 헤어라인, 두문자, 각주
번호, 낙관 하나, 밑줄 드로잉, 포커스 링. 브리프의 그래파이트
(`#8f8a82`)는 아이보리 위에서 약 3.2:1 — 읽기에는 모자란 수치라서 괘선
전용으로 강등시켰다. 보조 텍스트(영문 번역, 캡션, 각주)는 파생 토큰이
맡는다. 잉크 72% 알파, 합성하면 대략 `#5f5c5a`, 대비 6.2:1.

세리프 둘이 배역을 나눈다. 나눔명조는 페이지의 모든 한글을 맡는다 —
제호는 800 웨이트로, 본문은 400에 행간 2.05와 `word-break: keep-all`로.
명조는 숨 쉴 공간이 필요하고, 한국어는 단어 중간에서 꺾이면 안 되기
때문이다. Libre Caslon Text는 로마자로 응답한다. 번역문은 0.95em의
연한 잉크로, 폴리오와 캡션은 이탤릭으로, 그리고 두문자 하나로. 폰트
스택 순서도 조용히 일한다 — Caslon을 앞에, 명조를 뒤에 두면 마크업 없이
흘러든 한글도 로마자 서체를 통과해 한국어 세리프에 도착한다.

## 기법 — 자리를 지키는 도판 (scroll-scrub-pinned)

에세이의 시그니처는 접지 도판이다. 본문 문단들이 곁을 흘러가는 동안
도판은 자리를 지키고, 도판의 먹선은 독자의 스크롤이 그린다. 고정
자체는 공짜다 — 스프레드의 전체 높이를 차지하는 도판 칼럼 안의
`position: sticky`:

```css
.yeobaek-spread__fig .yeobaek-figure {
  position: sticky;
  top: clamp(4.5rem, 14vh, 8rem);
}
```

스크럽은 스프레드마다 하나씩 도는 작은 rAF 루프다. 매 프레임 정확히 한
번 읽고(스프레드의 rect) 한 번 쓴다(커스텀 프로퍼티) — 읽기가 쓰기보다
먼저이므로 레이아웃 스래싱이 없다. 원시 진행도는 dt로 정규화한 lerp로
부드럽게 만들어, 120Hz 디스플레이가 60Hz보다 두 배 빠르게 긋는 일이
없도록 했다:

```ts
// useScrollScrub.ts — 한 번 읽고, 한 번 섞고, 한 번 쓴다
const span = Math.max(Math.min(rect.height * 0.85, vh * 1.1), 1);
const raw = (vh * 0.7 - rect.top) / span;
const target = Math.min(1, Math.max(0, raw));
value += (target - value) * (1 - Math.pow(0.86, dt / 16.7));
el.style.setProperty("--yb-scrub", value.toFixed(4));
```

IntersectionObserver가 루프를 켜고 끄기 때문에 화면 밖의 스프레드는
프레임 비용이 0이다. `vh * 1.1` 상한은 모바일에서 중요하다. 단일 칼럼
스프레드는 세로로 길어지는데, 상한이 없으면 도판이 화면을 떠난 지 한참
뒤에도 획이 그려지고 있게 된다.

`prefers-reduced-motion`에서는 훅이 `--yb-scrub: 1`을 한 번 쓰고,
미디어 쿼리가 도판을 `position: static`으로 되돌린다 — 완성된 채로
정적으로 붙어 있는 도판. 자바스크립트가 없어도 CSS 폴백이 같은 일을
한다.

## 기법 — 스스로 그려지는 먹 (svg-line-draw)

페이지의 모든 획은 `pathLength=1`에 `stroke-dasharray: 1 1`을 쓴
패스다. 이렇게 하면 dashoffset 계산이 전부 암산 가능한 숫자가 된다.
에세이 도판은 스크럽을 그대로 매핑한다:

```css
.yeobaek-stroke {
  stroke-dasharray: 1 1;
  stroke-dashoffset: calc(1 - var(--yb-scrub, 1));
}

/* 낙관은 도판이 거의 완성될 때만 도착한다: 0.833→1 구간이 0→1로
   매핑되고, 브라우저가 opacity를 [0,1]로 공짜로 클램프해 준다 */
.yeobaek-seal {
  opacity: calc(var(--yb-scrub, 1) * 6 - 5);
}
```

`--yb-scrub`의 기본값이 루트에서 1이기 때문에, JS 없음과 모션 최소화
상태는 무언가를 숨기는 폴백이 아니다 — 그 자체로 완성된 도판이다.

화보의 파문은 리빌 임계점에서 시간 기반으로 한 번만 그려지고(1.5초에
걸쳐 dashoffset 1 → 0, 타원 세 겹이 250ms씩 어긋나게), 링크에는 고무줄
밑줄이 걸린다. 항상 존재하는 그래파이트 헤어라인 위로, 손맛이 살짝 밴
옥스블러드 SVG 패스가 호버나 포커스에서 오버슈트 베지어를 타고
그려진다:

```css
.yeobaek-link:hover .yeobaek-link__ink path,
.yeobaek-link:focus-visible .yeobaek-link__ink path {
  stroke-dashoffset: 0;
  transition: stroke-dashoffset 640ms cubic-bezier(0.32, 1.42, 0.5, 1);
}
```

베지어가 약 6% 오버슈트해서, 선이 팽팽함을 지나쳤다가 되돌아온다 —
그것이 고무줄이다. 순수 CSS라서 자바스크립트가 꺼져 있어도 동작하고,
모션 최소화에서는 획이 그냥 그려진 채로 존재한다.

## 기법 — 두 글자로 된 제호 (char-split-reveal)

제호 여백은 글자 두 개이므로, 분할 비용은 span 두 개다. `h1`은
`aria-label`에 단어 전체를 보관하고, 움직이는 글자들은 `aria-hidden`으로
숨긴 채 각자 자기 순번을 커스텀 프로퍼티로 갖는다:

```tsx
<h1 className="yeobaek-name" lang="ko" aria-label="여백">
  {NAME.map((ch, i) => (
    <span aria-hidden="true" className="yeobaek-name__ch"
      style={{ "--yb-ci": i } as CSSProperties}>{ch}</span>
  ))}
</h1>
```

CSS가 `calc(120ms + var(--yb-ci) * 190ms)`로 상승을 어긋나게 하고, 이
모든 것은 `.yeobaek-js` 루트 클래스 뒤에 게이트되어 서버가 렌더링한
표지는 처음부터 완성형이다. 두문자는 일부러 이 기법을 쓰지 않았다 —
CSS `::first-letter`라서 span도 없고, 보조 기술을 위해 재배열되는 것도
없다. 분할은 정직하고 저렴한 곳에서만, 나머지는 플랫폼이 이미 주는
것으로.

## 여백에서 걸어 나오는 각주

네 개의 번호 달린 각주가 15rem 여백 레일에서 에세이에 주석을 단다.
각각은 진짜 양방향 앵커 쌍이다 — 본문의 어깨번호가 각주로, 각주의
돌아가기 화살표가 본문으로 연결되어 키보드와 스크린 리더 사용자가 양
방향으로 오갈 수 있다. 등장은 리빌 옵저버를 재사용하되 가로 방향의
사전 상태를 준다:

```css
.yeobaek-js [data-reveal="margin"] { transform: translateX(22px); }
.yeobaek-js [data-reveal].is-visible { opacity: 1; transform: none; }
```

임계점에 닿으면 각주는 여백 쪽에서 미끄러져 들어온다 — 빈 곳에서
걸어 나오는 것이다. 페이지 전체가 하고 있는 농담과 같은 농담이다.
1080px 아래에서는 레일이 접히고 각주는 왼쪽 괘선을 두른 채 해당 문단
아래에 앉는다. 호버 전용 정보는 없고, 터치에서 잃는 것도 없다.

## 판형의 규율

조용한 일은 칼럼 수학이다. 에세이 본문은 36rem에서 멈춘다 — 로마자
기준 약 예순다섯 자, 각주 4가 지면에서 스스로 고백하는 그 수치다.
한국어 문단은 행간 2.05, 로마자는 1.78로 서로 다른 속도로 숨 쉬지만,
한 쌍이 끝날 때마다 같은 베이스라인 리듬으로 돌아온다. 여기서
'다단'은 CSS 칼럼이 아니라 편집 그리드를 뜻한다 — 본문 칼럼, 여백
레일, 도판 칼럼. 에세이 전체를 CSS 칼럼에 흘렸다면 고정 도판이
죽었을 것이다(단편화 컨텍스트와 sticky는 함께 살 수 없다). 진짜
`column-count: 2`는 정확히 한 번, 코다에서 등장한다. 옅은 단 사이
괘선과 함께 — 에세이는 잡지 자신의 몸짓으로 끝난다.

## 접근성 노트

- **모션 최소화:** 도판은 고정이 풀리고(`position: static`) 모든 획은
  완전히 그려진 채이며(`--yb-scrub`은 1에 머문다), 각주와 리빌은
  움직임 없이 존재하고, 밑줄은 드로잉 없이 존재하고, 표지는 안무를
  건너뛴다. 정지된, 완성된 한 권.
- **JS 없음:** 모든 사전 리빌 상태는 마운트 시 추가되는 `.yeobaek-js`
  뒤에 게이트된다. 자바스크립트가 꺼지면 페이지는 깨끗하게 인쇄된
  문서다 — 잡지가 강등되어야 할 바로 그 상태.
- **시맨틱:** 도판은 진짜 `figure`/`figcaption` 안의 `role="img"`에
  이중언어 레이블, 분할된 제호는 `aria-label` 뒤에 span을 숨기고,
  각주는 `role="note"`에 레이블 달린 돌아가기 링크, 포커스는 2px
  옥스블러드 `:focus-visible` 링.
- **대비:** 잉크 15:1, 옥스블러드 8.7:1, 보조 6.2:1 — 모두 AA 이상.
  3.2:1의 그래파이트는 단 한 단어도 나르지 않는다.

## 성능 노트

- rAF 루프는 두 개(스프레드당 하나)뿐이고, 각자 자기 스프레드가 넉넉한
  옵저버 마진과 교차하는 동안만 산다. 한 프레임의 비용은
  `getBoundingClientRect` 한 번과 커스텀 프로퍼티 쓰기 한 번.
- 움직이는 모든 것은 `transform`, `opacity`, `stroke-dashoffset`이다.
  고정은 네이티브 sticky이지, 스크립트로 미는 레이아웃이 아니다.
- 리빌은 IntersectionObserver 하나가 `class`와 `transitionDelay`를
  명령형으로 쓴다 — 페이지 어디에서도 스크롤이 React 리렌더를
  일으키지 않는다.
- 이미지 0, 의존성 0. 가장 무거운 자산은 서체 두 벌이고, 한글 서브셋은
  next/font에 맡겼다.

## 리믹스 아이디어

1. **시집.** 그리드는 그대로, 에세이 자리에 시를 넣는다. 연은 Block이
   되고, 시마다 삽화 한 점이 고정된다. 스크럽은 제목 아래 밑줄을
   긋는다 — 끝까지 읽은 사람에게만 완성되는 밑줄.
2. **미술관 월텍스트 시스템.** 도판은 오브제 라벨이 되고, 여백 레일은
   큐레이터 노트를 싣고, 옥스블러드는 소장품 등록번호의 빨강이 된다.
   비움의 화랑이 실제 도록이 되는 것.
3. **활판 스페시먼.** 뼈대는 같지만 도판은 활자 견본이 되고, 스크럽은
   붓질 대신 거대한 앰퍼샌드에 먹을 올린다. 판권 원장에는 포인트
   크기가 자라고, 코다는 자신이 설명하는 바로 그 서체로 조판된다.
