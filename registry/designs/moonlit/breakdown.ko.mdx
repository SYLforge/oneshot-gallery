# 달빛 — 제작기

## 훅: 잠든 도시의, 한 건의 주문

어딘가의 도시가 새벽 두 시이고, 누군가는 여전히 잠들지 못했다 — 그리고
또 다른 누군가는 오토바이를 타고 그 사람에게 저녁을 가져오고 있다.
달빛은 오직 그 여섯 시간에만 존재하는 브랜드이고, 이 페이지는 한 건의
주문이 곧 한 화다: 01:47의 접수에서 02:23의 완료까지. 방문자는
고객이 아니라 독자이고, 페이지는 웹툰의 문법을 스크롤 자체가
페이지 넘김으로 읽히게끔 밀어붙인다.

이 디자인의 도박은 하나다. 배달 트래커는 이미 이야기라는 것. 상태
업데이트에는 이미 막이 있다 — 주문, 주방, 도로, 문. 웹툰의 패널 호흡과
배달 앱의 트래커는 거의 같은 리듬을 원한다는 걸, 페이지는 그것을
명시적으로 만들고, 한 발 더 나아가 하나의 질문을 던진다 — 라이더의
밤은 과연 어떤 느낌인가? — 그리고 그것을 감청색과 네온, 그리고 단
한 점의 따뜻함으로 대답한다. 따뜻함은 오직 한 끼가 손에 건네지는
그 순간에만.

## 아트 디렉션

이 엔트리는 빵! (PPANG!) 의 차가운 쌍둥이다. 빵!이 따뜻한 새벽의
빵집(주황, 빵, 아침 빛)으로서 웹툰이 가장 따뜥할 수 있는 모범이라면,
달빛은 일부러 정반대를 한다. 깊은 감청색의 밤 `#0b1020`, 네온 블루
`#4da8ff` 를 유일한 차가운 액센트로, 그리고 새벽 전달의 순간에만
허락된 호박색 `#e89b4c` 하나. 팔레트를 붙잡는 규칙: 어느 순간 화면 위의
호박색을 세어 보라. 0이거나, 아니면 전달 장면 전체여야 한다. 그 사이는
전부 틀렸다 — 차가운 UI를 꾸미는 약간의 따뜻함이 바로 실패의 모양이다.

블루는 본문 텍스트에도 허용된다(감청색 위에서 7.5:1, AAA). 그래서
하프라이트의 빨강과 달리 블루는 일하는 UI 색이다: 활성 상태, 라이브 표시,
섹션 헤드의 밑줄. 이것이 페이지를 "웹툰일 뿐 아니라" 배달 앱으로 읽히게
만드는 부분이다. 차가운 액센트는 일해야 한다. 페이지 정체성의 절반이
"이것은 실제 서비스이고, 여기에 운영 시간이 있고, 주문 트래커가 있다"이기
때문이다.

서체 넷은 배역이 넷이다. Gugi는 스플래시 — 워드마크 `달빛` 과 전달
헤드라인, 한국어 디스플레이가 쓰이는 단 두 곳. Noto Sans KR은 산문의
목소리 — 시 줄, 상태, 라이더의 내면 독백을 포함한 모든 한국어 라인.
Space Grotesk는 영어 디스플레이 — `MOONLIT` 워드마크, 섹션 헤드, 영어
본문. Space Mono는 기계 — 모든 타임스탬프, 주문 번호, 거리, 운영 시간.
본문 스택은 Space Grotesk에서 Noto Sans KR로 이어지며 마크업 없이
한글을 받아내고, `:lang(ko)` 가 Noto Sans KR을 고정하며
`word-break: keep-all`을 건다.

## 기법 1 — 타자기: 실시간처럼 업데이트되는 배달 상태

이 페이지의 시그니처는 한국어 상태 라인이 실시간 배달 트래커가
업데이트되듯 한 글자씩 타이핑된다는 것이다. 훅은 문자열을 요청
시점에 타이핑하는 하나의 함수다:

```ts
// useTypewriter.ts — `start`가 참이 될 때 `text`를 타이핑한다
const start = seen && (inView || true);
const { shown, typing } = useTypewriter(RIDER_KO, start, 55);
```

훅은 호출 쪽에서 IntersectionObserver로 게이트된다 — `start`는 패널이
화면에 스크롤되어 들어온 뒤에만 참이 되므로, 닿지 않은 상태는
가만히 있다. 보이는 문자열은 훅이 갱신하는 `aria-hidden` 스팬에
들어 있고, 온전한 문자열은 그 옆의 시각적으로 숨겨진 스팬에 있다.
그래서 스크린 리더와 브라우저 찾기는 온전한 텍스트를 받고,
페이지는 자바스크립트 없이도 온전히 읽힌다.

커서는 네온 블루의 `▋` 블록으로, 540ms마다 `steps(1, end)` 키프레임으로
깜빡인다. 타이핑 중에는 커서가 강제로 보이고(`is-typing` 클래스),
타이핑이 끝난 뒤에도 커서는 계속 깜빡인다 — 트래커가 다음 단계를
기다리는 방식 그대로. 모션 최소화에서는 이것이 접힌다: 훅이 몇 프레임
안에 종료 상태까지 달리고(모션 최소화에서는 속도가 1 글리프/프레임으로
설정된다), 텍스트는 통째로 도착하며 커서는 미디어 쿼리로 숨겨진다.

같은 훅이 라이더 소개의 상태 라인과 네 개의 트래커 단계(주문 접수,
조리 중, 배달 출발, 배달 완료) 모두를 모는데, 트래커의 각 한국어
라벨은 해당 행이 화면에 들어올 때만 타이핑된다. 그래서 페이지는 한
번에 쏟아지는 게 아니라 네 개의 챕터 박자로 읽힌다.

## 기법 2 — SVG 선 그리기: 스크롤하며 스쳐 지나가는 네온

차가운 액센트의 시그니처는 SVG 네온 튜브다. 히어로에 하나, 거리
패널에 두 개. 각각은 `feGaussianBlur` + `feMerge` 필터로 둘러싸인
단일 SVG `<path>`로, 스트로크 자체에 후광을 입힌다("채움 없음;
스트로크는 네온-브라이트; 끝모양 둥글게"). 그리기는
`stroke-dasharray` + `stroke-dashoffset`이다:

```css
.moonlit-neon__tube {
  stroke-dasharray: 620;
  stroke-dashoffset: 620; // 꺼짐
}
.moonlit-js .moonlit-hero.is-lit .moonlit-neon__tube {
  stroke-dashoffset: 0; // 1400ms에 걸쳐 그려진다
  animation: ml-flicker 3.7s steps(1, end) 1.6s infinite;
}
```

`.is-lit` 클래스는 Hero 와 StreetSequence 컴포넌트 안의
IntersectionObserver가 더한다. 패널이 화면에 들어오기 전에는 튜브가
흐릿하다("스트로크: 스틸", 대시오프셋 풀). 스크롤해 들어오면 스트로크가
"네온-브라이트"로 밝아지고 대시오프셋이 1400ms에 걸쳐 0으로
이징된다 — 사인이 켜진다. 그리고 3.3–4.1s의 `steps(1, end)` 플리커
키프레임이 이어받는다: 대부분 안정적이고 한 주기당 두 번 빠르게
떨어지는, 저녁부터 켜져 있던 피곤한 네온 튜브의 버징.

거리의 두 사인은 그리기 타이밍을 180 / 460ms 어긋놓아 같이 켜지지
않게 한다 — 두 사인이 서로 다른 두 회로에 의해 켜진 것처럼 읽힌다.
트래커의 세로 진행 튜브도 같은 어휘를 쓴다: 길이 400짜리 패스의
대시오프셋이 `--moonlit-progress`가 0 → 1로 감에 따라 400(진행 없음)에서
0(완료)으로 스크럽된다. 활성 단계는 읽는 사람의 스크롤과 함께
단조롭게 전진한다 — 타이머도 없고 거짓도 없다 — 그래서 선은 언제나
주문이 얼마나 읽혔는지에 정직하다.

## 기법 3 — 마퀴: 텅 빈 거리 위 헤드라이트 속도선

텅 빈 거리 패널에는 사람 없는 움직임이 필요하다. 라이더는 액자
밖, 움직이는 중임이 암시된다 — 그리고 그것을 파는 것이 헤드라이트
궤적이다. 두 줄의 얇은 차가운 블루 빛 줄기가 순수 CSS 마퀴로 패널을
가로질러 쓸고 지나간다:

```css
.moonlit-street__trail-row {
  width: 200%;          // 패널 너비 두 배 분량의 줄기
  position: absolute;
}
.moonlit-js .moonlit-street.is-running .moonlit-street__trail-row--a {
  animation: ml-trail 5.4s linear infinite;
}
@keyframes ml-trail {
  from { transform: translate3d(0, 0, 0); }
  to   { transform: translate3d(-50%, 0, 0); }
}
```

각 줄기는 양 끝이 투명한 얇은 `linear-gradient` 막대(높이 1–1.5px)로,
부드러운 블루 박스섀도 후광을 더해 막대가 아니라 움직임으로 읽히게
한다. 두 줄은 5.4s와 3.6s(하나는 역방향)로 돌아 결코 겹치지 않는다.
`.is-running` 클래스는 IntersectionObserver에서 `once: false`로 설정되어
패널이 화면을 떠나면 마퀴가 멈추고, 돌아오면 다시 시작한다 — 화면 밖
rAF는 없다. 모션 최소화에서는 정지 프레임 하나로 떨어지고, JS가
없으면 줄기들은 첫 위치에 가만히 있으며 패널은 그냥 완성되어 있다.

마퀴 기법 아이디는 그라디언트 플라자(키네틱 라디오 티커에 쓰는)에서
재사용했다. 둘은 갤러리 전체 기법 분류의 같은 태그가 구체적으로는
다른 것을 의미한다는 점에 정직하다 — 태그가 뜻하는 것은 "transform로
구동되는 순환 티커"이고, 두 엔트리 모두 그것을 벌어받는다.

## 전달 — 한 뷰포트의 따뜻함

페이지의 논증은 05장에 산다. 다섯 장 동안 페이지는 감청과 블루,
고독이었고; 05장만이 따뜻해도 좋다. 그리고 오직 05장만. 섹션 전체가
오른쪽 가장자리에서 새어 들어오는 방사형 호박색 블룸을 얻고, 오파시티가
0 → 1로 1800ms에 걸쳐 오른다 — 문틈으로 새워 들어오는 새벽. 전달
이미지는 자기만의 따뜻한 후광 오버레이를 얻는다. 제목은 Gugi,
호박-브라이트: "따뜻하게, 도착했습니다."

카피는 페이지의 논증을 한 문장으로 만든다: 그가 건넨 것은 한 그릇의
밥이 아니라, 아무도 누구에게도 닿지 않는 이 시간에 닿았다는 따뜻함이었다.
도시의 다른 모든 불은 꺼졌다. 당신의 불과, 그의 헤드라이트만이 여전히
켜져 있다. 사인오프는 이중언어이고, 무심하지만 다정하다: "잘 드세요. —
좋은 밤 되세요." / "EAT WELL. — GOOD NIGHT." 그리고 06장이 차가운
UI를 가져온다: 스틸과 블루, 운영 시간, 커버리지 목록, 주문 전화.
호박은 한 챕터였다.

## 모션의 수학

- **타자기 스텝 55ms.** 여섯 글자 한국어 상태 라인은 약 330ms가
  걸린다 — 실시간으로 느껴질 만큼 빠르고, 한 글자가 눈에 박힐 만큼
  느리다. 트래커의 단계 라벨은 60ms로, 페이지의 챕터 박자이기 때문에
  살짝 더 느리다.
- **네온 그리기 1400ms ease-night.** 튜브가 켜지는 것을 볼 수 있을
  만큼 길고, 두 번째 사인의 460ms 어긋남이 지연이 아니라 두 번째
  사인으로 읽힐 만큼 짧다.
- **플리커 3.7s / 3.3s / 4.1s.** 세 주기는 공약수가 없어 어둠이 눈에
  띄게 반복되지 않는다. 플리커 키프레임은 한 주기당 두 번의 빠른
  딥(48%에서 약 50%, 79%에서 약 50%)이지 부드러운 맥박이 아니다 —
  네온은 깜빡이지, 페이드하지 않는다.
- **헤드라이트 궤적 5.4s / 3.6s.** 두 줄, 하나 역방향. 비 3:2,
  공통 박자 없음.
- **새벽 블룸 1800ms.** 페이지에서 가장 느린 모션, 의도적으로 —
  새벽은 서두르지 않는다.

## 성능 노트

- 모든 애니메이션은 transform, opacity, stroke, filter — 레이아웃은
  결코 아니다. 마퀴, 네온 그리기, 블룸, 커서 깜빡임, 펄스 점, 플리커
  전부 컴포지터 전용이다.
- 히어로 패럴랙스는 하나의 passive `scroll` 리스너가 하나의 rAF를
  예약하고, transform을 한 번 쓴다. 마운트 이후 레이아웃 읽기는 없다.
- IntersectionObserver(reveal, inView, 마퀴를 위한 once-false inView,
  트래커의 활성 단계 관찰자)는 모두 언마운트 시 해제된다. 새는 없다.
- 모든 타이핑 훅은 자기만의 rAF를 가지고 클린업에서 취소한다; 모션
  최소화에서는 훅이 몇 프레임 안에 종료 상태까지 달리고 반환된다.
- 이미지는 avif + webp 소스를 가진 `<picture>`와 명시적
  `width`/`height`로 레이아웃 시프트가 없다. `loading="lazy"`는
  히어로(`loading="eager"`)를 제외한 모든 이미지에.
- 미디어 페이로드 합계는 네 AVIF를 합쳐 약 843KB — 5120KB 예산 아래
  넉넉한 여유.

## 접근성 노트

- **모션 최소화:** `usePrefersReducedMotion`(`matchMedia` 위의
  `useSyncExternalStore`)이 훅 동작을 모티고, `prefers-reduced-motion`
  미디어 쿼리가 나머지를 덮는다: 네온 사인은 플리커 없이 켜진 상태를
  유지하고, 헤드라이트 궤적은 첫 프레임에 멈추고, 타이핑 커서는
  숨겨지고, 새벽 블룸은 풀 오파시티에 고정되며, 모든 `[data-reveal]`은
  종료 상태에 앉는다. 페이지는 완결되고 차분한 정적 문서다.
- **JS 없음:** 숨김 사전 상태(`[data-reveal]`, 흐린 네온 튜브, 빈
  타이핑 라인)는 전부 `.moonlit-js` 뒤에 있다. 자바스크립트가 없으면
  페이지는 완성되어 있다 — 모든 이미지 보이고, 모든 상태 라인
  완료(시각적으로 숨겨진 스팬이 온전한 문자열을 들고 있다)며, 모든
  네온 사인 켜져 있다.
- **타자기 시맨틱:** 모든 타이핑 라인은 시각적으로 숨겨진 스팬
  (`moonlit-sr`)에 온전한 문자열을 가진다 — 스크린 리더와 브라우저
  찾기를 위해. 보이는 애니메이션 스팬은 `aria-hidden`이다. 상태
  블록은 `role="status"`와 `aria-live="polite"`를 달아, 라이더 배차가
  타이핑될 때 스크린 리더가 읽어준다.
- **트래커의 정직:** 활성 단계는 타이머가 아니라 스크롤과 함께
  단조롭게 전진한다 — 페이지는 주문의 상태에 결코 거짓말하지 않는다.
- **대비:** moonlight 16.1:1 (AAA), moonlight-dim 7.8:1 (AA+),
  neon-blue 7.5:1 (본문 텍스트 기준 AAA — 활성 상태, 라이브 표시,
  섹션 헤드에 사용), ember 8.3:1, steel 5.7:1 (본문 텍스트 기준 AA —
  챕터 번호, 헤드, 비활성 단계). 전달 헤드라인은 Gugi의 큰 사이즈로
  따뜻한 배경 위에서 AA를 넉넉히 통과한다.
- **키보드:** 전 구간 네이티브 `<a>`와 `<button>`-형 앵커; 네온 블루의
  커스텀 `:focus-visible`에 후광 박스섀도. CTA 버튼은 소스 순서대로
  도달 가능하다.
- **터치:** 호버로만 의미를 갖는 것은 없다. 히어로 패럴랙스는 터치에서
  조용히 낮아진다(장식이지 정보가 아니다); 마퀴는 포인터가 아니라
  자율적으로 돈다; 트래커는 스크롤로 전진하며 터치에서도 동일하게
  작동한다.

## 리믹스 아이디어

1. **24시간 식당.** 문법 전체를 두고, 네 점의 일러스트를 카운터
   스툴의 네 순간으로 바꾼다. 호박은 전달에서 커피로 옮겨 간다 —
   페이지의 유일한 따뜻한 것이 한 끼가 아니라 컵이 된다. 트래커는
   좌석 가용성 티커가 된다.
2. **야간 교대 간호사.** 구조는 같고 업종만 바꾼다: 라이더는 방문
   간호사가 되고, 트래커는 투약 일정이 되며, 호박은 잠든 환자의 활력
   안정된 순간에만 남는. 네온 사인은 병원 복도 조명이 된다.
3. **새벽 3시의 휴게실.** 서비스업의 짝: 라이더는 여러 인물 중 하나가
   되고, 페이지는 앙상블이 되어 각자 자기의 타이핑 상태를 갖는다.
   호박은 누군가가 앉는 단 한 순간에만 계속 예약된다.
